# Program Start Time Disagrees With What Is Live: What to Record

> **In short:** Confirm the exact authorized channel and program before comparing times. Record source scheduled start and end, Norva guide times by view, device clock, time zone, guide window, and a minimal live-state observation at documented timestamps. Add one control channel and another supported device when practical. Do not copy media, estimate from memory, or assume the schedule, channel mapping, clock, or live source is automatically correct.

A program can appear to start early or late because the schedule changed, a previous program overran, the wrong channel is selected, the guide time context differs, or the observed media is another version. Evidence must identify the boundary without recording protected content.

## Confirm channel identity

Record privacy-safe source label, channel name, number or source identifier where visible, logo state, region or version cue, and guide row. Same-name channels can have distinct schedules.

The [TV guide handbook](/blog/tv-guide-troubleshooting-handbook/) provides the identity and time matrix.

## Confirm program identity

Record short title cue, episode or event cue where visible, source description cue, scheduled date, and adjacent previous and next listings. Do not copy full descriptions or rely on a presenter, logo, or scene alone.

## Capture the source schedule

Through the provider's official authorized route, record scheduled start and end, time label or zone only where explicitly shown, and confirmation timestamp. Note any source update. Do not assume an unlabeled time is local or UTC.

## Capture Norva guide times

Record the grid cell, now-and-next area, and program detail only where each displays time. A mismatch in one view is different from every guide surface showing the same value.

If now-and-next text itself remains old, use the [stale information timeline](/blog/now-next-information-stale/).

## Record device time context

Capture local clock, named time zone, UTC offset where visible, automatic or manual state, selected guide date, and visible window. If the difference is one hour across channels, use the [one-hour offset check](/blog/guide-listings-shifted-one-hour/).

## Make a minimal live observation

When the household is authorized to view the source, record only whether the recognizable program identity before and after the boundary matches the scheduled title cue. Note timestamp and uncertainty. Do not capture, upload, transcribe, or distribute the media.

## Observe a transition window

Record one point shortly before the listed start, one at the boundary, and one after it, without claiming continuous observation if gaps exist. Note whether the source schedule changed during the window.

## Add a control channel

Choose one channel with a clear scheduled transition in the same time context. Record identical fields. A control that aligns narrows the symptom but does not prove the affected schedule's cause.

## Check mapping if identities disagree

If the live program and guide listing consistently belong to different visible channels, use the [wrong-program mapping guide](/blog/wrong-program-mapped-channel/) before editing schedule times.

## Compare another supported device

Use the same account, profile, source, channel, guide window, and close timestamp. Record both clocks, zones, and application versions. Different guide times on one screen establish a cross-device difference, not its cause.

## Avoid corrective guessing

Do not manually shift schedules, rename channels, change device time, edit program titles, repeat refreshes, or clear application data before preserving the record. A manual correction can mask a channel or zone mismatch.

## Classify the discrepancy

Use source schedule changed, live transition differs from source listing, wrong channel identity, wrong program identity, one-hour context difference, one-view guide difference, one-device difference, observation uncertain, or unknown. Do not declare a provider or Norva fault without verified evidence.

## Prepare support evidence

Use the [guide issue template](/blog/guide-issue-support-evidence/) with masked identities, paired times, clock and zone, minimal observation, control channel, device versions, and action log.

## Common mistakes and limitations

- Matching channels or programs from names alone.
- Estimating the boundary from memory.
- Ignoring clock, zone, and guide window.
- Capturing or sharing protected media.
- Shifting schedule times before checking mapping.
- Treating one observation as a universal pattern.

## Frequently asked questions

### How much live content should I record?

None. Record only minimal identity and timing observations needed to compare the authorized source with the schedule.

### Does an overrun mean the guide is wrong?

It establishes a difference between scheduled and observed transition, not which system should have changed or why.

### What if every channel is one hour early?

Use the time-context checklist before investigating individual program data.
