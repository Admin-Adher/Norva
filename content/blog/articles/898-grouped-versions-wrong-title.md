# Grouped Versions Under the Wrong Title? Check Their Identifiers

> **In short:** Freeze the same account, profile, source selection, filters, sorting, grouping, device, and application version. Record the parent title and each grouped version's source label, title, year, media type, season or episode, duration, edition, language cues, artwork, synopsis, progress, favorite state, and timestamp. Compare authorized source records and another supported device. Do not infer Norva's matching algorithm or split, merge, rename, play, or delete cards before preserving evidence.

A grouped card can look wrong because an unrelated item appears beneath its parent title, an alternate version carries different metadata, or the parent presentation itself changed. The first task is to identify every visible entry.

## Capture the parent card

Record parent title, year, media type, artwork, synopsis, source summary, category, and view. Note whether the title differs between category, search, and detail pages. Do not assume the parent title is an identifier.

The [category and metadata handbook](/blog/category-metadata-troubleshooting-handbook/) provides the field matrix.

## Inventory every grouped version

For each child card or selector entry, record source, title, year, media type, season, episode, duration, edition, language, artwork, and visible version label. Use privacy-safe codes if household titles are sensitive.

## Confirm source records

Through each provider's official authorized route, verify the corresponding entry and its current metadata. Record timestamps. Source records are inputs to compare, not proof of how Norva should match or select a parent.

## Compare media type and sequence

A film, series, season, episode, special, and bonus item should remain distinct in evidence. If an episode is grouped beneath a series or another episode, include season, episode, title, and duration cues.

## Compare year, duration, and edition

Use year and runtime as identity signals, not unique identifiers. The [runtime guide](/blog/runtime-differs-between-versions/) explains how alternate cuts can differ. Record edition and language cues before calling a child unrelated.

## Compare text and artwork

Record title, localized title, synopsis, poster, and background independently. One mismatched image does not prove the underlying version is unrelated. Conversely, shared artwork does not prove identical identity.

## Protect household state

Record progress, completion, and favorite state for the parent and each child. Do not play, mark complete, unfavorite, delete, or choose several versions merely to test grouping. The [progress identity guide](/blog/progress-reset-after-item-identity-change/) protects before-and-after evidence.

## Freeze grouping and filters

Record enabled sources, availability, category, year, rating, audio, subtitles, search, sorting, and grouping. A filter can change which child supplies the visible parent presentation. Do not infer why.

## Preserve the timeline

Record source edits, import or refresh, first grouped observation, title change, application update, and current state. If a group split after refresh instead, use the [split-version guide](/blog/grouped-versions-split-after-refresh/).

## Compare another supported device

Use the same account, profile, sources, filters, grouping, and close timestamp. Record both application versions. Different grouping or title on one screen is cross-device evidence, not proof of where matching occurs.

## Avoid forced correction

Do not split, merge, rename, edit metadata, remove a source, repeat imports, clear application data, or delete a child before saving the full comparison. No regrouping or recovery outcome should be promised.

## Classify the group

Use parent title differs by view, alternate edition, localized title, different media type, wrong season or episode, unrelated source record, artwork-only mismatch, device-specific grouping, changed after source refresh, or unknown. Keep this observable classification separate from matching theories.

## Prepare support evidence

Use the [metadata evidence pack](/blog/metadata-support-evidence-pack/) with parent and child table, source confirmation, context, household state, devices, versions, timeline, and actions.

## Common mistakes and limitations

- Treating the parent title as a unique identifier.
- Matching children by artwork or title alone.
- Ignoring media type, episode, duration, or edition.
- Changing progress while testing versions.
- Splitting or deleting cards before evidence capture.
- Guessing Norva's internal matching algorithm.

## Frequently asked questions

### Can I manually split the wrong item?

Do not assume a control or outcome not documented by current Norva guidance. Preserve evidence and ask support.

### Does matching artwork prove the versions belong together?

No. Compare source, title, year, media type, duration, edition, language, and other visible identity cues.

### Should I rename the source item?

Not before preserving every original field, confirming metadata ownership, and following provider and Norva support guidance.
