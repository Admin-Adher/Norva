# How to Build a Themed Shortlist With Favorites

> **In short:** Write the theme as one operational sentence naming audience, occasion, and constraints. Set inclusion, exclusion, capacity, and expiry before choosing. Pull exact items from favorites, test identity and current authorized availability, then check the shortlist for meaningful variety. The shortlist is a temporary view; it should not rewrite durable favorite state.

“Comfort movies” or “weekend science fiction” can inspire a list, but they do not yet define one. A useful brief lets another person understand why every item belongs and when the collection ends.

Example structure: “Up to six standalone mysteries under two hours for three adults on Saturday, excluding titles anyone watched this month.” Use verified details for the real shortlist.

## Make the theme testable

Turn adjectives into observable fields. “Short” becomes a stated runtime ceiling. “Family choice” becomes named household suitability rules. “Recent” becomes a release-year window. “Available tonight” becomes a source check timestamp.

Do not infer sensitive content suitability from category alone. Review the metadata and household criteria actually required.

## Gather candidates from favorites

Start with durable favorites because they already carry a future action or preference. Add external candidates only if the shortlist’s purpose allows it, and do not automatically promote them to favorites.

Use [favorites as a discovery starting point](/blog/start-discovery-from-favorites/) when the initial list is too narrow. Keep recommendations distinct from the saved seed that produced them.

## Verify exact identity

For each candidate, record title, media type, release context, version, language, and runtime where relevant. DCMI metadata terms distinguish type, format, language, identifier, and relation, which are useful for enforcing the brief.

If two cards look alike, compare identity before counting them as variety.

## Apply inclusion and exclusion separately

An item first has to pass every required inclusion rule. Then test exclusion rules. Record the reason briefly:

- pass—all required fields verified;
- fail—runtime exceeds ceiling;
- unknown—required language track not confirmed;
- defer—source availability not checked.

Do not convert unknown to pass to fill the list.

## Check variety without tokenism

Variety can mean media type, duration, tone, release era, language, or familiarity. Choose dimensions relevant to the occasion. Avoid adding an item merely to make the grid look different when it fails the theme.

Run a pairwise check on the final candidates: each item should satisfy the theme for a distinct, stated reason. If two items offer the same role, keep the better-verified option unless deliberate repetition is part of the brief.

## Separate shortlist and favorite state

Removing an item from the themed collection should not unfavorite it. Adding a recommendation to the shortlist should not make it a durable favorite. W3C label guidance supports controls that state whether an action changes the collection, the favorite, or both.

For a same-day decision, use [the tonight shortlist](/blog/create-tonight-shortlist/) and its tighter expiry. For durable admission, use [the favorite decision card](/blog/decide-what-to-favorite/).

Norva can organize favorites and source metadata across supported devices under the same account, subject to current behavior. Verify whether the product supports custom shortlists or use a private temporary note without assuming a feature.

## Close the shortlist

At expiry, record selected, not selected, unavailable, and unresolved. Delete the temporary collection or note. Review only candidates whose future action changed; leave the underlying favorites untouched.

## Common mistakes and limitations

- Using a vague theme with no constraints.
- Choosing before setting list capacity.
- Treating category labels as complete evidence.
- Mixing recommendation and favorite state.
- Assuming current availability will persist.
- Keeping an expired shortlist indefinitely.

## Frequently asked questions

### How many items belong in a themed shortlist?

Choose the smallest capacity that provides a real choice without recreating the full favorites list.

### Can one item appear in several themes?

Yes. The lists express different temporary purposes; preserve one underlying identity.

### Should every shortlisted item be available now?

Only if the brief requires immediate viewing. Otherwise, label availability separately.
