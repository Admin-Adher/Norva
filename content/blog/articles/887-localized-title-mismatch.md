# Localized Title Mismatch? Compare Language and Source Metadata

> **In short:** Confirm the same logical item and version before comparing titles. Record source label, year, media type, season or episode, duration, original title, localized title fields, Norva title in each view, interface and profile language, audio and subtitle cues, device, application version, and timestamp. Change one supported language context at a time, avoid editing source metadata first, and never infer an undocumented title-selection rule from one sample.

A title can vary by language, territory, edition, source, or view. An unexpected translation is not necessarily the wrong item, and a familiar title is not enough to prove identity.

Keep the original wording available until every compared language context has been recorded and reviewed carefully.

## Confirm item identity

Compare source label, year, media type, duration, edition, season, episode, artwork, and version cues. If those fields differ, first investigate whether the screens show different entries.

The [category and metadata handbook](/blog/category-metadata-troubleshooting-handbook/) provides the identity matrix.

## Record each source title field

Through the provider's official authorized route, record the original, display, localized, alternate, season, and episode titles where the source exposes them. Use the source's own labels. Do not assume one field is universally primary or that Norva consumes every field.

## Record the Norva title by view

Capture the title in category, search, detail, episode list, favorite, progress row, and grouped-version view only where relevant. A mismatch limited to one view is different from a title used everywhere.

## Record language context

Note device language, Norva interface language, active profile, source metadata language, audio badge, subtitle badge, and visible track list. Language metadata and playback tracks are related observations but do not prove which title should appear.

The [audio badge guide](/blog/audio-badge-disagrees-track-list/) explains how to keep badges separate from track evidence.

## Preserve exact text safely

Copy capitalization, accents, punctuation, articles, transliteration, season numbering, and parenthetical edition cues. For a sensitive household item, use a redacted sample in support material while retaining enough distinguishing characters privately.

## Change one language context

If current Norva controls allow it, switch one supported interface or profile language, record the result, then restore baseline. Do not change device, profile, source, filters, and language together. Never assume the observed change reveals Norva's internal selection hierarchy.

## Compare another supported device

Use the same account, profile, source, item, filters, grouping, and language context close in time. Record both application versions. A title difference on one device is cross-screen evidence, not proof of a local language cache.

## Verify year and version alongside title

An alternate release year or duration may identify another edition rather than a localization error. Use the [release-year verification](/blog/incorrect-release-year-metadata/) before requesting a title correction for an uncertain version.

## Preserve the timeline

Record source title confirmation, source edit if any, import or refresh, first Norva observation, language changes, application updates, and current state. Use current official timing guidance only when published.

## Avoid premature corrections

Do not overwrite original or localized source fields, rename files, merge versions, or repeat imports before saving the comparison. Establish who owns the metadata and whether the provider supports corrections. No persistence across future updates should be promised.

## Classify the mismatch

Use different item or version, source original title, source localized field, interface-language difference, profile-language difference, view-specific title, device-specific title, source changed after import, or unknown. Keep the classification separate from any guessed selection logic.

## Common mistakes and limitations

- Comparing different editions or episodes.
- Treating track language as proof of title language.
- Ignoring view, profile, and interface context.
- Editing source titles before preserving the baseline.
- Assuming one sample reveals a selection hierarchy.
- Sharing sensitive household titles unnecessarily.

## Frequently asked questions

### Which title should Norva display?

Use current Norva documentation for any rule. Otherwise record source fields and visible context without promising a selection outcome.

### Does audio language control the title?

Do not infer that relationship. Record audio, subtitle, interface, profile, and metadata language as separate evidence.

### Should I replace the original source title?

Not before confirming metadata ownership, preserving every existing field, and checking provider and Norva support guidance.
