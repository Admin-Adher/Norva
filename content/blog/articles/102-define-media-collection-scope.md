# How to Define the Scope of a Personal Media Collection

> **In short:** Define scope with explicit inclusion and exclusion rules across media type, source, ownership or authorisation, household users, language, time period, and active versus archive status. Give uncertain items a temporary review queue. A good scope makes the first project finishable without pretending the rest of the collection does not exist.

Scope is a decision boundary, not a description of everything you own. It says which material receives attention now and why. Without that boundary, every exception can expand a weekend project into an endless migration.

## Write one scope sentence

Use this pattern:

> This phase covers **[media types]** from **[authorised sources]** used by **[people]** for **[purpose]**, while **[explicit exclusions]** remain outside until **[review date]**.

The sentence should allow another household member to classify most items without asking what you meant.

Start from the purpose in [the personal media collection planning guide](/blog/plan-personal-media-collection/). If the scope does not support that purpose, change the scope rather than adding work.

## Set seven boundaries

### 1. Media type

Decide whether the phase includes films, episodes, recordings, home videos, audio, or another clearly named class. Different media types may need different metadata and browsing structures.

### 2. Source

List each compatible source included. Record who owns it or is legally authorised to use it. Keep unverified sources outside the active plan until that status is resolved.

### 3. Household users

Name the people or viewing contexts the organisation must serve. A single-person catalogue can use different rules from a shared collection with profiles and several language needs.

### 4. Language

State whether language refers to original audio, preferred audio, subtitle availability, interface language, or descriptive metadata. These are not interchangeable.

### 5. Time

Limit the first phase by acquisition period, last use, release period, or another observable boundary. Avoid vague labels such as “old” unless the plan defines them.

### 6. Activity

Separate items used regularly from items retained for occasional or archival reasons. The [active-versus-archive framework](/blog/separate-active-and-archive-collections/) helps avoid deleting valuable but infrequently used material.

### 7. Decision authority

State who may rename, merge, archive, or remove. In a shared household, visible ownership prevents one person's cleanup from undoing another person's organisation.

## Create an explicit exclusion list

An exclusion list is constructive. Examples include:

- source not yet inventoried;
- rights or authorisation not confirmed;
- media type needing a different workflow;
- damaged or unreadable items requiring specialist review;
- duplicates awaiting comparison;
- family material requiring consent;
- archive material outside this phase.

Exclusion does not mean deletion. It means “not changed under this plan.”

## Use a review queue for uncertain cases

Create one temporary category or worksheet with:

- item or group identifier;
- reason it is uncertain;
- person who can decide;
- evidence needed;
- review date;
- permitted interim action.

Do not leave uncertainty scattered across informal notes. A central queue protects momentum without forcing irreversible choices.

## Test the boundary against edge cases

Choose twelve sample items:

- clear inclusions;
- clear exclusions;
- items spanning two media types;
- duplicates across sources;
- multilingual versions;
- one inactive but valued item;
- one item with uncertain authorisation.

Ask two people to apply the rules independently when the collection is shared. Any disagreement reveals a boundary that needs clearer wording.

## Common mistakes and limitations

- Defining scope as “everything.”
- Including a source before confirming ownership or authorisation.
- Using “multilingual” without defining which language fields matter.
- Treating archived as unwanted.
- Letting every household member apply different hidden rules.
- Deleting exclusions instead of preserving them for review.
- Expanding scope whenever an interesting exception appears.

Norva can organise a compatible source the user owns or is legally authorised to use, but it cannot resolve ownership, rights, or missing source metadata on the household's behalf.

## Frequently asked questions

### Can scope change later?

Yes. Version the scope, record why it changed, and retest edge cases. Do not silently rewrite the boundary during active cleanup.

### How narrow should the first phase be?

Narrow enough to complete and evaluate. One media type or one well-understood source is often a useful pilot.

### What happens to out-of-scope items?

Leave them unchanged, document where they remain, and give them a future review trigger if they still matter.
