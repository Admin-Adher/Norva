# How to Evaluate Norva for Three to Five Profiles

> **In short:** First prove that the household genuinely needs three to five distinct viewing contexts. Current editorial guidance describes Norva Family as supporting up to five profiles, but verify the official plan at decision time. Test label recognition, shared-screen selection, separate progress and favorites, one shared-title collision, preferences, screen handoffs, administration, privacy, and correction before subscribing.

Capacity is only the first question. A five-profile setup that people cannot identify or maintain may create more confusion than a smaller, carefully designed household model.

Use the [Norva evaluation framework](/blog/norva-evaluation-framework/) to set non-negotiable household requirements before building the simulation.

## Map contexts before creating profiles

List every proposed context and its recurring purpose. Examples might include personal viewing, a shared household context, or a supported language preference. Merge duplicates only when progress, favorites, and preferences can genuinely be shared without conflict.

Do not create profiles for devices or rooms. Profiles represent viewing contexts; devices should be managed separately.

## Verify the current plan

Check the official pricing and terms pages for current profile limits, billing, and cancellation details. Editorial product guidance currently identifies up to five profiles for Norva Family, but this claim requires human verification before publication.

Do not assume that a profile count also specifies simultaneous playback, device, or source limits. Verify every relevant limit independently through current official information.

## Create privacy-safe labels

Use the [first household profile guide](/blog/create-first-household-profile/) to produce short, distinct labels without full names, birth years, schools, workplaces, health details, locations, or source information. View the set from the normal television distance.

If two labels truncate to the same visible text, rename them before testing. Color or avatar alone should not carry the entire distinction.

## Run a selection test

Start from the shared screen and ask every participant to choose the intended profile and reach a known item. Rotate the order so later participants do not simply copy the path. Record time, actions, wrong-profile entries, and requests for help.

Use the worst result as well as the average. A household workflow can fail when one participant cannot reliably select the correct context.

## Test state separation

Give each profile one neutral test item, distinctive progress point, and favorite. Choose supported audio or subtitle preferences only for media known to contain those tracks. Inspect each profile and record any collision or ambiguous state.

Profiles organize viewing context, but should not be assumed to be separate security accounts. Review actual visibility on shared screens.

## Test a shared-title collision

Select one work that at least three profiles may view. Create different progress positions or preferences and verify them after profile switches. Confirm that all participants use the same version so variant differences do not invalidate the comparison.

Record correction effort after one intentional wrong-profile session. Do not delete entire histories or recreate profiles without preserving the result.

## Test personal and shared screens

Move one profile from the shared TV to a supported personal screen, then back. Verify account, profile, item identity, actual resume position, and favorite state. Repeat with a second profile only if it represents a different device or preference condition.

More devices increase account and trusted-session administration. Include device review in the workload score.

## Assign household administration

Define who controls the account, sources, billing, profiles, device records, and support cases. Also define who can request a rename or deletion and how that decision will be recorded. Avoid a single informal administrator whose absence makes the system impossible to maintain.

Review privacy disclosures, support routes, and retained setup evidence. Shared households should minimize catalog and identity details in screenshots.

## Compare benefit with complexity

Use the current price and terms. Measure whether profiles reduce collisions enough to justify selection steps, labels, device management, support, and household training. A larger plan is valuable only when it enables passed essential workflows.

Complete the [trial evaluation scorecard](/blog/norva-trial-evaluation-scorecard/) before the decision deadline with input from all affected viewers.

## Frequently asked questions

### Should every household member receive a profile?

Only when a distinct recurring context is useful. A profile created without a purpose adds selection and maintenance effort.

### Are profile limits the same as device limits?

Do not assume so. Check each current limit and term separately on official Norva pages.

### What if one participant cannot use the profile picker reliably?

Treat it as an essential usability finding. Simplify labels and contexts, retest, and reject the setup if the intended viewer still cannot select safely.
