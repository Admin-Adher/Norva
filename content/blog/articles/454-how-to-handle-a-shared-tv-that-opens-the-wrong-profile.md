# How to Handle a Shared TV That Opens the Wrong Profile

> **In short:** Do not press play. Read the visible account and profile label, return to profile selection through documented controls, and choose the intended context before opening media. After viewing, leave the TV at an agreed neutral state and test one reopen. If the wrong profile repeatedly appears, record exact startup and exit states rather than assuming an automatic-selection rule.

A shared TV may reopen the last visible screen, a remembered profile, or another state depending on current product and device behaviour. The first observed profile is not necessarily the intended viewer.

Tell the next viewer not to interact while the startup state is captured. Even opening a detail page can replace the first visible evidence and make a repeat incident harder to compare.

## Freeze before playback

Keep the screen unchanged and record:

- visible profile label;
- focused control;
- item or continuation card shown;
- whether the app was already open;
- previous household user's exit action;
- TV and app version if visible.

Do not press Back or select repeatedly before preserving the first state.

## Confirm the account layer

Verify that the TV is signed into the intended household account. An unfamiliar library can indicate the wrong account, not merely the wrong profile.

Account sign-in and profile selection need separate correction paths.

## Return to profile selection

Use the visible profile or account navigation control. Move remote focus one step at a time and identify the highlight before activation.

W3C focus-visible guidance describes why keyboard focus should be perceivable. The actual TV remote focus must be observed on the current Norva interface.

Use [the active-profile check](/blog/how-to-confirm-the-active-profile-before-playback/) before resuming any item.

## Choose the intended profile

Read the text label and use icon or colour only as supporting evidence. Then verify the candidate item, version, proposed position, audio, and subtitles.

Do not use the first continuation card to identify the profile; two household members can watch the same series.

## Define a neutral exit

Agree whether users should return to profile selection, a library home, or sign out when the TV is not trusted. Use only a state the current interface supports.

The [complete household profile guide](/blog/the-complete-guide-to-household-media-profiles/) covers shared-screen naming and handoff conventions.

## Test one reopen

After leaving the agreed state, close and reopen through normal documented controls once. Record the first profile and focus state. Do not declare a rule from one observation; this becomes the baseline.

## If the problem repeats

Use [the profile-specific support report](/blog/how-to-document-a-profile-specific-problem-for-support/) with exact steps, expected and observed results, versions, and a privacy-safe screenshot. Remove profile names, account email, private history, and source details.

Do not clear app data or reset the TV before support evidence is captured.

Also compare whether the incident occurs after an ordinary exit, device standby, or full app close, but test those paths in separate runs. Combining them prevents a clear conclusion.

## Common mistakes and limitations

Avoid pressing play to discover the profile, recognising by avatar alone, changing several exit settings, and assuming the app must always open profile selection.

TV platform and app lifecycle behaviour can vary. This workflow prevents accidental state changes and documents repetition; it does not guarantee a specific startup screen.

## Frequently asked questions

### Should the TV always ask for a profile?

Do not claim that without current product documentation. Verify the visible startup behaviour and use the household pre-play rule.

### Can I lock the TV to one profile?

Do not assume a profile-lock feature. Profiles are organisational contexts, not passwords.

### Should I sign out after every session?

That depends on device trust and household policy. A neutral profile-selection state may be sufficient on a trusted TV.
