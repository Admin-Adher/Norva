# Search or Filters: Choose the Faster Discovery Route

> **In short:** Use search when you remember an identity clue: title fragment, person, series, or alternate name. Use filters when you want any item satisfying attributes: source, type, category, year, availability, audio, or subtitles. Combine them only after the first route produces a set worth narrowing, and switch when zero or excessive results show the question was framed incorrectly.

The control should match the task. “Find this film” is different from “choose any available French-language comedy tonight.”

## Use the identity-versus-attribute route card

| Your question | Known evidence | First route | Second route if needed |
|---|---|---|---|
| “Where is this title?” | title/person/series clue | Search | filter by type/year/source |
| “What can we watch?” | acceptable attributes | Filters | sort or search within set |
| “Which version is this?” | work identity plus source/track | Search | version filters/details |
| “What exists in this category?” | category is the subject | Category browse/filter | search within category |

If you cannot phrase the question, list the clue you possess and the choice you need to make.

## Choose search for identity

Start with search when you know:

- part or all of the title;
- a cast or creator name;
- original or translated title;
- series name or episode fragment;
- an approximate year paired with a title;
- a distinctive metadata clue.

Search can reveal the stored wording and record type before you commit to filters. Use [global-versus-category search](/blog/global-search-vs-category-search/) to choose scope.

## Choose filters for acceptable sets

Start with filters when the goal is:

- only currently available choices;
- one media type;
- a broad category or year range;
- a required audio or subtitle option;
- favourites that remain available;
- one authorised connected source.

Filters are strongest when the field is reliable and the requirement is non-negotiable. A filter based on incomplete metadata can hide valid candidates.

The Library of Congress Basic Search guide explains that facets narrow large results but can exclude records lacking the field used for the limit. That warning applies conceptually to any metadata-driven filter.

## Combine routes in two passes

### Search then filter

Use when a known clue returns many works. Search a person, title fragment, or series, then narrow by type, year, or source.

### Filter then search

Use when the acceptable set is firm. Select availability or source, then search a title fragment inside that set if the interface supports scoped search.

Keep the first route visible. If the combined result is empty, remove the second condition before changing the first.

## Know when to switch

Switch from search to filters when:

- results contain the right theme but too many candidates;
- one reliable attribute separates them;
- spelling or title form is no longer the uncertainty.

Switch from filters to search when:

- the filtered set remains very large;
- you remember a distinctive identity clue;
- several filter combinations produce the same candidates;
- the desired item may lack one filtered field.

Use [the filter strategy guide](/blog/media-filter-strategy-guide/) for multi-condition browsing.

A control is useful when it removes a meaningful uncertainty. Going from 300 to 200 results may be less valuable than separating films from episodes.

## Do not confuse filtering with sorting

Filtering removes candidates that do not meet a condition. Sorting keeps the set but changes order. If every candidate remains acceptable and you merely want the newest or most relevant first, use [the filtering-versus-sorting guide](/blog/choose-filtering-or-sorting/).

Norva can organise and filter compatible authorised sources, but field availability and search behavior depend on source metadata and current product support.

## Common mistakes and limitations

- Using filters to guess a forgotten title.
- Searching broad words when requirements are attribute-based.
- Applying search and many filters simultaneously.
- Forgetting which condition caused zero results.
- Filtering by incomplete language or category metadata.
- Sorting when the goal is to remove candidates.

Neither route can compensate for absent records or wrong metadata. Use controls and diagnostics when expected candidates remain missing.

## Frequently asked questions

### Is search always faster for one title?

Usually when the title clue is reliable. A very short common title may benefit from a year or type filter after the baseline.

### Should availability be the first filter?

Only when current availability is required. Otherwise, it may hide a title you want to identify or troubleshoot.

### Can I search inside filtered results?

Only if the interface supports scoped search and clearly preserves the active set. Test with a known control item.
