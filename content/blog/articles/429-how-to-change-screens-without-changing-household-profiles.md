# How to Change Screens Without Changing Household Profiles

> **In short:** Record the active source profile before leaving the first screen, then select and verify that same intended profile on the target before opening a continuation card. Only after profile identity matches should you compare item, version, and progress. If playback starts under the wrong profile, pause immediately and preserve the evidence before switching.

A screen change and a profile change are separate actions. Combining them can place progress, history, favourites, or preferences in another household member's context.

## Identify the profile on the source

Open a stable profile or account area and record the visible profile name or icon. Avoid relying on colour or avatar alone when several profiles look similar. Do not include private personal data in a note that will remain on a shared screen.

Norva can preserve progress, history, favourites, and preferences under the same account across supported devices. Profiles separate those viewing contexts when the current plan provides them.

## Pause before leaving

Pause the item and record title, episode, version, and approximate position. Keep the source profile unchanged during the transition.

The [cross-device handoff state guide](/blog/a-state-by-state-guide-to-cross-device-viewing-handoff/) explains why account, profile, source, and item should be checked in dependency order.

## Select the target profile first

Open Norva on the supported target and go to the visible profile selection or account context before opening media. Read the profile label, then compare it with the source record.

W3C consistent-identification guidance reinforces the value of identifying repeated functions consistently, but actual profile naming and navigation must be observed on the current interface.

Use the [active-profile preflight](/blog/how-to-confirm-the-active-profile-before-playback/) if the target does not make profile context obvious.

## Rebuild the media state

Once the profile matches:

1. confirm authorised source access;
2. find the exact item or episode;
3. select the intended version;
4. compare progress;
5. verify audio and subtitles;
6. check target audio output;
7. resume once.

Do not use a continuation card as evidence of the active profile. The content of the card may be stale or visually similar across profiles.

## Handle a wrong target profile

If the wrong profile is visible before playback, return to profile selection through documented controls and choose the intended one. If playback already began, pause immediately and note the item, position, and duration of accidental activity as accurately as possible.

Do not clear another person's full history or change their preferences to hide the mistake. Follow [the wrong-profile progress prevention workflow](/blog/how-to-prevent-progress-from-landing-on-the-wrong-profile/) and contact support when corrective controls are unclear.

## Close the source boundary

After the target resumes correctly, leave the source paused, close it, or sign out according to the device and household routine. Do not start both screens merely to compare profiles unless current terms and source conditions allow the intended use.

## Household handoff convention

Agree on one verbal or written convention: “screen, profile, item.” Before a handoff, say or record those three fields in that order. This makes the profile part of the request rather than an assumption.

For shared TVs, choose profile names that are clear without exposing unnecessary personal information.

## Common mistakes and limitations

Avoid selecting media before checking the profile, recognising profiles by colour alone, changing profiles on the source during handoff, and confusing account sign-in with profile selection.

Profiles organise viewing context; they do not guarantee privacy from anyone who can access the same signed-in account. Device support and current plan capacity also apply.

## Frequently asked questions

### Can I change devices and profiles in one handoff?

You can choose to, but that is a new viewing context. Record the intent explicitly instead of treating it as continuity.

### Is the profile icon enough to verify identity?

Use the visible name or another stable label as primary evidence. Icons can be similar or changed.

### What if the target opens the wrong profile automatically?

Pause before playback, return through documented controls, and select the intended profile. Report repeated ambiguity if the interface does not show context clearly.
