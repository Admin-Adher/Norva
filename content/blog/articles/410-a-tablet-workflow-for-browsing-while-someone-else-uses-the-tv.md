# A Tablet Workflow for Browsing While Someone Else Uses the TV

> **In short:** Use the tablet as a separate planning screen: confirm your own profile, browse without starting playback, save only deliberate choices, and leave the TV session untouched. If you later hand off or start on another device, verify current plan terms, source access, item identity, and progress rather than assuming simultaneous playback is permitted.

This workflow is useful when one household member is watching and another wants to explore a shared catalogue. The key is separating “planning” from “playing.” A tablet can help review titles, episodes, favourites, and variants without becoming an accidental remote for the TV session.

## Assign each screen one role

State the roles aloud or in a note:

- **TV:** active session owned by the current viewer.
- **Tablet:** browsing and planning under the intended tablet user's profile.
- **Later target:** optional screen for a future session or handoff.

Do not treat profile capacity as a device or simultaneous-playback limit. Norva and Norva Family describe profile allowances, while current prices and conditions belong on the official pricing and terms pages. Verify any concurrent-use rule before relying on it.

## 1. Confirm the tablet profile

Open Norva on the supported tablet and check the active account or profile before browsing. Separate profiles help preserve distinct history, favourites, progress, and preferences when the current plan provides them.

If the correct profile is unavailable, stop rather than borrowing the TV viewer's context. The goal is to prevent a planning task from changing another person's recommendations or history.

**Observable result:** the tablet visibly shows the intended profile before any library action.

## 2. Enter the library without starting media

Choose the relevant library section and browse using filters or search. Avoid play, resume, or episode activation while the tablet remains a planning screen. Opening a detail page is usually sufficient to inspect available metadata, but use only controls currently shown.

The [tablet viewing workflow guide](/blog/the-complete-guide-to-tablet-viewing-workflows/) explains how to preserve filters and return context in a large grid.

## 3. Build a short candidate list

Select no more than a few candidates and record why each fits: intended viewer, approximate duration if shown, series position, available language or subtitle tracks, and source/version label. Do not infer missing metadata.

Reviewing favourites can be a useful starting point; the [larger touch-screen favourites workflow](/blog/how-to-review-favorites-on-a-larger-touch-screen/) separates active choices from an overgrown saved list.

**Observable result:** each candidate has a visible identity and one reason for inclusion.

## 4. Avoid controlling the TV indirectly

Do not activate cast, mirror, remote-control, or output-selection controls unless the product and devices explicitly support the intended action and the current TV viewer agrees. Browsing on a second screen is not the same as controlling the first screen.

Keep tablet audio muted only if that is appropriate, or leave media stopped. A preview can still create unexpected sound or progress.

## 5. Prepare a future handoff

If the candidate will move to the TV later, capture:

- title and year, or series, season, and episode;
- selected version or source label;
- intended profile;
- whether playback should start fresh or resume;
- any required audio or subtitle choice.

Then close the tablet detail view without starting it. The [cross-device handoff state guide](/blog/a-state-by-state-guide-to-cross-device-viewing-handoff/) explains what must match on the later target.

## 6. Close the planning session

Confirm that the TV continued under its original viewer and that the tablet did not start playback. If the tablet is shared, return it to the agreed screen or sign out as appropriate. Do not leave a private account open for convenience.

Norva states that the same account can preserve catalogue, progress, history, favourites, and preferences on supported devices. That continuity makes accurate profile and item selection more important, not less.

| Tablet |  | Browse and plan | Search, filter, inspect, save deliberately | Playback would begin |

After the task, add two checks: “TV state unchanged” and “tablet playback not started.” This record is a behavioural boundary, not proof of any simultaneous-use entitlement.

## Common mistakes and limitations

Avoid using the same profile for convenience, pressing resume to verify an item, assuming a tablet is automatically a TV remote, changing a shared favourite without agreement, or confusing profile count with concurrent playback permission.

Source access, rights, plan conditions, supported devices, and network state can affect what happens next. This workflow supports planning while the TV is occupied; it does not promise that two playback sessions may run together.

## Frequently asked questions

### Can two people play different items at the same time?

Do not infer that from profile capacity or cross-device sync. Check the current pricing, terms, source conditions, and supported-device behaviour before planning simultaneous playback.

### Will opening a detail page change progress?

A detail review should be treated separately from playback, but verify actual behaviour. Avoid activating play or resume during a planning-only session.

### Should both people use the same profile?

Separate profiles are preferable when independent history, favourites, progress, and preferences are wanted and the current plan provides them.
