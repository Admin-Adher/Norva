# When to Group Multiple Versions of the Same Title

> **In short:** Group entries when you can establish that they represent the same work and when the differences—such as language or another source-supplied attribute—remain useful at selection time. Keep remakes, sequels, similarly named works, and ambiguous records separate. Diagnose accidental duplicates before treating them as intentional versions.

Grouping can turn a noisy catalogue into one understandable choice. Done carelessly, it can hide meaningful differences or combine unrelated media behind the same artwork.

## Apply the identity test first

Before comparing quality or language, ask whether the records represent the same underlying work. Review the evidence available from the connected source:

- title and alternate title;
- release year;
- series, season, and episode identity where relevant;
- duration, when supplied;
- description and credited people;
- source identifiers or version labels.

No single field is conclusive. Matching artwork is especially weak evidence because artwork can be reused or assigned incorrectly.

If identity is uncertain, stop and use the [duplicate-title diagnostic](/blog/diagnose-duplicate-media-titles/). A reversible pause is safer than an incorrect merge.

## Apply the difference test second

Once identity is established, list what distinguishes the entries. The difference should help a viewer choose. Examples can include language availability, edition, source-supplied format information, or another clearly labelled attribute.

Norva can group variants from a compatible media source. The source and media determine which attributes and tracks are actually available; grouping does not improve or create them.

Use the [audio and subtitle badge guide](/blog/audio-subtitle-language-badges/) when language labels are part of the choice.

## Use the identity-difference matrix

| Same underlying work? | Difference useful at selection? | Recommended action |
| --- | --- | --- |
| Yes | Yes | Group as versions and preserve clear labels |
| Yes | No | Investigate whether one record is an accidental duplicate |
| No | Yes or no | Keep separate |
| Uncertain | Unknown | Do not group; inspect metadata first |

This matrix is deliberately conservative. It reduces catalogue noise without pretending that a title match is enough to establish identity.

## Choose a representative version carefully

A grouped title needs a default or representative entry in the browse view. Choose it using stable, understandable criteria rather than an assumed “best” version:

- currently available through the source;
- carries the household's required language option;
- clearly labelled;
- has complete metadata;
- matches the version most often used by the relevant profile.

Do not claim that a technical label guarantees visible quality. [What determines video quality](/blog/what-determines-video-quality/) explains why the complete playback chain matters.

## Keep the version choice visible

Grouping is successful only if the viewer can still answer:

- How many versions are present?
- What distinguishes them?
- Which one is selected?
- Can I switch without losing my place in the catalogue?

Use short, factual labels. “English audio” is clearer than “Version B.” Avoid labels such as “best” or “premium” unless they have a precise, verified meaning.

## Cases that should usually remain separate

- A remake and the original.
- A film and its sequel.
- Different episodes sharing a generic title.
- A theatrical work and an unrelated documentary about it.
- Two records with conflicting years and descriptions.
- Items whose only common evidence is artwork or a short title.

Series require extra caution. Grouping episodes can destroy season order, so identity should include the series, season, and episode context.

## Review grouped titles after source changes

Catalogue information can change when the connected source updates. Periodically sample grouped items and check:

- whether all versions still resolve to the same work;
- whether one has become unavailable;
- whether labels still distinguish them;
- whether a new version belongs in the group;
- whether progress and preferences point to the intended option.

Do not rebuild every group after a minor metadata change. Review the affected title and preserve stable structures when they remain accurate.

## Common mistakes and limitations

- Grouping by title text alone.
- Hiding version differences behind identical labels.
- Treating a duplicate record as a useful edition.
- Assuming a resolution or format label guarantees playback quality.
- Combining episodes that should remain in sequence.
- Failing to recheck a group after source metadata changes.

## Frequently asked questions

### Are two entries with the same title always versions?

No. They may be different works, remakes, episodes, or accidental duplicates. Establish identity with multiple metadata fields before grouping.

### Should an unavailable version stay in the group?

It can remain if its presence and availability state are useful and clearly shown. If the goal is a playable-only browse view, use an availability control rather than changing identity.

### Does grouping combine the media files?

Grouping is an organisational presentation. It does not change the underlying source or create new media attributes.
