# Choose the Most Practical Screen for Source Setup

> **In short:** Choose a trusted, private, supported device where you can inspect an address accurately and enter credentials from protected storage without observers. A computer may help compare fields, mobile may offer a password manager, and TV reflects the environment, but controls vary. Verify current Norva behavior and never assume a source added on one screen synchronizes everywhere. Complete setup privately, test on the intended device, and remove clipboard or temporary copies.

Norva currently advertises Web, mobile, and TV availability on supported devices. Platform availability does not mean every model, version, or setup path behaves identically.

## Start with device trust

Exclude public computers, borrowed phones, hotel televisions, unmanaged work devices, and screens with unknown software. Use an updated device under the account owner's control with a screen lock and secure network.

CISA recommends timely updates, strong account protection, and phishing awareness.

## Compare privacy of entry

A television can be visible across a room and may show an on-screen keyboard. A computer can be screen-shared or monitored by browser extensions. A phone can expose clipboard and notification previews. Choose the environment whose risks you can control.

The [secure details guide](/blog/collect-source-details-securely/) describes protected transfer from a password manager.

## Compare input accuracy

Long source addresses can contain punctuation, mixed case in paths, ports, and encoded characters. A physical keyboard and large text view can reduce entry mistakes. A television remote may make case and symbol entry slower.

Do not use convenience as a reason to send the address or password through unprotected messaging.

## Compare password-manager support

Prefer a device where the authorized user can reveal or autofill the source credential without copying it into a note. Verify that autofill targets the correct field and application. Disable screen recording and avoid shared clipboard features when they create exposure.

Never install an unknown keyboard or password tool solely for setup.

Carefully check that accessibility needs are met before entry begins.

## Verify current cross-device behavior

Check whether source settings are account-synchronized, device-local, or require setup on each platform. Norva's privacy notice states that source settings are used for connection, but this draft does not claim an exact propagation model.

Record the observed result rather than assuming Web setup changes TV automatically.

## Include the final playback environment

Even if entry is easiest on another screen, verify source label, catalog, playback, audio, subtitles, and profile behavior on the intended supported television or mobile device. Network and codec conditions can differ.

The [catalog-baseline guide](/blog/baseline-before-second-source/) provides a small repeatable verification sample.

## Choose a privacy-safe label

Create the label before the credential screen so no endpoint detail is copied into it. The [privacy-safe source naming guide](/blog/privacy-safe-source-display-name/) prevents host, owner, location, and provider information from appearing on shared screens.

## Plan cleanup and recovery

After setup, clear temporary clipboard items where supported, remove private notes and screenshots, close account pages, and verify password-manager sharing. Keep the official recovery route and protected master credential.

If entry fails, use the [credential-error guide](/blog/credential-entry-error-without-exposure/) on the same controlled device before switching environments.

## Common mistakes and limitations

- Using a public or borrowed setup device.
- Entering secrets while screen sharing.
- Assuming every platform supports autofill.
- Sending long credentials through chat for convenience.
- Assuming source settings synchronize everywhere.
- Skipping final-device playback verification.
- Leaving clipboard, screenshots, or browser tabs open.

## Frequently asked questions

### Is a computer always best for setup?

No. It can improve input accuracy, but trust, privacy, current support, password-manager access, and final-device verification all matter.

### Can I add the source on mobile and use it on TV?

Verify current Norva cross-device behavior; do not assume the configuration propagates automatically.

### Should I photograph the TV error?

Only after removing or obscuring source, account, notification, and credential details, and share it through official support.
