# A Norva Evaluation Checklist for Solo Viewers

> **In short:** A solo viewer does not need household complexity to benefit from organization, preferences, or cross-screen progress, but should verify that those gains solve real friction. Test one authorized source, a representative catalog sample, the primary screen, any important secondary screen, playback, resume, language needs, privacy, and recurring effort. Compare the result with the current workflow and current Norva plan terms.

Solo viewing removes profile collisions but not catalog, version, device, or continuity problems. The right question is whether Norva reduces enough recurring effort to justify another account and software layer.

Use the [Norva evaluation framework](/blog/norva-evaluation-framework/) to mark requirements before interacting with the interface.

## Describe the current friction

List the three tasks that consume the most effort today. Common examples include remembering an episode, choosing among versions, finding a known title, moving from TV to phone, or selecting the same subtitle track repeatedly. Record how often each issue occurs.

Do not count hypothetical features as benefits. A capability matters only when it improves a real task at an acceptable cost.

## Confirm source readiness

Verify that the compatible source is controlled and legally authorized for the intended use. Check what catalog, metadata, variants, audio, and subtitles it actually supplies. Norva does not include media or rights.

If there is only one small, stable list, a manual note or the source's direct interface may remain sufficient. The [manual progress comparison](/blog/norva-vs-manual-viewing-progress-tracking/) provides a measured alternative.

## Test catalog value

Choose five known items across movies, series, and a variant case if available. Measure the time and actions needed to find each, understand its hierarchy, and choose the intended version. Reset filters between tasks.

Score whether organization reduces ambiguity. A visually attractive grid without faster or more reliable selection should receive no extra functional credit.

## Test the primary screen

Run the complete viewing path on the screen used most: sign in or open, select profile if required, find the item, start, pause, seek, choose tracks, exit, and resume. Record input actions, errors, and recovery steps.

For TV, test remote focus and Back behavior. For web, test the intended browser and input method. For mobile, include a normal interruption and return.

## Test screen switching only if needed

If the viewer genuinely moves between screens, create a distinctive position on one and inspect the displayed and actual resume positions on another. Also test one favorite or preference transition.

Do not award continuity points merely because the same catalog is visible. State, item identity, and navigation effort are the useful outcomes.

## Test personal preferences

Use known media containing the desired audio or subtitle tracks. Set the preference, reopen the item, and test a second compatible title. Track availability still depends on the source, media, device, rights, and supported behavior.

A solo profile can simplify personal choices, but a missing source track cannot be solved by profile configuration.

## Review privacy and maintenance

Read current privacy disclosures and map account, source configuration, device, viewing-context, support, and conditional offline data. Decide whether the value justifies that footprint. Review how often the source, devices, and app will need attention.

Do not assume that being the only viewer eliminates shared-screen or device-loss exposure.

## Compare current price with measured value

Use the official pricing page at decision time. Compare the plan with the frequency and impact of passed tasks, not with the number of features advertised. Include setup, troubleshooting, and cancellation terms.

Use the [trial scorecard](/blog/norva-trial-evaluation-scorecard/) before any deadline and reject the fit if an essential source, playback, accessibility, or privacy requirement fails.

## Frequently asked questions

### Is Norva only useful for households?

No. A solo viewer may value catalog organization, preferences, and continuity, but those benefits should be tested against the person's actual workflow.

### Do I need more than one profile?

Not necessarily. One clear viewing context may be enough. Additional profiles add administration unless they separate genuinely distinct use cases.

### When should a solo viewer decline?

Decline when the source is unsuitable or unauthorized, an essential workflow fails, the privacy or terms are unacceptable, or measured benefits do not justify current cost and effort.
