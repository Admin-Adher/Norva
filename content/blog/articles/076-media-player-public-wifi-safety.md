# Using a Media Player on Public Wi-Fi: A Safety Checklist

> **In short:** Treat public Wi-Fi as a shared environment. Confirm the network name with the venue, keep your device and browser updated, use HTTPS, protect accounts with unique passwords and multi-factor authentication where available, and sign out when finished. These steps reduce risk; they do not make an untrusted network risk-free.

Hotel, airport, train, and café networks can be convenient for browsing a library or preparing a journey. A safer workflow starts before you connect and ends after you disconnect. It also recognises that a padlock icon confirms an encrypted connection to a site, not that the site itself is trustworthy.

## Before connecting

Complete these checks on a trusted connection when possible:

1. Install current operating-system, browser, and app updates.
2. Confirm that your screen lock is enabled.
3. Use a unique password for every important account.
4. Enable multi-factor authentication wherever the account provider offers it.
5. Download permitted items in advance if you do not need a connected session.
6. Record how to sign out of active sessions or change a password if needed.

The FTC recommends updates, strong passwords, and two-factor authentication as practical protections on public networks. CISA also recommends long, random, unique passwords and a password manager. For travel preparation that reduces dependence on shared networks, use the [responsible offline-viewing checklist](/blog/offline-viewing-travel-checklist/).

## Verify the network, not just the signal

A convincing network name is not proof that it belongs to the venue. Ask a staff member or consult an official sign for the exact name and any expected sign-in page. Avoid a network that only resembles the stated name.

Once connected:

- check that the address in the browser is the one you intended to visit;
- confirm that the page uses `https://` before entering credentials;
- do not bypass certificate or security warnings;
- avoid opening sign-in links received through unexpected messages;
- disconnect if the network repeatedly redirects you to unrelated pages.

HTTPS encrypts traffic between the browser and the website, but a scam website can also use HTTPS. Read the domain carefully before signing in.

## Limit what the session needs

Use the public network for the smallest reasonable task. Avoid changing sensitive account details unless necessary. Do not share passwords with travel companions, support agents, or venue staff.

If you connect a compatible media source that you own or are authorised to use, keep its credentials separate from your Norva account credentials. The [credential protection guide](/blog/protect-media-source-credentials/) provides a dedicated recovery and rotation plan.

On a shared or borrowed device, do not ask the browser to remember passwords. A private-browsing window can reduce local history, but it does not hide activity from the network, the website, or account providers. It is not a substitute for HTTPS or safe sign-in habits.

## Close the session deliberately

When finished:

1. stop playback and allow any intended progress update to complete;
2. sign out if the device is shared or not under your control;
3. close browser tabs and the app;
4. tell the device to forget the public network if automatic reconnection is unnecessary;
5. reconnect through a trusted network before reviewing account activity;
6. change a password promptly if you entered it on a suspicious page.

Signing out matters because closing a tab does not always end an authenticated session.

Copy this compact record into a note and mark each item only after observing it:

## Common mistakes and limitations

- Assuming every public network is malicious or, conversely, completely safe.
- Treating the padlock as proof that the organisation behind a site is legitimate.
- Reusing one password across the media player, email, and source account.
- Ignoring operating-system warnings to finish a session quickly.
- Leaving a shared device signed in.
- Expecting private-browsing mode to provide network anonymity.

No checklist removes every risk. If the task is sensitive and can wait, use a network you control.

## Frequently asked questions

### Is cellular data always safer than public Wi-Fi?

It avoids the venue's shared Wi-Fi, but no connection method eliminates account, device, phishing, or website risks. Keep the same update, password, MFA, and domain-verification habits.

### Should I download media before travelling?

That can reduce reliance on shared networks when the device, source, and associated rights permit offline access. Verify the download before departure.

### What should I do after a suspicious sign-in?

Move to a trusted connection, change the affected password, review available account sessions, enable MFA if offered, and contact the relevant provider through an official support route.
