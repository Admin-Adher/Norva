# Pair Your First TV With Norva More Safely

> **In short:** Pair only a television you control, while the screen is not visible to guests or a camera. Verify the Norva account on the secondary device before entering the temporary code, never save or share that code, and confirm the correct account and profile afterward. Finish by testing remote focus and reviewing the resulting device record.

A television is a shared, highly visible screen. Pairing should therefore be treated as a brief access ceremony with clear control of the display, code, account, and resulting session. Use the [first-screen selection guide](/blog/choose-first-norva-screen/) to confirm that TV is the appropriate onboarding surface.

## Control the room and display

Confirm that the television belongs to the household or is administered with explicit permission. Close unrelated apps and remove on-screen notifications. Position the screen so the temporary pairing information cannot be seen through a window, reflected, photographed, or captured by a meeting camera.

Do not pair on a public, hotel, rental, workplace, or shared venue television unless current official guidance explicitly supports the scenario and you can remove the session immediately afterward.

## Prepare a trusted secondary device

Use a phone, tablet, or computer you control to access the current official Norva pairing flow. Confirm the website or app identity and the intended account before entering anything. Complete the account security steps in the [Norva onboarding journey](/blog/norva-onboarding-complete-journey/) first.

Avoid search-result advertisements, shortened links, or codes requested through a message. Navigate through the current official interface or official Norva support guidance.

## Treat the pairing code as a secret

Enter the temporary code only in the intended official flow. Do not read it aloud, photograph it, paste it into chat, or include it in notes. A code that expires should be allowed to expire; do not repeatedly submit an old value.

If someone untrusted may have seen the code, cancel the process and start a fresh official pairing session. Do not rely on the code being short-lived as a substitute for secrecy.

## Confirm both sides before approval

Check that the television identifies the expected Norva session and that the secondary device shows the intended account and action. Read any permissions or confirmation text. Do not approve merely because a code matches.

If either side shows an unexpected account, device, location cue, or action, stop. Preserve only sanitized evidence after the live code is gone.

## Verify the resulting TV session

After pairing, confirm the correct household profile picker or intended profile appears. Navigate through a few non-sensitive areas with the remote. Focus should remain visible, directional movement should be understandable, and Back should behave consistently with the current interface.

Norva's supported TV experience is designed for remote navigation, but television models, operating systems, input devices, and app versions can differ. Record the actual environment rather than assuming identical behavior.

## Review the device record

Inspect the account's current device or security view where available. Confirm that the newly paired television can be recognized by time and broad device information. Avoid assigning a device name that reveals a precise address, room, person, or source.

The [TV-first household evaluation](/blog/norva-for-tv-first-households/) extends this check to viewing distance, focus paths, playback, and household handoffs.

## End failed attempts cleanly

If pairing fails, record the TV platform, Norva surface, local time, step, and sanitized message. Let any displayed code expire or cancel it through the interface. Close the session on both devices before retrying once.

Never send a live code, password, token, or unredacted account screen to support.

## Common pairing mistakes

- Pairing while the code is publicly visible.
- Entering a code through an unverified link.
- Approving without checking the signed-in account.
- Photographing the screen for later use.
- Ignoring the new device record after success.
- Retrying without closing or expiring the failed session.

## Frequently asked questions

### Can I reuse a pairing code?

Do not assume so. Follow the current official flow. If a code expires or the session fails, close it and generate a fresh code through the official interface.

### Should I pair a hotel television?

Avoid leaving an account session on a television you do not control. Check current official support guidance and prefer a device whose session you can verify and remove.

### What evidence is safe to share after a failure?

Share the platform, app surface, local time, step, and sanitized error through official support. Exclude live or expired codes, passwords, tokens, source details, and full account identifiers.
