# Set and Verify Your First Subtitle Preference

> **In short:** Confirm that a known item actually contains the intended subtitle or caption track, set one preference on the correct Norva profile, and watch several short dialogue intervals. Check selection, visibility, timing, and readability, then exit and reopen the item. Record missing or unsupported tracks separately because a preference cannot create captions absent from the media.

Subtitle verification is both a preference check and a viewing-quality check. A menu can show the intended label while the text is unreadable, mistimed, or unavailable for the selected version. Start with the [first playback verification routine](/blog/verify-first-playback-session/) so basic playback controls are already understood.

## Choose a known captioned item

Use the source's authorized workflow to identify one familiar item with the target subtitle or caption track. Record the exact source label privately and note the edition or version. Do not assume that every release of the same title contains the same text tracks.

Actual subtitles depend on the source, media, device, rights, format support, and current product behavior. Norva can preserve a preference where supported, but it cannot supply a missing track.

## Set the profile preference

Confirm the intended profile and choose the preferred subtitle language or behavior in the current interface. Change no other language, grouping, or profile setting during the test. Save or close the control using its visible workflow.

If captions are essential for access, document that requirement separately from a casual language preference. The [caption-dependent viewing evaluation](/blog/norva-for-caption-dependent-viewing/) provides a broader pre-subscription test framework.

## Inspect the active track

Start the known item and open the subtitle control. Record whether the expected track appears and whether it is active automatically, requires manual selection, or cannot be selected. Confirm that the tested version matches the source baseline.

Do not copy subtitle dialogue into the evidence record. A track label and a few observable timing checkpoints are sufficient and disclose less content.

## Watch three short intervals

Choose three brief dialogue intervals near the beginning of the test window. Observe whether text appears, remains long enough to read, and broadly corresponds to speech timing. Note overlap, clipping, missing lines, or distracting delay without transcribing the words.

Readability also depends on screen distance, display size, contrast, background, and any available styling. Record these conditions rather than treating one television or phone result as universal.

## Exit and repeat

Leave the item through the interface, reopen it under the same profile, and inspect the active track. Then test a second compatible item that is known to contain the same preferred language. Record whether behavior is consistent, manual, or item-specific.

For a useful control, test an item known not to contain the preferred track. Classify the result as a missing-track case, not a preference defect.

## Compare screens cautiously

After same-screen behavior is clear, repeat one known interval on another supported Norva surface. Keep the account, profile, source, item version, and track constant. Differences may arise from device or format capabilities rather than the saved preference alone.

The [Norva onboarding journey](/blog/norva-onboarding-complete-journey/) places this result before the broader synchronization and first-week reviews.

## Common subtitle-test mistakes

- Inferring captions from a language badge or description.
- Testing a different edition than the source baseline.
- Changing audio and subtitles at the same time.
- Judging readability without recording screen conditions.
- Treating a missing track as a saved-preference failure.
- Sharing dialogue or private titles in support evidence.

## Frequently asked questions

### Are subtitles and closed captions identical?

Not always. Labels and content can differ by source and media. Use the track's actual source label and test what appears rather than assuming equivalence.

### Can Norva generate subtitles for a title with none?

No such assumption should be made. The preference selects among available supported tracks; it does not create a missing text track.

### What if captions work on one screen but not another?

Record the device, surface, item version, track, and format conditions on both screens. Keep all other variables fixed, then consult current official support guidance.
