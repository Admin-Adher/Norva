# How to Revisit Movies Saved but Never Started

> **In short:** Review a small batch rather than the whole favorites list. For each movie, restate why it was saved, judge current interest, verify the available version, identify the real blocker, and assign one action: schedule, verify, defer until a named trigger, keep as a discovery reference, or remove from favorites. A favorite is a current signal, not a lifetime obligation.

An unwatched favorite can mean several things: genuine anticipation, a missing required track, a runtime mismatch, duplicate versions, or an impulse save whose reason has expired. Treating all of them alike makes the list less useful.

## Work in a bounded batch

Choose a manageable number from the oldest, newest, or one category. Do not scroll until fatigue turns every decision into “keep.” Record the selection rule so the next review covers a different area.

## Build the triage ledger

| Movie | Why saved | Interest now | Version ready | Blocker | Suitable session | Next action |
|---|---|---|---|---|---|---|
|  |  | High / medium / low | Yes / no / unknown |  |  |  |

If the original reason is unknown, do not invent one. Re-evaluate the title from current metadata and decide whether it still earns a place.

## Verify current state

Check:

- correct work identity;
- current source availability;
- selected cut or edition;
- runtime;
- required audio and subtitles;
- whether a partial start or completed viewing already exists;
- duplicate or grouped versions.

Dublin Core separates properties such as `available`, `modified`, `hasVersion`, and `relation`. The practical lesson is to keep current availability, work identity, and saved preference as different states.

## Name the blocker precisely

Replace “not in the mood” with a condition that can change:

- requires a longer uninterrupted session;
- better for a particular group;
- track information is unverified;
- preferred cut is unavailable;
- no longer fits current interests;
- saved only as a comparison reference.

If time is the blocker, use [the runtime-based shortlist](/blog/build-runtime-based-movie-shortlist/) instead of repeatedly reconsidering a movie in sessions it cannot fit.

## Assign one next action

### Schedule

Place the movie in one realistic session or [a manageable weekend queue](/blog/create-weekend-movie-queue/). Scheduling without a time slot is another form of deferral.

### Verify

Create a small metadata task: confirm the cut, runtime, track, or relationship. Do not bury the task inside the favorite label.

### Defer with a trigger

Write “revisit when a two-and-a-half-hour session is available” or “when the group is present.” A trigger is better than an undefined “later.”

### Keep as reference

If the movie supports a future director, decade, or remake project, record that purpose. Consider moving it from an immediate-view favorite to a project manifest if the interface and workflow allow.

### Release

Remove the favorite when current interest and project value are both low. This does not delete the movie from the library; it restores meaning to the favorite signal.

## Use a one-in, one-decision review

When adding a new favorite, optionally review one older entry. The goal is not a rigid list size but a continuous connection between saving and revisiting.

W3C notification guidance recommends clear outcomes and recovery. A useful personal record similarly states what changed and how to reverse it: “removed from favorites; still searchable in the library.”

## Turn the final five into a choice

When five entries are ready now, use [the five-to-one decision method](/blog/choose-one-movie-from-five/) instead of adding more candidates.

## Common mistakes and limitations

- Reviewing the entire list in one sitting.
- Keeping a title because removing it feels like deletion.
- Scheduling without a real session.
- Treating unknown version data as ready.
- Preserving an expired impulse as a permanent priority.
- Mixing project references with immediate viewing choices.

## Frequently asked questions

### How old must a favorite be before review?

Use a cadence that keeps the list meaningful. Age can select the batch, but current purpose determines the decision.

### Should unavailable favorites be removed?

Not automatically. Separate durable interest from temporary availability and add a trigger or verification task.

### What if I still want everything?

Rank current readiness and purpose. A title can remain discoverable without staying in the immediate favorite set.
