# How to Evaluate Norva for a Two-Profile Household

> **In short:** Create two clearly named, privacy-safe viewing contexts and test them on the shared screen. Each participant should establish distinct progress, one favorite, and one supported preference, then switch profiles and verify that the contexts remain understandable. Include wrong-profile recovery, device handoff, household administration, privacy, and the current plan limit before deciding.

Two profiles are valuable only if they prevent real collisions without making every session harder. Current editorial product guidance describes the Norva plan as supporting up to two profiles, but the official pricing and terms must be rechecked before publication and purchase.

Start with the [Norva evaluation framework](/blog/norva-evaluation-framework/) and mark profile separation as essential or desirable.

## Define the two contexts

Decide whether the profiles represent two people, personal and shared viewing, or another recurring distinction. Each should have a clear purpose. Create privacy-safe labels that remain understandable from the normal television viewing distance.

Use the [first household profile guide](/blog/create-first-household-profile/) to avoid full names, contact details, locations, ages, or sensitive traits.

## Prepare distinct test states

Participant A chooses one known item and creates a distinctive progress position, favorite, and language preference where supported. Participant B does the same with a different item. Both use media whose available tracks are already known from the authorized source.

Record neutral item codes and approximate positions. Do not place complete viewing histories in a shared evaluation document.

## Test profile selection

From the shared screen's ordinary starting state, ask each participant to select the correct profile and reach the intended item. Count directional actions or taps, selection errors, and time. Check that focus and labels remain visible.

A beautiful profile picker still fails if the labels are truncated or household members cannot tell which context they are entering.

## Test separation

Open A's profile and inspect A's progress and favorite. Confirm that B's test state does not appear as A's own context. Then repeat for B. Test the supported preference only with the exact media version containing the desired track.

Do not infer complete isolation from one hidden row. Record progress, favorites, and preferences as separate observations.

## Test one shared title

Both participants use the same work at different positions or with different preferences. This is the strongest collision test because identity is shared while context should remain distinct. Confirm the exact version on both profiles.

The [manual progress comparison](/blog/norva-vs-manual-viewing-progress-tracking/) can show whether two manual logs would be simpler or more error-prone for this household.

## Test a wrong-profile mistake

Intentionally begin one short session under the wrong profile, then stop. Observe how easily the mistake is recognized and corrected through supported controls. Do not delete broad history or recreate profiles merely to clean the test.

Record whether the interface offers enough profile context before playback. A system is more resilient when common mistakes are visible and recoverable.

## Test screen handoff

If the household uses another supported screen, move one participant's known item from the shared screen to that screen. Verify account, profile, version, displayed position, and actual resume point. Repeat for the other participant only if the workflow is important.

Do not mix accounts or use a guest session in the comparison.

## Review administration and privacy

Identify who manages the account, sources, device records, billing, and support cases. Decide who may rename or remove profiles. Review what profile labels, favorites, progress, and recommendations are visible on shared screens.

Read current privacy, pricing, and terms pages. Profile separation should not be treated as a substitute for account security or access control.

## Frequently asked questions

### Must the two profiles represent two people?

No. They can represent two recurring viewing contexts, provided the purpose is clear and the current plan supports the setup.

### Do profiles guarantee privacy from each other?

Do not assume so. Review actual shared-screen visibility and official privacy and account controls. Profiles organize viewing contexts; they are not necessarily separate security accounts.

### What if two profiles are not enough?

Evaluate the current Norva Family profile limit and terms against the household's real contexts. Do not combine incompatible viewers merely to fit a lower limit.
