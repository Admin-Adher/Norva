# Offline Playback vs. Connected Playback: Key Differences

> **In short:** Offline playback is best when connection reliability is the main constraint and you can prepare eligible items in advance. Connected playback is best when a usable network is available and you want current source access without reserving a complete local copy. Neither mode is universally better; choose for the session.

The important distinction is where the playable media is available at the moment you press play. Offline mode depends on a prepared local copy. Connected mode depends on reaching the compatible source through a network.

## Side-by-side comparison

| Criterion | Offline playback | Connected playback |
| --- | --- | --- |
| Preparation | Item must be eligible, completed, and tested | Source and connection must be reachable |
| Network during viewing | Not required for the prepared media copy | Required throughout the session |
| Device storage | Uses local storage | Usually avoids a complete offline copy |
| Library freshness | Limited to what was prepared | Can reflect the currently connected source |
| Progress across devices | May wait until reconnection | Can update while connected, subject to current conditions |
| Travel reliability | Reduces dependence on route connectivity | Depends on network availability and quality |
| Availability | Conditional on device, source, media, and rights | Conditional on source, network, account, and media |

This table describes workflows, not a guarantee for a specific item or device.

## Choose offline for predictable disconnection

Offline preparation is useful for flights, underground routes, remote areas, or mobile-data limits. Its advantage comes from doing the network-dependent work earlier.

That preparation has costs:

- you must choose items in advance;
- each eligible item consumes device storage;
- transfers need time and a stable connection;
- the exact item and required language tracks must be tested;
- progress may not reconcile until connectivity returns.

Use the [offline travel checklist](/blog/offline-viewing-travel-checklist/) before depending on the local copy. Norva makes offline access available only when the device, source, and associated rights permit it.

## Choose connected playback for flexibility

Connected playback lets you browse the current catalogue from your source and make a choice at viewing time. It suits home or another place with a connection that is sufficiently stable for the selected media.

Its limitations are different:

- an interruption can affect playback;
- mobile data can be consumed;
- public networks require careful sign-in habits;
- source reachability and account state still matter;
- switching networks may not be seamless.

If you move between Wi-Fi and mobile data, follow the [network-switching progress workflow](/blog/switch-wifi-mobile-data-video/) instead of assuming the handoff will be invisible.

## Understand progress and freshness

Connected devices can exchange account state while online, but do not promise instantaneous sync. Offline playback can create a local progress state that needs to reconcile later.

When you reconnect:

1. use the same account and profile;
2. open the same item and version;
3. allow the device time to connect;
4. inspect the displayed progress;
5. avoid advancing a second device until the result is clear.

If two devices were used offline, the “correct” position may depend on your intent, not simply the latest timestamp.

## Compare storage with data use

Offline mode uses network data during preparation and local storage afterward. Connected mode uses network data during playback and generally requires less persistent media storage.

Quality, encoding, duration, and source packaging affect both file size and transferred data. Avoid universal numbers. Measure actual device storage with the [offline storage budget](/blog/storage-for-offline-video/) and review device data counters for connected sessions.

## Common mistakes and limitations

- Calling connected playback “storage free”; apps still use caches and local data.
- Assuming offline items remain available indefinitely.
- Treating a completed indicator as a disconnected playback test.
- Expecting progress to update while the device has no connection.
- Assuming every network switch preserves playback without interruption.
- Ignoring the source and associated rights in offline eligibility.

Actual behaviour varies by device, app version, source, media, and network.

## Frequently asked questions

### Can I mix offline and connected playback on one trip?

Yes. Prepare priority items for unreliable segments and use connected browsing when a suitable network is available. Verify progress when changing modes.

### Which mode uses less data?

That depends on what you prepare or play, quality choices, and whether items are replayed. Review measured transfers and device data counters rather than assuming a universal winner.

### Does offline mode improve video quality?

Not automatically. The quality available depends on the prepared item and source. Offline mode mainly changes when and where the media data is obtained.
