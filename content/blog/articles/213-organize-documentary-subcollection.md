# A Practical Workflow for a Documentary Subcollection

> **In short:** Organize documentaries with separate fields for form, subject, temporal coverage, spatial coverage, creator, relationships, and available version. Keep personal ratings, viewing progress, and temporary research themes outside that descriptive core. Add only an axis that answers a recurring retrieval question, normalize its values, and test it with known examples before applying it broadly.

Documentaries often cross topics, places, periods, and forms. A single “Documentary” category is too broad for discovery, while a permanent category for every subject creates a brittle maze.

## Start with retrieval questions

Write five questions the subcollection should answer, such as:

- Which documentaries cover this historical period?
- Which works focus on this place?
- Which are essay films, observational works, interviews, or compilations?
- Which creator's works are present?
- Which current versions have required tracks?

Do not add a field that no question uses.

## Build the seven-axis card

| Axis | What it describes | Example rule | Repeatable values? |
|---|---|---|---|
| Form | How the work is constructed | One primary, optional secondary |  |
| Subject | Main topic | Controlled preferred term |  |
| Temporal coverage | Period depicted | Named period or range |  |
| Spatial coverage | Place depicted | Consistent place hierarchy |  |
| Creator | Authorship/contribution | Verified role |  |
| Relationship | Series, part, source work | Typed link |  |
| Version | Cut, language, source item | Separate current record |  |

Dublin Core defines subject, temporal and spatial coverage, creator, contributor, relation, and version properties separately. That separation prevents “made in 2020” from being confused with “about the 2020s.”

## Define a small controlled vocabulary

Choose one preferred term for recurring concepts and keep alternate terms as aliases. For geographic values, decide whether the collection uses country, region, city, or nested levels. For periods, state whether boundaries are inclusive.

Do not force a complex professional taxonomy into a small personal library. Consistency and retrieval value matter more than the number of labels.

## Separate work, series, and version

A documentary movie can belong to a series or project while remaining a distinct work. An alternate cut or language version belongs beneath that work rather than becoming a new subject entry.

The FIAF manual's work–variant–manifestation–item model offers a useful boundary for moving-image collections. Apply only the level of detail that supports real choices.

## Handle short and feature forms intentionally

If runtime or presentation causes different browsing needs, use [the short-versus-feature separation guide](/blog/separate-short-and-feature-films/) to decide whether a distinct view helps. Do not assume one universal runtime threshold.

## Keep research themes temporary

A project such as “coastal communities” may combine place, subject, and a current question. Save it as a dated project manifest rather than a new permanent category. [The temporary-theme workflow](/blog/track-viewing-themes-without-clutter/) explains how to close and archive it.

For period-based curation, [the decade retrospective grid](/blog/build-decade-movie-retrospective/) adds coverage and selection reasoning without altering core metadata.

## Test the taxonomy

Select three controls:

- an obvious match;
- a borderline multi-topic work;
- a record with missing metadata.

Search and filter across each chosen axis. Record whether the right items appear, whether unknowns remain visible, and whether one work needs too many labels to be found.

## Maintain an exception queue

Put uncertain subjects, unverified creator roles, ambiguous dates, and duplicate relationships in a review queue. Unknown should remain a valid state. Do not invent precision simply to complete every field.

Norva may search, filter, and group compatible authorized sources in one interface, but documentary coverage and metadata depend on what the user connects. Verify specialist descriptions against appropriate authoritative sources.

## Common mistakes and limitations

- Using one field for release date and depicted period.
- Turning every temporary research question into a category.
- Mixing work and version metadata.
- Allowing uncontrolled synonyms to fragment retrieval.
- Assigning many subjects without a primary rule.
- Claiming completeness from a connected personal collection.

## Frequently asked questions

### How many subjects should one documentary have?

Use the fewest terms that answer recurring retrieval questions. Establish a primary-subject rule and add secondary terms only when useful.

### Should documentaries and documentary series be mixed?

They can share a discovery view, but preserve series, season, episode, and standalone-work relationships.

### What if the subject is disputed?

Record uncertainty or multiple supported terms with provenance rather than forcing one unsupported label.
