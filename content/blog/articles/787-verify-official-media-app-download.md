# Verify an Official Media App Download Before Installing

> **In short:** Start at Norva's known official support destination and confirm that the exact phone, tablet, television, or browser platform is supported. Open the documented platform store or download route independently, then compare the application name, publisher, listing address, permissions, version information, and update path. Do not install files supplied through messages, pop-ups, or remote support. If any detail conflicts, stop and ask official Norva support before entering credentials.

Search results, copied logos, reviews, and familiar screenshots can appear on a lookalike listing. Provenance means tracing the application from Norva's official guidance to the platform's recognized distribution channel and then to the installed record.

## Confirm the exact supported platform

Record device manufacturer, model, operating-system version, and store region. Check current Norva support rather than assuming that an application with a similar name supports the device.

If the product is unavailable through the documented route, do not bypass the limitation with a package from a forum, mirror, file-sharing site, or person offering private access. Compatibility and availability can change.

## Navigate independently

Type the known Norva address, use a trusted bookmark, or open the official store directly. Avoid links in unsolicited email, text, social messages, pop-ups, comments, videos, or support calls.

The [phishing-page recognition guide](/blog/recognize-media-account-phishing-page/) applies when a page asks for sign-in before allowing a download. A genuine-looking certificate or lock symbol does not prove the publisher.

## Compare the store listing

Check the displayed application name, publisher or developer, platform, category, listing address, privacy information, permissions, version notes, and supported devices where the store exposes them. Compare these details with Norva's official route.

Do not use review count, star rating, ranking, or advertising placement as the decisive identity check. Those signals do not establish publisher provenance.

## Evaluate requested permissions

Read the platform's data-safety or privacy information where available, then review actual permission prompts at installation and first use. A permission should match a feature the user understands, such as nearby-device access for an explicitly initiated connection.

Decline unexplained access and consult official support. Do not enable restricted system settings merely because an installer instructs you to bypass a warning.

## Keep the platform protection layer active

Use the device's documented application verification or harmful-app protection features and keep its system and store components current. Do not disable protection to force an installation.

The [system-and-app update guide](/blog/operating-system-vs-app-security-updates/) explains why an official application still depends on a supported platform.

## Verify after installation

Open the installed application from the platform launcher, not the original message. Recheck the application information page, permissions, update source, and publisher details. Confirm that sign-in or pairing uses the expected Norva flow.

Before television sign-in, use the [smart-TV security preflight](/blog/secure-smart-tv-before-sign-in/) to verify reset provenance, network, software, privacy controls, and device label.

## Respond to a suspected unofficial install

Do not enter credentials. Disconnect the application from further account activity and follow the device maker's current instructions for reviewing or removing suspicious software. If a password, code, payment, or remote-control permission was provided, use a known-good device and begin account incident response.

Preserve the listing address, timestamp, publisher display, and non-sensitive screenshots for official reporting. Do not redistribute the file as evidence.

## Common mistakes and limitations

- Clicking the first sponsored search result.
- Trusting a logo or star rating as proof.
- Installing a file sent by “support.”
- Disabling device protection to bypass a warning.
- Assuming the same store route exists on every platform.
- Entering credentials before checking publisher details.
- Keeping a suspicious package for others to test.

## Frequently asked questions

### Is a store listing automatically official?

No. Start from Norva's official platform guidance and compare the listing, publisher, supported device, and update route.

### What if Norva is not listed for my device?

Do not install an unofficial substitute. Ask Norva support about current supported options and compatibility.

### Should I allow every requested permission?

No. Allow access only when it matches an understood feature and current official guidance; review it again after setup.
