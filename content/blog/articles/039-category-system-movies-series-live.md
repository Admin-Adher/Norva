# A Simple Category System for Movies, Series, and Live Programs

> **In short:** Separate movies, series, and live programmes at the top level because each has a different navigation model. Use a small shared category vocabulary underneath, preserve show-season-episode hierarchy for series, and treat schedule information as time-sensitive rather than as a permanent category.

A mixed library becomes confusing when every format is forced into the same grid. A movie is usually a single selection, a series is a hierarchy, and a live programme is tied to time and schedule data when the source supplies it.

## Layer 1: format

Start with three clear format destinations:

- **Movies:** individual works browsed by title, category, year, language, or another supplied attribute.
- **Series:** shows that lead to seasons and episodes.
- **Live:** currently scheduled or upcoming programmes where the compatible source supplies relevant guide information.

This first split reduces ambiguity. A category named “Drama” can then exist within Movies and Series without mixing films and episodes in one row.

For the general principles behind labels, see [how media-library categories work](/blog/media-library-categories/).

## Layer 2: a controlled category vocabulary

Use a short set of stable, understandable categories. Add a category only when it supports a recurring choice and contains a meaningful subset.

Good category tests:

- Can a viewer predict what belongs there?
- Is it distinct from another category?
- Will the label still make sense in six months?
- Does it work within the selected format?
- Can the source metadata support it consistently?

Avoid building separate synonym categories such as “Science Fiction,” “Sci-Fi,” and “SF” unless the connected source deliberately distinguishes them.

## Layer 3: temporary refinements

Use filters for questions that change from session to session:

- year or period;
- audio language;
- subtitle language;
- availability;
- rating or score, where supplied and understood.

This keeps the permanent taxonomy small. A filter can be cleared; a large category tree is harder to maintain.

The [large movie collection guide](/blog/organize-large-movie-collection/) shows how this layered model reduces scrolling.

## Give series their own hierarchy

Do not create global categories for “Season 1” or “Episode 2.” Those labels make sense only beneath a specific show.

Use:

1. series;
2. season;
3. episode;
4. playback state as a personal overlay.

Specials and unusual orders should follow reliable source metadata and remain explicitly labelled. The [season and episode navigation audit](/blog/organize-seasons-episodes/) provides a representative-sample check.

## Treat live information as time-sensitive

Where the connected source supplies guide data, live browsing can use:

- channel or source-defined group;
- now and next;
- date and time;
- programme title;
- current availability.

Schedule information can be incomplete or change. It should not be copied into permanent categories as if it were static metadata. See [how TV guide data works](/blog/tv-guide-data-explained/) before troubleshooting a missing programme.

Do not claim that every source exposes guide information or that every listing is complete. The source remains responsible for the data it provides.

## Use the mixed-library blueprint

Map each control to one layer:

| Information | Best layer | Example use |
| --- | --- | --- |
| Movie, series, live | Format | Choose the navigation model |
| Stable subject or genre | Category | Create a recurring browse path |
| Year, language, availability | Filter | Ask a temporary question |
| Season and episode | Hierarchy | Preserve sequence and context |
| Current time and schedule | Live guide | Support time-sensitive choice |
| Favorite or progress | Personal state | Return to individual intent |

Audit ten items from each format. If the same label changes meaning between formats, rename it or keep it inside the relevant format.

## Keep the home view task-oriented

A useful mixed-library home does not need to display every category. It can prioritise:

- Continue Watching for active progress;
- Favorites for deliberate saves;
- recent activity;
- one entry point for each format;
- a search route for known-title retrieval.

The complete category structure can remain one level deeper. This preserves choice without turning the opening screen into an index.

## Common mistakes and limitations

- Mixing films and individual episodes in one category without context.
- Creating permanent categories for temporary year or language questions.
- Treating schedule data as fixed.
- Assuming every source supplies the same categories and programme information.
- Duplicating synonyms across the taxonomy.
- Using Favorites as a substitute for format structure.

## Frequently asked questions

### Can movies and series share category names?

Yes. Keep the format context visible so that the same label does not mix individual films with shows or episodes unexpectedly.

### Should live programmes be added to permanent categories?

Only when the category remains meaningful beyond the schedule. Time-specific programme information is better handled through current guide data supplied by the source.

### How many category levels are useful?

Keep the permanent structure shallow. Format plus one stable category layer is often easier to understand than several nested folders, while series retain their necessary season and episode hierarchy.
