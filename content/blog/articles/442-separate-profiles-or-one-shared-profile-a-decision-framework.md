# Separate Profiles or One Shared Profile: A Decision Framework

> **In short:** Use separate profiles when viewers regularly need independent progress, history, favourites, or language preferences. Use one shared profile when people mainly watch together and genuinely want a combined state. Check current plan capacity, shared-screen privacy, and the effort of choosing the right profile. Neither model is universally better.

The decision is about persistent viewing context, not the number of devices in the house. A profile should correspond to a clear pattern of use.

## Quick comparison

| Criterion | Separate profiles | One shared profile |
| --- | --- | --- |
| Independent progress | Clearer | Mixed by design |
| Personal favourites | Easier to interpret | Requires shared ownership |
| Language preferences | Can differ by context | One agreed default context |
| Profile selection | More choices at entry | Simpler entry |
| Mistake risk | Wrong-profile playback | One person's actions affect all |
| Privacy | Better context separation | History is intentionally shared |
| Capacity | Uses more available profiles | Uses fewer |

A profile is not a password. Someone who can access the signed-in account may still see or switch contexts.

## Choose separate profiles when state diverges

Separation is useful when viewers:

- watch the same series independently;
- maintain different favourites;
- use different audio or subtitle preferences;
- want distinct history and recommendations from the connected source;
- resume from different positions;
- use shared screens at different times.

Norva can preserve these states under the same account across supported devices. Available tracks still depend on the source and media.

## Choose a shared profile when viewing is genuinely joint

One profile may fit when the household always watches together, shares favourites, and agrees on progress and preferences. It can also reduce selection friction on a single shared screen.

Do not call a profile “shared” merely because capacity is limited. If independent contexts are being mixed unwillingly, revisit the structure or current plan options.

## Evaluate privacy separately

Separate profiles reduce accidental mixing but do not secure account data. Review device locks, sign-in persistence, account settings, and who can use the shared screen.

Norva's privacy policy describes current data practices. It does not turn a profile into access control.

## Check current capacity

Use the pricing page for current profile allowances and plan conditions. Do not convert profile count into a claim about devices or simultaneous playback.

The [profile-count planning guide](/blog/how-many-household-profiles-do-you-actually-need/) helps prioritise persistent contexts when capacity is constrained.

## Include the error cost

Ask what happens when someone selects the wrong context. For independent series viewing, one mistake can affect progress. For a shared family film night, a shared profile may be the intended state.

Use [the wrong-profile prevention workflow](/blog/how-to-prevent-progress-from-landing-on-the-wrong-profile/) when selection errors are frequent.

## Scenario guidance

### Couple watching mostly together

Start with a shared context if both people accept combined progress and preferences. Add separate profiles only for recurring independent viewing.

### Household with overlapping series

Separate contexts are usually clearer because the same episode list can carry different positions.

### Shared TV with occasional guest

Use an available approved context or a planning-only routine. Do not invent a guest feature or share credentials casually.

The [complete household profile guide](/blog/the-complete-guide-to-household-media-profiles/) covers naming, guest boundaries, and review.

## Common mistakes and limitations

Avoid one profile per device, treating a shared profile as a compromise nobody accepted, creating duplicate profiles with no clear owner, and assuming separation provides security.

Current plan capacity and features can change. This framework organises a household decision; it does not guarantee privacy, parental controls, or concurrent playback rights.

## Frequently asked questions

### Can two people share favourites but keep separate progress?

Do not assume partial state sharing exists. Use only the profile behaviour visibly supported, or keep a separate planning list outside the product.

### Is one profile simpler?

It reduces selection, but only when combined history and progress are actually desired.

### Can we change the structure later?

Review the current controls and consequences before changing or deleting profiles. Preserve important state and contact support when migration is unclear.
