# How to Explain Profile Boundaries to Household Members

> **In short:** Explain five separate layers: device access, Norva account, profile, connected media source, and current playback state. A profile owns progress, history, favourites, and preferences for its context, but it is not a password, device limit, parental control, or media entitlement. Finish with a teach-back: each person should explain which profile to choose and how to leave the screen.

Clear boundaries prevent rules from sounding arbitrary. “Use your profile” is easier to follow when everyone knows what the profile changes and what it does not.

Choose examples from ordinary household routines without naming private media. One clear example of wrong progress and one of a shared favourite usually explains the boundary better than a long list of abstract warnings.

## Start with the five-layer model

Use plain language:

1. **Device:** the TV, tablet, mobile device, or compatible browser.
2. **Account:** the signed-in Norva service context.
3. **Profile:** the viewing context for progress, history, favourites, and preferences.
4. **Source:** compatible media access the user owns or is authorised to use.
5. **Session:** the item, version, position, audio, and subtitles currently active.

A problem at one layer should not be “fixed” by changing another.

## Explain what profiles do

Give concrete examples:

- independent series positions stay in separate contexts;
- favourites represent the profile owner's intent;
- language preferences can follow the profile when tracks exist;
- a profile can be used across supported devices.

Norva does not include a media catalogue; source access remains separate.

## Explain what profiles do not do

Profiles do not automatically:

- protect account settings with a password;
- define device or simultaneous-session limits;
- provide parental controls;
- grant media rights;
- hide history from someone who can access the signed-in account;
- create missing audio or subtitle tracks.

Use current pricing and terms for plan conditions, not household assumptions.

## Teach the pre-play rule

Say: “Read the profile label before play.” Then demonstrate account, profile label, item, version, and proposed position.

The [wrong-profile prevention workflow](/blog/how-to-prevent-progress-from-landing-on-the-wrong-profile/) turns this rule into a visible gate.

## Teach the shared-screen exit

Agree on a neutral profile-selection, library, or sign-out state supported by the current interface. Explain when sign-out is required, especially on borrowed or untrusted devices.

Do not teach a hidden shortcut or undocumented remote sequence.

## Use teach-back

Ask each member to answer:

- Which profile owns my independent progress?
- What do I check before play?
- What happens when we watch together?
- What should I do for a guest?
- How do I leave the screen for the next person?
- Who handles account or source changes?

W3C labels-and-instructions guidance supports clear instructions, though this household method is not a formal accessibility assessment.

## Common mistakes and limitations

Avoid jargon, presenting profiles as surveillance, using profile count as a device rule, and teaching controls that have not been verified.

Household understanding does not create technical restrictions. Anyone with access to the signed-in account may still reach profile information.

## Retest understanding after a real handoff

During the next shared-screen handoff, ask the incoming viewer to identify the account, profile, source, media version, and device layer before selecting playback. Correct only the misunderstood layer and repeat the teach-back; avoid adding new rules when the original one was simply unclear.

## Frequently asked questions

### Should children receive a different explanation?

Use age-appropriate language, but do not present Norva profiles as parental controls. Follow applicable household and legal responsibilities.

### Is a profile private?

It separates viewing context, not account access. Device and account controls determine who can open it.

### What if people forget the rule?

Improve labels and leave the screen in a neutral state. Re-teach the pre-play check without monitoring private activity.
