# When a Couple Should Use Separate Viewing Profiles

> **In short:** A couple should use separate profiles when either partner regularly watches the same media independently, wants different favourites or history, or needs distinct language and subtitle preferences. One shared profile is simpler when viewing is genuinely joint and combined state is desired. Many couples need separate personal contexts plus an agreed routine for shared viewing, subject to current plan capacity.

The right structure depends on behaviour, not relationship status. Two partners can watch almost everything together, or use the same TV for completely independent libraries.

Begin with the current pattern rather than an idealised routine. A structure that fits actual weekday, travel, and joint sessions is easier to maintain than one designed around rare exceptions.

## Map joint and independent occasions

List recurring sessions:

- planned viewing together;
- one partner continuing alone;
- overlapping series watched at different speeds;
- different genres saved for later;
- distinct audio or subtitle needs;
- travel or mobile viewing by one partner.

A profile is justified by persistent state separation, not by owning a separate device.

## Signals for separate profiles

Choose separation when:

- progress conflicts repeatedly;
- one person's favourites obscure the other's choices;
- history should remain personally meaningful;
- language preferences differ;
- recommendations from the connected source are more useful when contexts stay distinct;
- each partner wants independent handoff across supported devices.

Norva can preserve progress, history, favourites, and preferences under the same account. Available tracks still depend on the source and media.

## Signals for one shared profile

A shared profile can fit when the couple watches nearly everything together, agrees on favourites and preferences, and wants a single progress state. It also reduces profile-selection steps on a shared TV.

Do not choose one profile solely because selecting profiles feels inconvenient. Clear names and a pre-play check may solve that friction without mixing desired state.

## Handle mixed viewing explicitly

A couple may use personal profiles for independent sessions and choose an agreed context for joint viewing. Do not invent a special couple profile feature; use only profiles available under the current plan.

## Include privacy expectations

Separate profiles organise context but do not hide the account from someone who can access the signed-in device. Discuss whether viewing history is intended to be private, shared, or simply not reviewed without consent.

## Common mistakes and limitations

Avoid assuming couples always share taste, one profile per device, creating duplicate profiles without ownership, and treating separate profiles as secrecy controls.

Current features and plan terms can change. This decision framework cannot guarantee privacy, parental controls, or concurrent playback.

## Review after routines change

Revisit the decision after a schedule change, shared-device change, accessibility need, or sustained shift from joint to independent viewing. Keep the existing arrangement when it still reflects reality; profile separation is a tool, not a relationship judgment.

## Frequently asked questions

### Do couples need a third shared profile?

Only if recurring joint viewing needs a distinct state and current capacity supports it. Do not create one for rare occasions automatically.

### Can partners use separate profiles on the same TV?

Profiles are designed for separate contexts when available. Confirm the active label before playback on the shared screen.

### What if one partner watches ahead?

Use separate personal contexts for that series or agree which progress state the shared profile owns.
