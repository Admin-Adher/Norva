# How to Estimate the Effort of Onboarding With Norva

> **In short:** Estimate work rather than promising a fixed setup time. Count source authorization and readiness, secure account creation, first-screen setup, profile decisions, one-source connection, catalog sampling, accessibility checks, and a second-screen comparison only if needed. Assign extra effort to unresolved dependencies and shared administration. Start with a minimum verified setup, record actual work, then revise the estimate before expanding.

Onboarding effort varies because households start with different sources, screens, profiles, access needs, and levels of preparation. A universal number of minutes would hide the dependencies that create most rework. A work-unit estimate is more honest and more useful.

## Define the minimum verified setup

Describe the smallest state that would let you evaluate Norva: one secure account, one supported screen, one purposeful profile, one compatible authorized source, and a three-item catalog sample. Add another screen only when continuity is essential.

The [complete onboarding journey](/blog/norva-onboarding-complete-journey/) provides the full sequence; this article estimates its work.

## Score source readiness first

A prepared source with clear authorization, a known administrator, stable reachability, and known metadata requires less investigation than an uncertain one. Use the [source-readiness gate](/blog/evaluate-source-readiness-for-norva/) before assigning setup effort.

If a source dependency is unresolved, count it separately. Do not hide source remediation inside the Norva estimate.

## Count decisions and actions differently

Use a simple model:

- One unit for a known, reversible action with clear input.
- Two units for a choice, comparison, or household handoff.
- Three units for an unresolved dependency requiring outside confirmation.

The units are relative complexity, not minutes. Replace them with your observed durations only after a real task is completed.

## Estimate account and security work

Include official-domain verification, unique password creation and storage, recovery review, shared-device decisions, privacy and terms reading, and subscription-provider awareness. Do not record passwords or recovery codes in the estimate.

Assign an owner for account administration so repeated conversations are not mistaken for product setup time.

## Estimate the first screen

Record support confirmation, installation only where applicable, sign-in, input method, and one navigation route. The Web version works in a compatible browser without installation; mobile and TV contexts have different preparation and input needs.

Use the [first-screen decision guide](/blog/choose-first-norva-screen/) when the choice itself is unresolved.

## Estimate profiles by recurring context

Count the discussion required to define each recurring viewing context, choose a privacy-safe name, assign an administrator, and verify preferences. Norva includes up to two profiles and Norva Family up to five, but profile capacity is not a reason to create every possible profile on day one.

The [first-profile guide](/blog/create-first-household-profile/) keeps this work purposeful.

## Estimate source connection and sampling

Include secure credential retrieval, current-interface entry, connection observation, and a small identity-based sample. Add units for known source metadata problems, but do not promise Norva will correct them. The first source should be verified before a second is considered.

Use the [sample verification guide](/blog/verify-sample-after-source-connection/) for observable completion criteria.

## Add accessibility and language checks

If audio, subtitles, captions, remote focus, or readability is essential, count a representative media sample and each required screen. Preferences cannot create tracks the source does not provide. An essential access test deserves an owner and pass threshold, not a note to “check later.”

## Add verification and rework reserve

Budget work for a clean exit and return, one repeat run, evidence cleanup, and one support-ready record for unresolved issues. Add a rework reserve for uncertain compatibility, source instability, shared-account decisions, or multiple essential screens.

Do not expand to every device until the minimum setup passes.

## Common mistakes and limitations

- Promising a universal setup duration.
- Counting unresolved source work as a simple action.
- Configuring every profile and device immediately.
- Omitting security, privacy, and accessibility review.
- Measuring clicks while ignoring decisions and handoffs.
- Failing to revise the estimate after observed work.

## Frequently asked questions

### Is browser onboarding always faster?

Not necessarily. A compatible browser removes installation, but source readiness, account decisions, profiles, and verification still create work.

### Should I estimate every household device?

No. Estimate the first supported screen and only the secondary screens that represent distinct essential workflows.

### Can I convert work units into minutes?

Only from your own observed tasks. Universal conversion would imply a speed claim that source and household differences cannot support.
