# Program Description Looks Wrong? Verify the Listing Context

> **In short:** Confirm the exact channel, listing, date, and time before comparing descriptions. Preserve short distinguishing excerpts from the authorized source and each relevant Norva view, plus program or episode cues, language, profile, filters, guide window, device, application version, and timestamp. Add one control listing and another supported device when practical. Avoid copying full descriptions, editing source text, or assuming stale text and wrong mapping are the same issue.

A description can look wrong because it belongs to another episode, another time slot, a same-name program, another channel, a different language, or an older source value. Exact listing context comes before text correction.

Keep the comparison tied to one exact scheduled airing.

## Confirm channel identity

Record source label, channel name, number or identifier where visible, logo state, and region or version cue. A description from a same-name channel is not a valid text comparison.

The [TV guide handbook](/blog/tv-guide-troubleshooting-handbook/) provides the channel matrix.

## Confirm listing identity

Record program title cue, episode or event cue, start and end times, date, previous and next listings, and guide row. Use detail view where available. Do not compare source text captured for another airing.

## Preserve concise excerpts

Record only enough words to distinguish source and Norva values, along with approximate length and language. Do not reproduce complete protected descriptions in public notes or screenshots.

## Verify source text

Through the provider's official authorized route, capture the exact description field, language, confirmation time, and related episode fields. If multiple language fields exist, keep them separate. Do not assume one is universally selected.

## Record Norva text by view

Compare grid cell, now-and-next, detail page, search result, and episode view only where a description appears. Wrong text in one compact view is different from every view showing the same value.

## Check episode fields

Record series, season, episode number, and episode title where visible. If those fields are absent, use the [episode-data checklist](/blog/episode-data-missing-in-guide/). If they point to another episode, classify identity before description.

## Check channel-program mapping

If the description matches another visible channel's listing at the same time, use the [channel-program crosswalk](/blog/wrong-program-mapped-channel/) and preserve both rows.

## Freeze language and guide context

Record interface language, account, profile, source, channel group, filters, search, selected date, guide window, device clock and zone, device, and application version.

## Compare a control listing

Choose a nearby listing from the same source and view whose source and Norva excerpts align. A normal control limits scope but does not reveal field-selection or refresh logic.

## Preserve the timeline

Record source edit and confirmation, guide open or refresh, first wrong-text observation, view changes, application update, and current state. If old text persists after a source change, use the [now-and-next timeline](/blog/now-next-information-stale/).

## Compare another supported device

Use the same account, profile, source, channel, listing, filters, language, guide window, and close timestamp. Record both application versions. Different text on one device is cross-screen evidence.

## Avoid text edits first

Do not overwrite the source description, rename the program, change episode metadata, remap channels, repeat refreshes, or clear application data before saving the comparison.

## Classify the result

Use different channel, different listing or airing, different episode, source text differs, language difference, view-specific text, device-specific text, old source value, changed after documented action, or unknown. Avoid claims about priority, refresh, or storage.

## Prepare support evidence

Use the [guide issue record](/blog/guide-issue-support-evidence/) with listing identity, short excerpts, language, source confirmation, views, time context, control, devices, versions, and actions.

## Common mistakes and limitations

- Comparing another channel, airing, or episode.
- Relying on memory instead of short excerpts.
- Copying complete protected descriptions.
- Ignoring language and guide window.
- Editing source text before evidence capture.
- Treating text mismatch as proven mapping behavior.

## Frequently asked questions

### How much description text should I share?

Use only short distinguishing excerpts, field labels, language, and timestamps—not the complete description.

### What if the description matches another channel?

Preserve both channel identities and schedules and follow the mapping diagnostic.

### Should I correct the source description?

Not before confirming listing identity, metadata ownership, and provider and Norva support guidance.
