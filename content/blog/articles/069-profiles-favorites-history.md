# Profiles, Favorites, and History: Keeping Shared Libraries Personal

> **In short:** Treat the catalog as shared source information and the interaction around it as viewer context. Profiles identify the viewer; favourites record intentional saves; history records prior activity; progress records a stopping point; and preferences guide future choices when options exist. Consistent profile selection keeps those signals easier to understand.

Several people can browse the same catalog without wanting the same personal view. The useful distinction is not “mine versus yours” at the media-rights level. It is shared catalog facts versus the interaction signals created by each viewer.

## Map shared facts and personal signals

This ownership map is the original evidence element for the article.

| Category | What it describes | Sensible context |
| --- | --- | --- |
| Catalog title and artwork | Source-supplied item | Shared |
| Grouped versions | Available source entries | Shared |
| Audio and text inventory | Options on an exact version | Shared item fact |
| Active profile | Who is using the interface | Viewer context |
| Favourite | An intentional save | Personal or deliberately shared |
| History | Prior interaction | Viewer context |
| Progress | Where a session stopped | Viewer context |
| Language preference | Preferred option when available | Viewer context |

Norva publicly confirms organisation, grouped versions, profiles, recommendations from the connected source, and synchronisation of progress, history, favourites, and preferences. Before publication, a product reviewer must confirm how each category is scoped to profiles in the current release.

## Profiles answer “who?”

Choose the profile before browsing. That single decision supplies context for the interactions that follow. A profile is not necessarily a separate authenticated account, private area, or age restriction.

The [individual-profile guide](https://norva.tv/blog/individual-household-media-profiles/) explains when a recurring viewer deserves a distinct context.

## Favourites answer “what do I want to find again?”

A favourite should be intentional. In a shared household, avoid using one person’s favourite list as a catch-all household queue. If two people independently want the same item, each can save it under the relevant context where the current product supports that behaviour.

Review favourites occasionally. Remove items only after confirming that the correct profile is active.

## History answers “what happened before?”

History is descriptive, not necessarily an endorsement or future plan. A brief preview can appear beside a completed item. Mixed profiles make the record harder to interpret and can lead to false sync diagnoses.

Use [the separate-profile history guide](https://norva.tv/blog/separate-profiles-watch-history/) when activity from several viewers appears combined.

## Progress answers “where should I continue?”

Progress is most useful when the same viewer profile and exact catalog version remain consistent across screens. Two versions with similar artwork can have separate media entries, and another profile can have another stopping point.

Before reporting a sync failure, confirm account, profile, item, and version. Norva supports progress synchronisation across supported devices, but viewer context still matters.

## Preferences answer “what should be tried first?”

Language and subtitle preferences can reduce repeated setup. They do not guarantee that every item contains the preferred track. The actual inventory remains dependent on the media and authorised source.

Do not turn a preference into an assumption about another household member. Ask before changing a shared session.

## Adopt a clean handoff routine

1. End playback through the normal controls.
2. Return to a clear browsing state.
3. Switch to the next viewer’s profile.
4. Confirm the profile before opening history or favourites.
5. Verify the exact version before playback.

The broader [shared-library organisation workflow](https://norva.tv/blog/organize-shared-household-library/) turns this into a five-rule household agreement.

## Common mistakes and limitations

- Treating history as a curated watchlist.
- Treating favourites as a universal household queue.
- Switching profiles only after playback starts.
- Assuming similar artwork means the same version.
- Presenting profiles as security or parental controls.
- Claiming every category is isolated identically without a current product check.

The plan names Norva and Norva Family describe profile allowances: currently up to 2 and up to 5 respectively. They do not establish device or simultaneous-playback limits.

## Frequently asked questions

### Is a favourite the same as watch history?

No. A favourite is an intentional save; history records prior activity. One item can appear in both for different reasons.

### Can two profiles favourite the same item?

Where the current product scopes favourites by profile, that is a reasonable outcome. A reviewer must verify the exact behaviour before publication.

### Does deleting history remove the media?

Do not assume so, and do not change data without checking the current control. The catalog remains associated with the connected source; history is an interaction record.

### Are profiles private from one another?

Not necessarily. Treat profiles as viewer contexts, not authentication boundaries, unless a verified control explicitly says otherwise.
