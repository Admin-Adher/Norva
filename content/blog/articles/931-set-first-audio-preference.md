# Set and Verify Your First Audio Preference

> **In short:** First confirm which audio tracks the authorized source actually provides. Set one preference on the intended Norva profile, test it with a known multilingual item, exit cleanly, and reopen the item. Then test a second compatible item. Record missing-track cases separately because a language preference cannot create audio that the source or media does not contain.

An audio preference expresses a viewing choice; it is not a guarantee about every title. Verification must separate the preference, the source track inventory, and the player result. Complete the [first playback routine](/blog/verify-first-playback-session/) before testing persistence so basic start and control issues are already excluded.

## Choose the right profile

Confirm which household profile should own the preference. A shared profile may need a neutral household choice, while a personal profile can reflect one viewer's recurring language. Do not change several profiles during the same test.

Record the profile using a privacy-safe shorthand. Avoid full names, emails, precise locations, or source identifiers in evidence.

## Establish the source track baseline

Through the source's authorized workflow, choose a familiar item with at least two known audio tracks. Record the language labels and any broad edition distinction needed to recognize the item. Do not infer an available track from the title, poster, country, or description.

Actual audio availability depends on the media and source. Device capabilities, rights, and current supported behavior may also affect playback choices.

## Set one preference

Open the current Norva preference control and choose the intended language. Read the interface label rather than relying on a remembered location, because presentation can differ by supported surface. Save or close the control using the visible workflow.

Do not simultaneously change subtitles, version grouping, or profile. One-variable testing keeps the result interpretable.

## Test the known item

Start the item and inspect the active audio choice. Listen briefly to confirm that the selected track matches the label when that can be determined safely. Record whether the preferred track was selected automatically, required a manual choice, or was unavailable.

If the item has multiple versions, confirm that the tested version is the one used for the source baseline. A different edition can contain a different track set.

## Test persistence on the same item

Exit through the interface, reopen the same item under the same profile, and inspect the active audio choice again. Record the result rather than assuming a saved setting persisted. A successful repeat on one item is evidence for that environment, not a universal promise.

The [first progress sync check](/blog/run-first-progress-sync-check/) should remain separate; audio preference and playback position answer different questions even when both are associated with a profile.

## Test a second compatible item

Choose a second item whose source baseline includes the preferred language. This distinguishes an item-specific result from broader profile behavior. Keep the same profile, screen, and network.

Then choose, if useful, one item known not to contain the preferred track. The correct observation may be a documented fallback or another available choice. Do not report that absence as a preference-setting failure.

## Compare another screen only later

Norva can keep preferences across supported devices under the same account, but the actual media and device still determine what can play. Once same-screen behavior is understood, repeat one known-item check on another supported surface without changing the source baseline.

Follow the [Norva onboarding journey](/blog/norva-onboarding-complete-journey/) to place that cross-screen check alongside progress, favorites, and trusted-device review.

## Common audio-preference mistakes

- Assuming a language from artwork or description.
- Testing the wrong version of an item.
- Changing audio and subtitle preferences together.
- Treating a missing source track as a saved-setting defect.
- Generalizing one successful item to every device and format.
- Sharing screenshots that expose household or source details.

## Frequently asked questions

### Can Norva add an audio track that is not in the media?

No. A preference can help choose among available supported tracks; it cannot create a track absent from the source or media.

### Why might two versions have different languages?

Versions can be separate media records with different encodes, editions, rights, or source metadata. Verify the exact version before comparing track lists.

### Should audio and subtitle preferences use the same language?

Not necessarily. They serve different needs. Test each setting independently and choose the combination appropriate for the profile.
