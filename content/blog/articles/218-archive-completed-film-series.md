# How to Archive a Completed Film Series Without Losing Context

> **In short:** Archive the viewing project, not the movies. Verify what “complete” meant, freeze the relationship graph and viewing route, record each selected version and outcome, preserve deliberate exclusions and unresolved branches, then remove the project from active queues. Keep an archive date and reopen trigger so a new installment or corrected relationship creates an addendum rather than erasing the original project.

Here “film series” means a connected set of movies or franchise, not a season-based television series. Completion belongs to a declared project scope; the underlying works remain active library records.

## Confirm completion against scope

Retrieve the original route manifest and ask:

- Which works were included?
- Which viewing order was used?
- Were spin-offs, remakes, or supplements intentionally excluded?
- Did every included work reach a completed or documented terminal state?
- Are any interruptions or version uncertainties unresolved?

If scope was never recorded, reconstruct it before archiving. “Everything” is not a useful completion rule when franchise boundaries are disputed.

Use [the franchise viewing-order map](/blog/arrange-film-franchise-viewing/) to document release, story, thematic, or selective routes.

Do not replace a selected version with whatever is currently available later. The archive records what happened. A future rewatch can create another entry.

## Remove only active state

Clear temporary queue position, open project reminders, and active theme membership after the manifest is safe. Preserve:

- work identity and relationships;
- favorites that still reflect current intent;
- viewing history or progress records as supported;
- personal ratings and dated notes;
- version comparisons;
- the archived route.

W3C notification guidance recommends clear outcomes and recovery. An archive action should say what moved out of active view, what remains, and how to restore it.

## Test retrieval

Close the active project, then find the archive through its name, one included work, completion date, and relationship. Open two entries and confirm their version notes and route positions remain understandable.

If retrieval fails, the project is hidden, not archived.

Also test the archive from another supported device or after a fresh session when the workflow depends on synchronized state. Confirm that active filters do not make the project appear missing. Record whether the manifest itself, only the underlying works, or both are expected to travel across devices; do not infer that scope from one successful screen.

## Define the reopen trigger

Possible triggers include:

- a verified new installment;
- a newly available alternate cut;
- corrected relationship metadata;
- a group rewatch with another route;
- an unresolved branch becoming available.

Reopen as version 2 or an addendum. Preserve the earlier completion statement because it was true for its scope and date.

For temporary cross-franchise projects, [the viewing-theme archive method](/blog/track-viewing-themes-without-clutter/) may fit better. Before closing, use [the movie library workflow checklist](/blog/movie-library-workflow-checklist/) to catch stale active state.

Norva may retain history, progress, favorites, and version context across supported devices while organizing compatible authorized sources. Exact archive or collection controls should be verified in the current product; an external manifest remains a safe fallback.

## Common mistakes and limitations

- Deleting work records instead of closing the project.
- Claiming universal franchise completion without scope.
- Replacing historical version notes with current availability.
- Losing deliberate exclusion reasons.
- Archiving before testing retrieval.
- Rewriting the old manifest when a new work appears.

## Frequently asked questions

### Should completed movies leave favorites?

Only if favorites mean future intent in your workflow. Completion alone should not decide the state.

### What if a new installment appears?

Create an addendum or new project version and preserve the original scope and completion date.

### Is archive the same as delete?

No. A soft archive removes active project state while keeping identity, relationships, history, and recovery information.
