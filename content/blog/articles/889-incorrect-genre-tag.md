# Incorrect Genre Tag? Separate Source Data From Display Rules

> **In short:** Confirm the exact item and source version, then record every labeled genre, category, collection, or tag field at the authorized source. Separately capture Norva genre tags, category placement, search results, profile, filters, grouping, device, application version, and timestamp. Compare one unaffected item from the same source. Preserve the original values and avoid assuming how Norva maps, combines, prioritizes, or displays source classifications.

“Genre” can refer to a provider field, a category label, a search facet, or a descriptive tag. The same word on two screens does not prove the fields have the same role.

Write down the field's exact screen location as well as its wording so later reviewers compare equivalent presentation contexts.

## Confirm item and version identity

Record source label, title code, year, media type, duration, edition, season, episode, and language cues. If the comparison involves different versions, resolve identity before calling one genre wrong.

The [category and metadata handbook](/blog/category-metadata-troubleshooting-handbook/) provides the field matrix.

## Transcribe source fields separately

Through the provider's official authorized route, record every visible classification field with its exact label and value. Keep genre, category, collection, keyword, folder, and parental label separate. Do not assume Norva consumes or combines them.

## Record Norva presentation by view

Capture genre text on the detail page, category placement, search facets, card badges, and related views where relevant. A mismatch limited to one screen is different from a value shown consistently throughout the interface.

## Freeze view controls

Record account, profile, enabled sources, availability, year, rating, audio, subtitles, category, search query, sorting, and grouping. A genre filter can alter which card or version appears and make the comparison switch identities.

## Compare a control item

Choose one unaffected entry from the same source and media type with clear source classification. Record identical fields and view context. A matching control limits scope; it does not reveal an internal mapping rule.

## Check category placement

Note whether the item appears in expected, unexpected, multiple, or no categories. An uncategorized item is a separate observation covered by the [classification check](/blog/uncategorized-items-after-import/). Do not equate category membership with a detail-page genre tag without verified documentation.

## Compare same-name categories carefully

If the unexpected genre matches a duplicate-looking category label, compare source scope, media type, parent view, and member samples using the [duplicate-category guide](/blog/duplicate-category-names/).

## Preserve timing and ownership

Record source field confirmation, source edit history, import or refresh, first Norva observation, application update, and current state. Identify who owns the source metadata and whether the provider supports corrections. Do not claim an update schedule that current official guidance does not publish.

## Avoid editing to test a theory

Do not replace genres, move folders, rename categories, repeat imports, or clear application data before saving the baseline. A source edit changes the input and can create a second problem. If support recommends a correction, record before and after.

## Classify the mismatch

Use different item or version, source genre absent, source genre present, source category differs, Norva view-specific tag, category-placement difference, profile or filter difference, device-specific presentation, source changed after import, or unknown. Never state an undocumented display or priority rule as fact.

## Prepare support evidence

Use the [metadata evidence pack](/blog/metadata-support-evidence-pack/) to include item cues, exact field labels, both values, views, stable context, timeline, and one control. Exclude credentials, complete catalogs, and unrelated household data.

## Common mistakes and limitations

- Treating genre, category, folder, and keyword as one field.
- Comparing different versions or filtered cards.
- Assuming category membership defines detail-page genre.
- Editing the source before preserving original values.
- Inventing mapping or priority behavior.
- Exporting a full catalog for one tag mismatch.

## Frequently asked questions

### Does a source genre guarantee the same Norva tag?

Do not assume a mapping that current Norva documentation does not verify. Record both fields independently.

### Should I change the source genre?

Only after preserving evidence, confirming ownership, and following provider and Norva support guidance.

### What if the item appears in several categories?

Record each category and view. Multiple placements do not by themselves prove the displayed genre is wrong.
