# How to Introduce a New Household Profile Without Clutter

> **In short:** Add a profile only when a recurring viewer or group needs separate progress, history, favourites, or preferences. Define its owner and purpose first, verify current plan capacity, choose a short distinct name, set only essential language intent, and run one harmless session. Do not fill the profile with speculative favourites; let real use establish its context.

Profile clutter begins when a household creates choices before deciding who owns them. A new profile should have a one-sentence purpose and a clear shared-screen entry and exit routine.

## Confirm that a profile is needed

Write:

“This profile belongs to ___ for ___ recurring viewing context because ___ state must remain separate.”

If the reason is a one-time guest, one device, or visual symmetry, reconsider. The [profile-count worksheet](/blog/how-many-household-profiles-do-you-actually-need/) distinguishes persistent contexts from temporary demand.

## Verify current capacity and controls

Review the official pricing page for current profile allowances and plan conditions. Do not infer device count or simultaneous-playback permission from profile capacity.

Use only the profile creation and management controls currently visible in Norva. If consequences are unclear, contact support before replacing or deleting an existing context.

## Choose a clear, private name

Make the opening characters distinct from existing labels. Use only enough personal information for recognition on a shared screen. Avoid email addresses, birthdays, source details, or sensitive descriptions.

The [shared-TV naming audit](/blog/how-to-name-profiles-clearly-on-a-shared-tv/) tests labels from the normal viewing distance.

## Configure minimum preference intent

Record, without overconfiguring:

- preferred audio language or “decide per item”;
- subtitle on/off intent or “decide per item”;
- fallback when a track is absent;
- whether the profile is personal or an agreed recurring group.

Available audio and subtitles depend on the source and media. A preference cannot create tracks.

## Do not prefill history

Avoid starting many items merely to teach or decorate the new profile. That creates artificial progress and history. Add a favourite only when the profile owner genuinely wants it and the item identity is verified.

Recommendations shown by Norva come from the connected source; do not promise that onboarding actions will produce a specific recommendation outcome.

## Introduce the shared-screen routine

Show the owner how to:

1. read the profile label;
2. enter the library;
3. confirm item and version;
4. pause and return;
5. hand off the screen;
6. sign out when the device is not trusted.

The [complete household profile guide](/blog/the-complete-guide-to-household-media-profiles/) explains why profile selection is separate from account security.

## Run one harmless session

Choose an authorised item with clear metadata. Confirm the profile, start a short section, pause, and reopen the profile to verify the expected progress and preferences. Do not claim a measured result from one test.

Record any confusing name, icon, or entry path and correct only that issue.

Possible review triggers include changed household membership, repeated wrong-profile starts, no recurring use, a plan change, or a privacy concern.

## Review without arbitrary deadlines

Assess the profile after enough normal use to know whether its purpose exists. Do not publish an invented time limit. If it remains unused, first preserve any genuine state and understand deletion consequences before removing it.

## Common mistakes and limitations

Avoid creating a profile per device, copying another profile's favourites, using sensitive names, starting items to fill history, and deleting an older context to make space without review.

Current features and plan capacity can change. This workflow reduces organisational clutter but does not provide parental controls, account security, or guaranteed recommendation behaviour.

## Frequently asked questions

### Should I add favourites during setup?

Only items the profile owner genuinely wants. Avoid speculative bulk additions.

### Can one person have several profiles?

Only when distinct recurring contexts justify the complexity and current capacity supports them. One profile across supported devices is usually the clearer baseline.

### When should an unused profile be removed?

After confirming it has no continuing owner or purpose and reviewing the consequences through current controls. Preserve important state first.
