# When a Browser-Based Media Player Is the Most Convenient Choice

> **In short:** A browser-based player is convenient when you already have a compatible computer, do not want to install an application, and benefit from keyboard or pointer input. It is less suitable on an untrusted shared computer, when a remote-first TV interface is important, or when the browser and connected source do not meet current requirements.

“No installation” removes one setup step, but it does not remove every decision. The computer, browser, account context, source availability, and privacy of the environment still determine whether Web is the right choice.

## Use Web for a quick, controlled start

Norva's Web experience opens in a compatible browser without installation. That can fit:

- a personally controlled laptop or desktop;
- a temporary session on a supported work area where installation is not appropriate;
- a search-heavy task that benefits from a keyboard;
- a personal large-screen session at a desk;
- access to the same account state used on other supported devices.

For setup-specific checks, use [how to set up Norva Web](/blog/set-up-norva-web/) alongside the current on-screen instructions.

## Prefer another experience when context demands it

TV may be a better fit when:

- the session is shared across a room;
- remote navigation matters;
- sofa-distance readability is central;
- a longer viewing session is planned.

Mobile may be more appropriate when:

- the screen needs to move with you;
- the session is personal and close;
- touch input is preferable;
- you intend to begin a cross-screen handoff.

Compare the contexts in [Norva Mobile, Web, or TV](/blog/norva-mobile-web-tv-comparison/). The comparison is not a quality ranking.

## Run the browser suitability checklist

Answer yes or no:

1. Is the browser compatible with the current Norva Web experience?
2. Is the computer controlled by you or a trusted person?
3. Is the compatible media source available for this account context?
4. Would keyboard or pointer input improve the task?
5. Is installation unavailable or undesirable?
6. Do you need progress and preferences from another supported device?

Web is a strong candidate when the first three are yes and the later answers favour the browser. If the device is not trusted, convenience should not override account privacy.

## Protect the account on shared computers

On a computer you do not control:

- avoid saving credentials in the browser;
- confirm that you are using the official Norva address;
- sign out when finished;
- close the browser session;
- review device or account controls if the session remains visible;
- do not expose source credentials.

Consult the [Norva Privacy Policy](https://norva.tv/privacy) for current information about data handling and account controls. Do not infer anonymity, end-to-end encryption, or zero data collection.

## Understand continuity across screens

Norva synchronises progress, history, favourites, and preferences across supported devices. This means a browser session can participate in a wider workflow:

1. inspect a title on Web;
2. add it to Favorites;
3. start or resume on another supported screen;
4. return to Web with the account's updated state.

The [playback progress sync guide](/blog/playback-progress-sync-explained/) identifies the profile, title version, and connectivity checks required when a resume point looks wrong.

## Do not overstate browser portability

Browser-based does not mean every browser, operating system, source, or media item is supported. Use the qualifiers:

- in a compatible browser;
- on supported devices;
- with a compatible media source you are authorised to use;
- when the source and media provide the required options.

If the page fails to open or playback cannot start, verify the official support guidance rather than repeatedly changing browser security settings without understanding the effect.

## When a browser is especially useful

### Catalogue maintenance

Keyboard and pointer input can make searching, reviewing metadata, or comparing several entries more direct.

### A temporary no-install session

Web avoids local application installation on a compatible computer. Account hygiene still matters.

### Cross-screen planning

The browser can serve as a place to inspect and save choices before using a TV. Favorites and preferences can follow on supported devices.

### Personal desk viewing

A desktop or laptop may suit a private session when the room-scale TV experience is unnecessary.

## Common mistakes and limitations

- Assuming every browser is supported.
- Saving account access on an untrusted computer.
- Treating Web as identical to a remote-first TV interface.
- Expecting media options absent from the connected source.
- Assuming no installation means no account state or data processing.
- Leaving a shared browser session signed in.

## Frequently asked questions

### Does Norva Web need to be installed?

No. Norva states that the Web experience opens in a compatible browser without installation.

### Can my progress follow from Web to TV?

Norva synchronises progress across supported devices. Use the same account, profile, and media version, and make sure the devices can connect.

### Is a private browsing window enough on a shared computer?

It can reduce local browser persistence, but it does not replace signing out, verifying the official address, or following the computer owner's security policy.
