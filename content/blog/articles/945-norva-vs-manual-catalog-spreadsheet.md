# Norva vs. a Manual Catalog Spreadsheet: What Changes?

> **In short:** A spreadsheet offers explicit columns, manual correction, and portable files, but every entry and update requires a person or separate automation. Norva can organize a compatible authorized source, group source-derived variants, and connect catalog discovery with playback. Compare both using the same representative sample, including setup, updates, errors, privacy, recovery, and current cost.

A spreadsheet is not merely a simpler Norva, and Norva is not merely a visual spreadsheet. One is a general-purpose record; the other is a media player and organizer. The comparison should focus on the household tasks each workflow must complete.

## Define a fair sample

Choose twenty non-sensitive records from a source you own or are legally authorized to use. Include movies, two multi-season series, a variant case, and a few items with language or subtitle metadata. Use neutral item codes in the test log.

Norva does not include a media catalog. The source supplies the media and available metadata; the spreadsheet also requires the evaluator to obtain and maintain legitimate source information.

## Measure initial setup

For the spreadsheet, define columns, enter records, validate formats, create filters, and decide where the file is stored. For Norva, include account setup, source connection, initial import observation, and a [small catalog verification](/blog/verify-first-catalog-sample/).

Measure hands-on time and elapsed time separately. An import can take time without requiring continuous work, while manual entry may be shorter in elapsed time for a tiny sample but more labor-intensive.

## Compare identity and hierarchy

Search for known movies and inspect two series. In the spreadsheet, check whether rows, season fields, and episode fields are understandable without special knowledge. In Norva, inspect work, season, episode, and version presentation under neutral filters.

Record corrections required in both workflows. Do not edit source metadata during the benchmark; preserve a stable baseline.

## Compare variants

For the selected variant case, record which fields distinguish edition, quality, audio, subtitles, or source identity. A spreadsheet can expose every chosen column but relies on consistent entry. Norva may group variants using source-derived information but cannot guarantee a correct merge when identity data is missing or conflicting.

Score how quickly a viewer can choose the intended version, not merely how compact the catalog appears.

## Compare search and filtering

Run five tasks: find by title, locate a series episode, identify a language option, filter by a useful category, and return to the unfiltered view. Count actions, time, errors, and whether the result is understandable to another household member.

A spreadsheet's flexibility may require carefully maintained columns. Norva's purpose-built controls may reduce configuration but constrain the fields exposed in the current interface.

## Test an update and correction

Add one authorized source item and correct one known metadata value through the appropriate source administration. Observe how each workflow reflects the change. In the spreadsheet, measure manual entry and verification; in Norva, observe the supported refresh or import behavior.

Do not assume instant updates. Record actual signals, elapsed time, and any intervention.

## Compare playback handoff

From each catalog, reach the intended item and begin playback through the authorized workflow. The spreadsheet may rely on a link or manual source navigation. Norva connects its catalog presentation to supported playback. Count broken links, wrong versions, repeated authentication, and recovery steps.

Use the broader [Norva evaluation framework](/blog/norva-evaluation-framework/) for playback controls and cross-screen continuity beyond this handoff.

## Compare privacy and recovery

The spreadsheet may reveal the entire catalog, viewing notes, source locations, or household identities in one file. Review access, sharing, backups, version history, deletion, and device loss. Norva's current privacy policy should be evaluated for account, source configuration, device, and viewing-context data.

Neither workflow is automatically private. Minimize fields, use neutral labels, and exclude credentials from both.

## Compare recurring effort and cost

Project monthly additions, corrections, backups, household questions, and device changes. Use current Norva pricing rather than a fixed amount. For the spreadsheet, include software cost if any and the value of maintenance time.

Use the [trial scorecard](/blog/norva-trial-evaluation-scorecard/) to prevent a visually impressive result from hiding repeated maintenance or a failed requirement.

## Frequently asked questions

### Is a spreadsheet better for a very small catalog?

It may be sufficient when the catalog is small, changes rarely, and the household accepts manual maintenance and its privacy and backup model. Test real tasks rather than assuming.

### Can Norva fix all source metadata?

No. Organization depends partly on source-derived information. Preserve a source baseline and distinguish source corrections from presentation behavior.

### Can I use both workflows?

Yes, but dual maintenance has a cost. Define which system is authoritative and avoid copying credentials or unnecessary private viewing data into the spreadsheet.
