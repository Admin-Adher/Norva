# How to Arrange a Film Franchise for Different Viewing Orders

> **In short:** Build one verified relationship map, then derive viewing routes from it. Release order follows first public release; story order follows verified in-world chronology; thematic order groups a declared question; selective order keeps only the works needed for the viewer's goal. Keep remakes, spin-offs, and alternate cuts typed separately, and never present a disputed chronology as the only correct route.

A franchise is a network of works, not a folder whose filename order automatically explains the experience. The library should preserve facts once and allow several viewing paths.

## Inventory works before ordering them

Create one row per distinct work:

| Work | Release date | Relationship | Story period | Essential context | Available version | Evidence status |
|---|---|---|---|---|---|---|
|  |  | Sequel / prequel / spin-off / remake |  |  |  |  |

Do not list alternate cuts as additional franchise installments. Link those beneath the same work and choose a version later with [the alternate-cut workflow](/blog/choose-between-alternate-cuts/).

Dublin Core provides `isPartOf`, `hasPart`, `isVersionOf`, and broader relation properties. The useful principle is to state what a connection means rather than putting every related record in an undifferentiated group.

## Verify the relationship graph

For each connection, record whether it is explicit, strongly supported, disputed, or merely thematic. Separate:

- direct continuation;
- prequel;
- side story or spin-off;
- remake or rebooted continuity;
- crossover;
- shared setting without required sequence;
- bonus or supplemental material.

The FIAF cataloguing model distinguishes works from variants and manifestations. Preserve that boundary so a restored edition does not become a new installment and a remake does not become a cut.

## Build four route cards

### Release route

Sort distinct works by verified original release date. Use this route when the viewer wants the sequence in which audiences first encountered information. Decide where later supplemental material belongs and document same-day ties.

### Story route

Sort by verified story chronology. Use ranges or “uncertain placement” when a work spans periods or the chronology is disputed. Do not invent a precise position from vague clues.

### Thematic route

State one question—such as a character, institution, visual motif, or genre shift—and order works to develop that question. This route need not include every installment.

### Selective route

Define the minimum context required for a specific goal: a weekend, one character arc, or preparation for a new installment. Explain every exclusion so “shorter” does not become “arbitrary.”

## Compare the routes

| Route | Best for | Main risk | Entry requirement | Exceptions documented? |
|---|---|---|---|---|
| Release | Historical reveal order | Older pacing may challenge some viewers | None beyond first work |  |
| Story | In-world chronology | Spoilers or disputed dates | Chronology evidence |  |
| Thematic | Focused comparison | Curator bias | Declared question |  |
| Selective | Limited time | Missing context | Minimum-context test |  |

Choose based on the viewer, not on a claim that one route is universally correct.

## Select versions after the route

Once the work order is stable, verify the current cut, runtime, audio, subtitles, and source for every step. A route can be correct while a chosen version is unsuitable.

Use [the weekend queue workflow](/blog/create-weekend-movie-queue/) to translate the route into realistic time blocks without changing the relationship graph.

## Record progress and branch points

Mark completed works and the next planned step. At a branch, explain the choice: continue the main sequence, visit a side story, or skip to the next essential work. Keep the alternative route available rather than rewriting the franchise structure.

When the project finishes, [archive the completed film series softly](/blog/archive-completed-film-series/) so the route, versions, and unresolved branches remain recoverable.

Norva may group versions and organize compatible authorized sources, but franchise relationships and chronological claims depend on connected metadata and external verification. The user should review disputed or incomplete relationships.

## Common mistakes and limitations

- Treating every related title as one continuity.
- Mixing release date with local source-added date.
- Counting alternate cuts as installments.
- Presenting story chronology without uncertainty.
- Choosing versions before deciding the work route.
- Deleting excluded works from the relationship map.

## Frequently asked questions

### Is release order always safest for spoilers?

It often preserves historical reveal order, but supplemental material and re-edits can complicate that. Verify the specific route and versions.

### Where should a remake go?

Keep it as a related new work, usually in a separate continuity or comparison route unless evidence supports another structure.

### Can one franchise have several saved orders?

Yes. Preserve one relationship graph and save several named route manifests with distinct goals.
