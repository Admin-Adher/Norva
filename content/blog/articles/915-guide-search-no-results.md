# Guide Search Finds No Results? A Query and Data Check

> **In short:** Record the exact query, capitalization, punctuation, language, selected guide date and window, filters, source and channel scope, device, application version, and timestamp. Confirm the target listing currently exists in the authorized source and can be found by browsing the same guide window. Test a short title cue, one known control query, and another supported device. Do not infer which fields Norva searches or repeatedly refresh the guide.

No results can mean the target listing is outside the visible data, hidden by context, expressed in another language, absent from the source, or not matched by the query. It can also coexist with a search-interface or focus problem.

## Preserve the exact query

Record every character, including spaces, punctuation, accents, apostrophes, numbers, season or episode notation, and capitalization. Do not rely on memory after trying several variants.

The [TV guide handbook](/blog/tv-guide-troubleshooting-handbook/) provides the data and view matrix.

## Confirm guide scope

Record account, profile, authorized source, channel group, selected date, visible guide window, favorites or availability view, and all filters. A listing outside that scope is not a valid no-result test for the current context.

## Verify target presence at the source

Through the provider's official authorized route, confirm the exact channel, listing title cue, start/end times, language, and timestamp. If the source does not currently expose it, classify source absence before changing the query.

## Verify target presence by browsing

Navigate to the same channel and guide window using ordinary controls. Record whether the target appears in the grid or detail view. A browsable listing with no search match is different from a listing absent everywhere.

## Record language and title variants

Capture source title, localized or alternate title, episode title, and interface language where visible. Test one short distinguishing cue rather than many speculative translations. Do not assume which title field search uses.

If episode fields are missing, use the [episode-data checklist](/blog/episode-data-missing-in-guide/).

## Remove one visible filter

Change one source, group, date, favorites, or other visible filter, run the same query, then restore baseline. The [hidden-by-filter guide](/blog/guide-filter-hides-channels/) helps when channel scope is the determining context.

## Use control queries

Choose one currently visible listing with a short distinctive title and one very common cue. Record results. A distinctive control helps separate target-specific data from a screen where no query returns anything.

## Check input and focus state

Record whether the search field has visible focus, typed characters appear, the query can be cleared, and results respond to submission or text changes through documented behavior. Do not treat a remote-focus problem as missing search data.

## Compare another supported device

Use the same account, profile, source, group, filters, guide window, query, and close timestamp. Record both application versions and input methods. A result on one device is cross-screen evidence, not proof of search indexing behavior.

## Preserve the timeline

Record source listing confirmation, guide open or refresh, first query, each controlled variant, application update, and current result. Do not run dozens of variants or repeated refreshes that make the sequence unreadable.

## Avoid unsupported assumptions

Do not claim Norva searches titles, descriptions, channels, episodes, or future windows unless current documentation verifies it. Do not edit source titles or descriptions merely to force a match.

## Classify the result

Use source listing absent, outside guide window, hidden by filter or group, target browsable but not matched, language or title variation, no queries return results, input or focus issue, device-specific search difference, or unknown.

## Prepare support evidence

Use the [guide evidence template](/blog/guide-issue-support-evidence/) with exact queries, scope, target and control results, source and grid presence, language, device versions, timestamps, and actions. Add the [description context check](/blog/program-description-wrong/) only when wrong text affects identification.

## Common mistakes and limitations

- Searching outside the selected guide window.
- Ignoring group, source, favorites, or language context.
- Trying many variants without recording them.
- Assuming which metadata fields are searched.
- Editing source text to force a result.
- Treating an input-focus issue as no matching data.

## Frequently asked questions

### Does guide search include descriptions?

Use current Norva documentation for that claim. The interface result alone does not reveal which fields are searched.

### Should I try every spelling variation?

No. Use a small recorded set: exact cue, short cue, one localized cue when relevant, and a known control.

### What if browsing finds the listing?

Record that the listing is present in the same guide scope but the query did not match; this is useful focused evidence.
