# Synopsis Still Stale After Refresh? What to Compare

> **In short:** Confirm the exact item and source version, then preserve short distinguishing excerpts from the old synopsis, current authorized-source synopsis, and every relevant Norva view. Record language, account, profile, filters, grouping, device, application version, source edit and confirmation times, refresh request, visible state, and current observation. Compare another supported device and a control item, and avoid repeated edits or claims about undocumented refresh timing.

A synopsis that looks old may belong to another version, language, or view. A reliable comparison pairs exact text with identity and time instead of relying on memory.

## Confirm item and version identity

Record source label, title code, year, media type, duration, edition, season, episode, language, artwork, and grouping. Do not compare descriptions until the same logical entry is established.

The [category and metadata handbook](/blog/category-metadata-troubleshooting-handbook/) provides the identity matrix.

## Preserve concise text evidence

Record a short distinguishing phrase from the old text and the current source text, plus approximate length and language. Do not copy an entire protected description into public notes. For sensitive household content, use redacted sample codes.

## Confirm the current source synopsis

Through the provider's official authorized route, record the exact field label, language, current value cue, and confirmation time. If several descriptions or languages exist, keep them separate. Do not assume one field is selected universally by Norva.

## Record Norva text by view

Compare category card, search result, detail page, episode view, favorite, progress row, or grouped-version page only where synopsis text appears. A stale value on one screen is a view-specific observation.

If text remains visible without a usable entry, use the [metadata-without-entry guide](/blog/metadata-visible-media-entry-missing/).

## Freeze language and view context

Record interface language, profile, source selection, availability, filters, search, sorting, grouping, device, and application version. Change one supported language or view context at a time and restore the baseline.

## Build the edit and refresh timeline

Record source edit submission, source confirmation, import or refresh request, visible acknowledgment, completion or last change, first Norva observation, and every retry. Include timezone. Use only current Norva timing guidance when published; otherwise report elapsed time without a promise.

## Compare another supported device

Use the same account, profile, item, source, filters, grouping, language, and close timestamp. Record both application versions. Different text on one device is cross-screen evidence, not proof of local storage behavior.

## Use a control item

Choose one item from the same source and media type whose synopsis changed or remained stable as expected. Record identical fields. A healthy control narrows scope but does not reveal how Norva selects or retains descriptions.

## Avoid repeated corrections

Do not edit the synopsis again, rename the item, remove the source, repeat imports, clear application data, or reinstall before saving the first sequence. The [correction ownership guide](/blog/metadata-corrections-overwritten-after-refresh/) helps when a source refresh replaces a household edit.

## Classify the observation

Use different item or version, source text not yet confirmed, different source field, language difference, view-specific old text, device-specific text, source edit after refresh, updated after documented step, or unknown. Never label a cache, retention period, or field priority as fact without verified documentation.

## Build a support pack

Use the [metadata evidence pack](/blog/metadata-support-evidence-pack/) with item cues, short text excerpts, source field and time, Norva views, stable context, timeline, control item, and actions. Exclude complete descriptions, credentials, and private source addresses.

## Common mistakes and limitations

- Comparing descriptions from different versions or languages.
- Relying on memory instead of concise text evidence.
- Repeating edits and refreshes before saving the first sequence.
- Assuming one view represents every view.
- Inventing a refresh delay or field priority.
- Sharing full protected descriptions unnecessarily.

## Frequently asked questions

### How long should a synopsis update take?

Use a current Norva support statement if one exists. Otherwise report exact timestamps and elapsed observations without inventing a deadline.

### Should I edit the source text again?

Not before preserving the first edit, confirmation, refresh, and display sequence. Re-editing creates a new input.

### What text should I send support?

Use only short distinguishing excerpts, field labels, language, timestamps, and redacted item identity rather than complete descriptions.
