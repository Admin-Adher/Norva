# A Cross-Device Handoff Readiness Checklist

> **In short:** Before handoff, pause the source and verify account, profile, authorised source, item, version, position, audio, and subtitles. On the target, confirm support and rebuild those states before playback. Then check output, controls, privacy, and the final source close. Mark unknowns “defer”; do not convert them into assumptions.

This is the operational companion to the [state-by-state handoff guide](/blog/a-state-by-state-guide-to-cross-device-viewing-handoff/). It verifies state correctness, not performance or universal compatibility.

## Source-screen preflight

- [ ] The source device is supported and stable.
- [ ] Playback is paused.
- [ ] The intended account is active.
- [ ] The intended profile is visible.
- [ ] Full item or episode identity is recorded.
- [ ] Version or source label is recorded.
- [ ] Approximate position is recorded.
- [ ] Audio and subtitle states are recorded.
- [ ] Source connectivity or eligible local state is understood.
- [ ] The source will remain paused until target confirmation.

Do not begin with artwork or a continuation card as the only identity evidence.

## Target-device readiness

- [ ] The target is supported and opens the verified Norva route.
- [ ] The device shows no unresolved security or update warning.
- [ ] A trusted network is available when required.
- [ ] Power is sufficient for the transition.
- [ ] The control method is usable.
- [ ] The audio output can be verified.
- [ ] Shared-screen privacy has been reviewed.

If the target is offline, wait or use only eligible local media that was already prepared and tested.

## Core match gate

- [ ] Account matches.
- [ ] Profile matches.
- [ ] The same compatible authorised source is accessible.
- [ ] Item or episode matches.
- [ ] Version matches or an alternative is explicitly accepted.
- [ ] Proposed position is plausible.
- [ ] Required audio track is available.
- [ ] Required subtitle track is available.

The [handoff prerequisite guide](/blog/what-must-match-before-a-cross-device-handoff-can-work/) explains why the first five rows are blockers.

## Target playback check

- [ ] Starting level is reduced.
- [ ] Intended output route is selected.
- [ ] Play or resume is activated once.
- [ ] The scene and timeline match.
- [ ] Audio and subtitles are rechecked.
- [ ] Pause and return controls work.
- [ ] No unexpected account or source message appears.

Do not adjust the timeline until item and version identity pass.

## Shared-screen privacy check

- [ ] Profile and history exposure is acceptable.
- [ ] Credentials are not visible or saved unexpectedly.
- [ ] Notification previews are handled according to device policy.
- [ ] Account settings remain out of the viewing flow.
- [ ] The session has a defined sign-out or handoff endpoint.

A profile organises context; it does not secure a signed-in account from someone who can access it.

## Source close and target exit

- [ ] Target playback is confirmed.
- [ ] Source is closed, left paused, or signed out deliberately.
- [ ] Simultaneous-use permission is not inferred from profile count.
- [ ] Final position is recorded when needed.
- [ ] Temporary local media is reviewed.
- [ ] Audio, notification, or orientation changes are restored.
- [ ] Borrowed or untrusted devices are signed out.
- [ ] A reopen test confirms the intended session boundary.

## When a check fails

Stop at the first blocker. Preserve the source and target states, then use [the cross-device failure report](/blog/how-to-document-a-cross-device-handoff-failure/). Include exact steps, expected and observed results, versions, and one controlled comparison. Remove all private data.

## Common mistakes and limitations

Avoid checking from memory, skipping profile identity, accepting another version silently, testing sound at the previous device's level, and closing the source too early.

Updates, source availability, network state, rights, and device support can change. A completed checklist does not guarantee future handoffs.

## Frequently asked questions

### Must every row pass for a simple move?

Mark genuinely irrelevant presentation rows not applicable. Core account, profile, source, item, and version states still need explicit decisions.

### Should the source stay open after the target resumes?

Only until target identity and playback are confirmed. Then follow the device and household routine.

### Is the checklist proof of device compatibility?

No. It documents one observed session on supported devices; it is not a universal certification.
