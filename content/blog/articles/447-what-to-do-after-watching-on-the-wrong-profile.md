# What to Do After Watching on the Wrong Profile

> **In short:** Pause immediately, record the wrong profile, item, version, and resulting position, then confirm the intended profile's current state without starting playback there. Do not assume progress can be transferred or erase another person's history. Use only visible, verified correction controls; if the right action is unclear, preserve the evidence and contact support.

The goal is to stop one mistake from becoming two. Continuing, seeking, changing favourites, or clearing history before documentation can overwrite useful state.

## Freeze the incident

Pause the wrong-profile session and note:

- account and wrong profile label;
- intended profile label;
- item or episode;
- version;
- approximate starting and current positions;
- audio and subtitles;
- device and local time zone;
- how playback was activated.

Do not rely on memory if several household members use the same series.

Ask both affected viewers to leave the item unchanged while the incident is reviewed. If another normal session must continue, choose unrelated media under the verified profile and record that separation. This preserves the disputed item state without blocking every household activity or creating a second ambiguous timeline.

## Inspect the intended profile without playback

Switch through documented controls and confirm the intended profile. Find the item detail state, but do not press resume. Record its proposed position.

Now you have two contexts: the wrong profile's changed state and the intended profile's existing state.

The [wrong-profile prevention guide](/blog/how-to-prevent-progress-from-landing-on-the-wrong-profile/) explains the pre-play gate to use afterward.

## Decide what needs correction

Separate possible effects:

- progress;
- history;
- favourite state;
- audio or subtitle preference;
- source-derived recommendations.

Do not assume one correction changes all of them. A visible history control is not necessarily a progress-transfer tool.

## Use only verified controls

If the current interface clearly offers removal, reset, or history management for the affected item, read the consequence before activation. Obtain the wrong profile owner's agreement when their context will change.

Do not create fake playback on the intended profile merely to copy the timestamp. That can introduce another inaccurate state.

When the interface presents several possible corrections, write down the expected effect of each before choosing. A control that removes a history entry may not transfer progress, restore preferences, or change a favourite, so verify the scope rather than relying on its label alone.

Norva documentation confirms cross-device preservation of progress and preferences, but does not establish a universal move-progress-between-profiles feature.

## Escalate when transfer or cleanup is unclear

Prepare a concise report with the two profile contexts, item and version, positions, exact actions, expected result, and privacy-safe screenshots. Use [the support-ready state report structure](/blog/how-to-document-a-cross-device-handoff-failure/) as a model.

W3C error-identification guidance reinforces describing the error clearly, but product correction depends on verified Norva controls.

## Restore a safe household state

After correction or documentation, return the shared device to profile selection or another agreed neutral screen. Rename confusing profiles if that can be done safely. Rehearse the profile-label check before the next session.

The [complete household profile guide](/blog/the-complete-guide-to-household-media-profiles/) covers naming and handoff conventions.

## Common mistakes and limitations

Avoid continuing under the wrong profile, seeking to another time, deleting full history, changing the intended profile to match, and claiming the state can be transferred without documentation.

Available correction controls may vary. This workflow protects evidence and household ownership; it cannot guarantee that every state can be moved, removed, or restored.

## Frequently asked questions

### Can I move progress to the correct profile?

Do not assume so. Use only a verified control or support instruction. Manual playback can create further inaccurate state.

### Should I delete the item from history?

Only if the interface clearly explains the action and the affected profile owner agrees. Preserve evidence first.

### What if only a few seconds played?

Pause and document it. The practical impact may be small, but the state should still be understood before correction.
