# How to Resolve Conflicting Preferences Between Profiles

> **In short:** Identify the exact conflict—audio, subtitles, favourites, or another persistent choice—then decide whether it belongs to personal profiles or an agreed shared context. Keep personal defaults separate, define a shared-session choice, and verify the tracks or options actually available on the selected media version. Do not treat one person's preference as the household default by accident.

A “preference conflict” often means two contexts were combined. The solution may be separate profiles, a shared profile rule, or a per-item choice rather than changing the whole account.

Resolve only the state that created the disagreement. Changing language, favourites, version, and profile structure together makes it impossible to tell which decision helped and can disturb unrelated household choices.

## Name the state in conflict

Separate these examples:

- preferred audio language;
- subtitle on or off;
- preferred subtitle language;
- favourites ownership;
- default version choice;
- history and recommendation context.

Do not bundle them into “settings.” Each state can have different ownership and product behaviour.

## Identify the viewing context

Ask whether the session is personal, joint, guest, or a recurring group. A personal profile should reflect its owner. A shared profile needs an agreed rule.

The [separate-versus-shared profile framework](/blog/separate-profiles-or-one-shared-profile-a-decision-framework/) helps when conflicts recur because contexts are mixed.

## Check actual media availability

Norva can preserve language and subtitle preferences, but available tracks depend on the source and media. A preferred language absent from one version cannot be created by changing profiles.

Use an authorised reference item and record the tracks visible in the selected variant. W3C caption guidance explains what captions are intended to convey but does not certify track availability or quality.

## Create a personal rule and shared rule

For each preference, write:

- personal profile default;
- shared-profile default;
- per-item exception;
- fallback when unavailable;
- who decides during a joint session.

No rule is universally best. The purpose is to make ownership visible before playback.

Record who can make an in-session exception and whether the profile should return to its previous setting afterward. This keeps a one-night accommodation from silently becoming a permanent shared default.

## Handle favourites separately

A favourite is an intentional saved choice, not a language preference. On a personal profile, the owner decides. On a shared profile, agree before removal.

Do not use a shared favourites list to resolve disagreement about what should play now; use a temporary planning note.

## Test on one item

Confirm the active profile, item, and version. Select an available audio/subtitle combination, play a short dialogue section, pause, and record whether the choice meets the context.

The [per-profile language review](/blog/how-language-preferences-should-be-reviewed-per-profile/) provides a source-target test card.

## Communicate before changing

Explain which profile will change and whether the decision is permanent or only for the item. On a shared profile, get agreement before changing a saved preference.

The [complete household profile guide](/blog/the-complete-guide-to-household-media-profiles/) offers a profile charter for these rules.

## Common mistakes and limitations

Avoid declaring one preference “correct,” assuming every item offers it, changing another person's profile, and using favourites as a substitute for session planning.

Source tracks, labels, and version availability vary. This framework cannot add media options, guarantee preference persistence, or settle broader household disputes.

## Confirm the shared rule in practice

Run one joint and one individual session after the change. Verify profile label, language, subtitles, and favourite state before playback, then ask whether the rule was understandable without verbal correction.

## Frequently asked questions

### Should a shared profile use the majority preference?

Not automatically. Agree on a rule that fits the recurring group and permits per-item exceptions.

### Can profiles have different subtitle preferences?

Norva can preserve preferences, subject to current product behaviour and track availability. Verify each profile and item.

### What if no available option suits everyone?

Choose another authorised version or item, or pause the joint session. Do not pretend an absent track exists.
