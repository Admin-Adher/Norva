# How to Review Continue Watching Before a Shared Session

> **In short:** Agree on the group title before opening it, then confirm the active profile, exact episode or version, shared checkpoint, required language or subtitles, and intended screen. Decide which profile will own the new progress after the session. This prevents a group watch from overwriting assumptions or mixing personal Continue Watching state.

A shared session often fails at the handoff: everyone remembers “where we were,” but the screen opens another profile, episode, or version. A two-minute preflight is quicker than reconstructing the group’s place afterward.

| Subtitles or other access needs |  |
| Intended supported screen |  |
| Post-session progress owner |  |

Use only the minimum detail needed. Do not display private viewing notes, account identifiers, or credentials on a shared screen.

## Confirm the profile before browsing

Read the current profile label and verify it with the group. If profiles are not proven to separate resume state in the current build, run a harmless scope test outside the shared session. Norva’s current public plan information describes up to two profiles for Norva and up to five for Norva Family; those numbers describe profiles, not devices or concurrent sessions.

If the row already mixes viewers, use [the shared-profile clutter protocol](/blog/shared-profile-resume-clutter/) to identify personal, shared, and unresolved items without clearing anything.

## Verify exact media identity

Compare the title, series, season, episode, edition, duration, source context, language, and subtitles. DCMI metadata terms distinguish these identity dimensions. A shared poster is not enough to prove the version matches the group’s checkpoint.

W3C media accessibility requirements reinforce that subtitles, captions, audio description, and language tracks are functional needs. Confirm them before the group settles in. Their availability depends on the connected source and media.

## Rehearse the handoff

One person reads the card while another opens the detail page. Before playback, the group confirms:

1. the visible profile is correct;
2. the card opens the expected item;
3. the chosen version meets access needs;
4. the displayed progress is compatible with the remembered checkpoint;
5. everyone agrees whether to resume slightly before the landmark.

Do not seek repeatedly among versions. If the checkpoint is ambiguous, preserve the current state and use [the active-item priority method](/blog/prioritize-active-resume-items/) to choose another ready title for the session.

## Close the session deliberately

At the end, state a neutral checkpoint, exit normally, and inspect the row once. Confirm who owns the next resume action and whether it remains shared. If the session used a temporary household profile arrangement, switch only after the new state has been observed.

Use [the profile-switch review](/blog/review-row-after-profile-switch/) immediately afterward so the next viewer does not continue in the wrong context.

Norva can retain progress and preferences across supported devices under one account, according to its public features. Exact profile and synchronization behavior must be verified in the current product before publication.

## Original evidence: shared-session handoff card

Pilot the card during one non-sensitive session. Ask each participant to verify one field, then have a different person reconstruct the title, version, checkpoint, and owner after the session. Record missing fields only, not personal reactions.

The exercise tests the household handoff, not the product’s performance. It is successful when the next session can resume without guessing and no personal entry was changed unintentionally.

## Common mistakes and limitations

- Opening the first matching poster before checking profile and episode.
- Treating one person’s memory as the shared checkpoint.
- Ignoring language or subtitle needs until playback starts.
- Switching profiles immediately after exit without observing state.
- Recording sensitive comments in the handoff card.
- Assuming profile capacity implies a parental-control feature.

The process cannot make an unavailable version or access option appear.

## Frequently asked questions

### Which profile should own a shared session?

Choose one explicitly and use it consistently. The best choice depends on the household’s verified profile setup.

### What if viewers remember different checkpoints?

Resume slightly before the earliest agreed neutral landmark or choose another title until the state can be investigated.

### Should the group keep a permanent log?

Usually not. Retain only the minimum checkpoint while it is useful, then remove the private note.
