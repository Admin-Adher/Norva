# Restart or Reset the Router? Know the Difference

> **In short:** A restart stops and starts the router through a documented software or power procedure and commonly aims to preserve configuration. A factory reset returns the device toward a manufacturer-defined default state and can erase network names, passwords, provider settings, security rules, updates, and device mappings. Verify the exact model instructions first.

The terms vary across products. A button labeled reset may perform different actions depending on press length, so never infer behavior from the word alone.

## Identify ownership and role

Determine whether the device belongs to the household, provider, landlord, workplace, or managed service. Record whether it is a router, modem-router, mesh controller, access point, or network terminal.

Do not restart or reset equipment without authorization. It may carry calls, security systems, accessibility tools, or work connections.

## Preserve the symptom first

Record exact playback event, time, affected devices, local and external tests, indicator states, and configuration changes. A restart removes volatile evidence and can change routes, channels, leases, and associations.

[The home-versus-provider guide](/blog/home-network-or-internet-provider-find-the-boundary/) helps collect evidence before intervention.

## Understand a documented restart

Use the approved administration control or manufacturer/provider power sequence. Follow specified shutdown time and component order; combined access equipment may have dependencies. Do not unplug cables randomly.

A restart can clear temporary state and renegotiate connections, but improvement does not identify which state caused the symptom. Recurrence matters.

## Understand a factory reset

A factory reset can remove custom configuration and require complete setup. Recovery may need provider credentials, device enrollment, security settings, reserved addresses, guest networks, parental controls, and mesh pairing. Some data or updates may behave differently by product.

Only use the exact official model procedure after confirming backup and recovery. NIST IR 8425A describes security outcomes expected of consumer-grade routers, underscoring that configuration is part of network protection.

## Use an escalation ladder

Begin with non-disruptive checks: service status, cables, ventilation, affected path, another device, and router interface indicators. Then restart only the affected app or player when appropriate. Restart network equipment only when evidence or trusted support guidance justifies it.

[The home-network checklist](/blog/a-home-network-video-checklist/) orders these checks and preserves rollback.

## Verify after a restart

Wait for official status indicators, then test local association, name resolution, external access, critical devices, and the same authorised title. Record what recovered and how long it remained stable. Do not declare a permanent fix from one replay.

[The baseline guide](/blog/how-to-record-a-home-network-baseline/) provides comparable measurements.

## Prepare before any reset

Confirm configuration export, recovery method, administrative credentials, provider activation requirements, current supported firmware, secure network settings, device list, and a suitable maintenance window. Ensure another communication channel is available if internet access is lost.

If any item is unknown, contact the device owner or provider instead of proceeding.

## Avoid unsafe shortcuts

Do not hold an unidentified button, use a pin without the manual, install unofficial firmware, disable security, or publish configuration files. NIST SP 800-153 emphasizes secure wireless configuration; performance troubleshooting should not weaken it.

## Frequently asked questions

### Does unplugging a router count as a restart?

It may for some models, but use the manufacturer or provider's documented sequence and consider connected equipment.

### Will a restart erase the Wi-Fi password?

A normal documented restart is generally intended to preserve configuration, but verify the exact device instructions.

### When is a factory reset appropriate?

Only when authorized official guidance supports it and complete recovery, security configuration, credentials, and outage impact are understood.
