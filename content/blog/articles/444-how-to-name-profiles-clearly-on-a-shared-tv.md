# How to Name Profiles Clearly on a Shared TV

> **In short:** Use short, distinct, privacy-conscious labels that identify the viewing context from the normal seating position. Make the words different at the beginning, avoid email addresses or sensitive personal details, and treat avatars or colours as supporting clues. Test every name under real remote focus before adopting it.

On a TV, profile labels are viewed from farther away and selected with directional controls. Two long names that differ only at the end can look identical when truncated.

## Name the context, not the device

A profile carries progress, history, favourites, and preferences across supported devices. Naming one “Living Room TV” confuses location with viewing ownership.

Prefer a person, agreed nickname, or recurring group label that answers “whose context is this?” Use only information the household is comfortable displaying on a shared screen.

## Make beginnings distinct

If two labels begin with the same long word, truncation can hide the difference. Put the distinguishing information first.

Examples should be household-created, not copied as universal recommendations. The key test is whether each current label remains distinct at the visible length.

W3C headings-and-labels guidance supports descriptive labels, while consistent-identification guidance supports stable naming for repeated functions.

## Avoid sensitive data

Do not use:

- email addresses;
- dates of birth;
- account roles such as “owner password”;
- medical or accessibility details;
- source credentials;
- private household information;
- labels that embarrass or expose a viewer.

A profile name is visible organisation, not authentication.

## Use icons as secondary evidence

Avatars and colours can speed recognition, but similar images, focus overlays, display settings, or colour-vision differences can reduce clarity. Pair every icon with a meaningful text label.

Do not create two identical names and expect colour alone to separate them.

## Test with the remote

From the usual viewing seat:

1. open profile selection;
2. move focus one step at a time;
3. read the focused label;
4. compare neighbouring profiles;
5. confirm the intended profile;
6. return without starting media.

Norva's TV experience is designed around remote navigation. Actual focus styling and truncation must be observed on the supported TV.

Use [the active-profile preflight](/blog/how-to-confirm-the-active-profile-before-playback/) after naming.

## Include shared contexts explicitly

A recurring group profile should say which group owns it in language the household understands. Avoid vague labels such as “Other” when several people may use it differently.

The [complete household profile guide](/blog/the-complete-guide-to-household-media-profiles/) explains when a shared context is appropriate.

## Rename without losing context

Before changing a name, record the profile's current favourites or another non-sensitive identity clue. Change only the label through verified controls, then reopen and confirm that the profile context remains the intended one.

Do not create a replacement profile merely to change a name unless current product instructions require it. If the consequence is unclear, contact support.

The [profile-preserving screen-change workflow](/blog/how-to-change-screens-without-changing-household-profiles/) helps verify the same profile after device handoff.

## Common mistakes and limitations

Avoid device-based names, duplicate labels, differences only at the end, sensitive data, icons without text, and testing while standing close to the TV.

Display size, scaling, focus styling, language, and interface version affect recognition. This method improves household clarity but does not create account security.

## Test names from normal distance

Ask each household member to identify the intended profile from the usual seating position, using the normal remote and current text size. If two labels truncate or begin alike, revise the wording before relying on color or icon differences.

## Frequently asked questions

### Should profile names use full legal names?

No. Use only enough information for recognition. A privacy-conscious nickname or agreed role may be better.

### Can two profiles use the same avatar?

They can only if the text labels remain clearly distinct. Treat the avatar as supporting evidence.

### Should a guest profile be named “Guest”?

Do not assume a dedicated guest profile exists. Name only an available, intentionally assigned context under current product controls.
