# How to Run a TV Media Storage Audit

> **In short:** Identify the TV administrator, record system capacity, free space, and any application-storage categories the device actually exposes, then inventory known offline items only where the device, source, and rights support them. Set a reserve, remove understood items through current controls, and remeasure. Finally verify account, profile, remote focus, catalog, retained media, known playback, tracks, exit, and return from the normal seating distance.

TV storage is easy to ignore because the screen is shared and often optimized for viewing rather than administration. A careful audit must distinguish system files, application data, caches, legitimate offline media, and unrelated applications without inventing controls that the television does not provide.

## Assign the shared-screen owner

Record the adult responsible for device updates, application state, account sessions, source administration, and cleanup approval. These roles may be split. Do not expose administrator passwords on the TV or in the audit sheet.

Place the review in the [maintenance handbook](/blog/media-app-maintenance-audit-handbook/).

## Capture TV storage totals

Use the television's current system controls to record total capacity and free space, plus any categories it displays for the media application. Labels and accounting vary by supported TV environment. Preserve the wording rather than applying mobile or browser assumptions.

Choose a minimum reserve based on update needs and other household applications; no universal amount is promised.

## Inventory known local media

Offline access exists only when the TV device, connected source, and associated rights permit it. If the current interface shows legitimate offline items, record neutral codes, current need, and last verification. If no such option is documented, do not claim the TV supports it.

The [TV-first evaluation](/blog/norva-for-tv-first-households/) separates remote viewing fit from storage capability.

## Separate application categories

Record application size, cache, data, offline media, or other categories only when the TV system presents them. “Data” may include sign-in or preferences, so do not treat it as disposable. Read the exact device control and consult current support when consequences are unclear.

Avoid deleting directories, resetting the TV, or uninstalling as a routine first step.

## Review upcoming household needs

Check planned offline or travel-related use, update windows, and household viewing schedules. A cleanup just before a disconnected need can remove the only prepared item. The [offline cleanup cadence](/blog/offline-storage-cleanup-cadence/) turns storage pressure and real events into review triggers.

Record broad dates, not private travel details or viewing histories.

## Remove the least risky item

Prefer an understood offline item no longer needed, through the current documented interface. Change one item or category at a time. Then return to the system storage view and record the observed free-space change.

Do not promise that displayed item size and recovered space will match exactly.

## Verify remote navigation

From normal seating distance, use the household remote to return to Norva, confirm visible focus, select the intended profile, and reach the known item. Record any focus trap or unreadable storage-related prompt. The TV experience is designed for remote navigation, but the actual supported setup still requires testing.

If an update preceded the issue, run the [application smoke check](/blog/post-app-update-smoke-check/) or [system-update audit](/blog/post-system-update-playback-audit/) separately.

## Verify playback and tracks

Use media with confirmed source tracks. Start, pause, seek, inspect available audio and subtitle options, exit, and return. Check retained offline media only under valid supported conditions. A missing source track or expired right is not a storage accounting failure.

Record one progress or preference marker only when it existed in the baseline.

## Handle device transfer or reset

Before sale, repair, or household transfer, use the television manufacturer's account-removal and erasure guidance plus current Norva support. Deleting individual media or application cache is not a complete device privacy process.

Minimize screenshots because a TV can expose profile names and private catalog details.

## Common mistakes and limitations

- Applying mobile storage labels to a TV.
- Assuming TV offline support exists.
- Clearing all application data without understanding it.
- Auditing from beside the screen instead of normal distance.
- Ignoring shared-screen ownership.
- Treating item deletion as full device erasure.

## Frequently asked questions

### Does every supported TV expose the same storage controls?

No universal control set should be assumed. Use the current device interface and documentation.

### Should I clear cache whenever the TV feels slow?

Not automatically. Record the symptom, storage state, and update context, then choose the least disruptive relevant action.

### Can a TV storage audit prove offline support?

No. Offline access must be verified separately for the device, source, rights, and media item.
