# Run a First Offline Readiness Check Where Supported

> **In short:** Treat offline viewing as conditional. Confirm that the device, source, media, rights, storage, and current Norva surface support the workflow. Prepare one small authorized item, disconnect the device deliberately, launch Norva, test playback and controls, then reconnect and clean up. Do not assume that a visible item is available offline or that every title can be stored.

Offline readiness is a chain of dependencies, not one switch. A failure at any link can make a prepared trip unreliable. Begin with the [first playback verification](/blog/verify-first-playback-session/) while connected so source or format problems are not misclassified as offline problems.

## Confirm permission and eligibility

Verify that you own the source or are legally authorized to use it and that offline use is allowed for the selected media and context. Rights and source rules can differ by item. A Norva subscription does not grant rights to store third-party media.

Check the current official interface and support guidance for device and feature eligibility. Mark the test not applicable when the workflow is not offered; do not try to bypass a missing or disabled control.

## Choose a small test item

Select a familiar, non-sensitive item known to play on the same device while connected. Prefer a modest duration for the first check. Record the broad item type and version without exposing the full household catalog.

If language or captions matter, verify that the exact chosen version contains those tracks before preparation.

## Review device storage and privacy

Confirm enough free storage using the device's own settings and leave a reasonable margin for normal operation. Protect the device with current security controls and avoid shared or unmanaged storage.

Norva states that supported offline media is encrypted on the device and is not uploaded to Norva. This does not remove the need to secure the device, follow rights, limit retention, and delete unneeded local copies.

## Prepare while connected

Use the visible supported workflow to prepare the one test item. Keep the app in the expected state and record completion through an actual interface signal, not merely an animation. Do not disconnect until preparation is confirmed.

Record the time, device category, Norva surface, item shorthand, selected version, and broad network type. Exclude source addresses, tokens, and private notifications.

## Create a genuine offline condition

Disconnect Wi-Fi and mobile data, or use the device's supported offline mode. Confirm through device controls that no alternate network remains active. Do not rely on poor reception as proof of disconnection.

Launch or return to Norva using the normal user flow. Locate the prepared item without assuming that every catalog area remains available offline.

## Test launch and controls

Start the item, play for several minutes, pause, resume, and seek a short distance. Inspect the expected audio or subtitle track if relevant. Record startup, control, and track results separately.

Close the app normally, reopen it while still offline, and test the item once more. This checks more than a player that remained buffered from the connected session.

## Reconnect and clean up

Restore the network, allow the normal account state to reconcile, and inspect progress where supported. Remove the prepared item through the current interface when the offline need ends. Confirm that device storage changes as expected without claiming exact byte recovery from an estimate.

Use the [offline-needs evaluation](/blog/evaluate-offline-viewing-needs-norva/) for repeated trips, storage budgets, caption requirements, and device-loss scenarios. The [Norva onboarding journey](/blog/norva-onboarding-complete-journey/) places this conditional check alongside privacy and device review.

## Common offline-readiness mistakes

- Assuming catalog visibility means local availability.
- Testing an item that never played while connected.
- Disconnecting before preparation completes.
- Leaving an alternate network active.
- Testing only a still-buffered player session.
- Keeping local copies longer than necessary.

## Frequently asked questions

### Is offline viewing available for every item and device?

No. Availability depends on the device, source, media, rights, storage, and current supported behavior. Verify the exact case before relying on it.

### Does Norva upload an offline copy?

Norva's current official privacy information says supported offline media is encrypted on the device and not uploaded to Norva. This statement still requires human fact-checking before publication.

### Should I test immediately before travel?

Yes, but also test earlier enough to resolve problems. Repeat a short disconnected launch near departure because rights, sessions, storage, apps, and media can change.
