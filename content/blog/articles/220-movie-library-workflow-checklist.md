# A Workflow Checklist for a Well-Run Movie Library

> **In short:** Pass five gates: authorized source scope, work identity and relationships, version control, browsing and viewing readiness, and maintenance. Test recent intake plus one complex movie with multiple versions or relationships. Any failed check enters an exception queue with an owner and next action; it is not hidden by extra categories, merged records, or an optimistic availability label.

A healthy movie library makes uncertainty visible and routine choices easy. The checklist measures whether the workflow supports that outcome, not whether every field is filled.

## Gate 1: source and profile scope

- [ ] Every connected source is owned or authorized for the user.
- [ ] The active profile or account context is known.
- [ ] Current availability is distinguished from permanent identity.
- [ ] Disconnected or refreshing sources have a visible state.
- [ ] Device-specific persistence is documented.
- [ ] The library is not described as a supplied movie catalogue.

Norva organizes compatible connected sources; users bring sources they are authorized to access. Treat source status as an input to the library, not as the definition of the movie work.

## Gate 2: work identity and relationships

- [ ] Preferred title is stable.
- [ ] Alternate titles remain searchable.
- [ ] Release date and creator identity distinguish collisions.
- [ ] Remakes are separate related works.
- [ ] Franchise links have explicit types.
- [ ] Shorts, features, and components follow a written policy.
- [ ] Unknown relationships remain marked unknown.

The FIAF Moving Image Cataloguing Manual distinguishes works, variants, manifestations, and items. A personal library can use fewer layers, but it should not collapse work identity into one source item.

Use [the complete movie workflow guide](/blog/movie-library-workflow-guide/) to trace a failure back to intake, identity, relationships, browse readiness, viewing, or review.

## Gate 3: version control

- [ ] Substantive cuts remain distinguishable.
- [ ] Format-only differences are not mislabeled as new cuts.
- [ ] Runtime belongs to the selected version.
- [ ] Audio and subtitle tracks are verified per version.
- [ ] Current source availability is recorded separately.
- [ ] Grouping does not combine tracks from different versions.
- [ ] Progress resolves to the intended version.

Use [the alternate-cut decision dossier](/blog/choose-between-alternate-cuts/) for the representative complex movie.

Dublin Core relation and version properties reinforce the benefit of linking versions without erasing their separate descriptions.

## Gate 4: browse and viewing readiness

- [ ] Search finds a known title and alias.
- [ ] Important filters pass positive and negative controls.
- [ ] Empty results expose active state and recovery.
- [ ] Sorting changes order, not membership.
- [ ] Mobile drawers retain visible filter state.
- [ ] TV focus and Back routes are predictable.
- [ ] The selected movie, version, runtime, and tracks are checked before playback.
- [ ] Interrupted viewing has a recoverable point.

Run this gate on the actual supported device used for viewing, not only a desktop browser.

## Gate 5: personal state and maintenance

- [ ] Favorites still represent current intent.
- [ ] Ratings use a defined rubric.
- [ ] Temporary themes have owners and review dates.
- [ ] Weekend queues match real sessions.
- [ ] Completed franchise projects are softly archived.
- [ ] Old filters and search text do not remain hidden.
- [ ] Exceptions have owner, evidence, and next action.
- [ ] Review cadence is appropriate to change volume.

[The completed film-series archive workflow](/blog/archive-completed-film-series/) removes finished projects from active work without deleting context.

## Use the exception card

| Record or workflow | Failed gate | Evidence | User impact | Owner | Next action | Recheck date |

|  |  |  |  |  |  |  |

Prioritize failures that cause the wrong movie or version to play, hide valid records, lose progress, or misrepresent source scope. Cosmetic completeness comes later.

## Run representative tests

Choose:

- one recent movie;
- one title collision;
- one alternate-cut group;
- one remake or franchise relationship;
- one missing-metadata record;
- one interrupted viewing;
- one archived project.

Passing only simple records does not validate the workflow's difficult boundaries.

## Common mistakes and limitations

- Using field completion as the only health metric.
- Fixing identity problems with more categories.
- Hiding unknowns from filter results.
- Testing web while ignoring TV or mobile state.
- Merging versions to remove duplicates.
- Closing exceptions without a rerun.

The checklist detects workflow risks; it cannot supply missing source metadata or decide legal authorization for the user.

## Frequently asked questions

### Must every check pass for every movie?

Apply relevant checks. Unknown optional metadata can remain; identity, version, and source-scope failures need explicit handling.

### How often should the checklist run?

Run after major source or product changes and at a cadence suited to intake volume and exception growth.

### What is the best representative test movie?

Choose a legitimate complex record with multiple versions, relationships, tracks, or prior progress—not the cleanest title in the library.
