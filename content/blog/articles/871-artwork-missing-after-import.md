# Artwork Missing After Import? Separate Data From Display

> **In short:** First confirm the catalog card or media entry exists; missing artwork alone does not mean the item is missing. For a few samples, record visible text metadata, source and version labels, artwork at the authorized source, Norva artwork state, device and application version, network, filters, and timestamps. Compare another supported device or trusted network, avoid editing source metadata first, and send support redacted paired observations.

Artwork is one presentation element. A blank image, generic fallback, wrong image, and absent card are four different observations and should not share one conclusion.

## Define the artwork symptom

Record whether the image area is blank, generic, incorrect, low quality, intermittently visible, or absent with the entire card. Note which view shows it: category grid, search, detail page, episode list, or another screen. Do not call the media missing when title and entry controls remain visible.

The [missing-item checklist](/blog/expected-items-missing-after-sync/) covers absent entries rather than absent images.

## Freeze catalog context

Record account, profile, authorized source label, category, filters, grouping, device, application version, network, and time. Keep those values stable. A search result and grouped category card may present different image contexts without proving different underlying data.

## Select representative samples

Choose up to three privacy-safe entries: one missing image, one normal image from the same category, and one affected entry on another view. Assign sample codes instead of exporting titles. Record media type, year, season or episode, source label, version, and visible text metadata.

## Confirm entry and text metadata

For each sample, record whether the card exists, title appears, description appears, category is correct, and playback or detail controls are present where expected. These observations separate image presentation from broader metadata or entry absence.

If metadata appears but no usable entry exists, use the [metadata-without-entry guide](/blog/metadata-visible-media-entry-missing/).

## Check artwork at the authorized source

Through the source provider's official route, record whether an image is currently visible for the same sample and whether the source recently changed it. Do not copy private image addresses or assume Norva retrieves artwork through a particular undocumented mechanism.

Source artwork presence confirms one layer only.

## Compare device and network context

Check the same account, profile, sample, filters, and grouping on one other trusted supported device. Record both application versions and close timestamps. If practical, compare the affected device on another trusted network without disabling certificate, privacy, or security controls.

The [one-device catalog guide](/blog/one-device-shows-old-catalog/) helps preserve a full cross-screen matrix.

## Avoid premature metadata edits

Changing titles, image files, addresses, categories, or source records creates a new input and can hide the original problem. Do not repeatedly import, clear application data, reinstall, or remove the source before saving the baseline. If current support recommends a step, record it as a separate comparison.

## Separate timing from causation

Record source artwork change time, import or sync state, and each Norva observation. Use current official timing guidance only when published. An image that appears later establishes a timeline, not automatically the internal reason for the delay.

## Classify the evidence

Use source image absent, source image present but Norva image absent, one-view difference, one-device difference, network-specific observation, wrong image, entry or metadata issue, resolved after documented step, or unknown. Avoid claims about image caches, transformation, or storage unless official evidence supports them.

## Common mistakes and limitations

- Treating missing artwork as a missing media entry.
- Comparing different views, profiles, or source versions.
- Editing source metadata before preserving the original state.
- Calling the cause a cache problem without evidence.
- Disabling network security controls to load an image.
- Sharing complete catalogs or private image addresses.

## Frequently asked questions

### Does missing artwork mean the import failed?

No such conclusion follows from artwork alone. Record entry, text metadata, source image, device, network, and timing separately.

### Should I replace the source image immediately?

Not before capturing the baseline. A source edit changes the input and can make the original comparison impossible.

### What should I send Norva support?

Send redacted sample codes, both artwork states, visible metadata, source and version labels, views, devices, application versions, networks, and timestamps.
