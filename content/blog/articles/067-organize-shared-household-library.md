# How to Organize One Media Library for Several Viewers

> **In short:** Keep the media source shared, but give recurring viewers distinct profiles and one simple routine: select the profile before browsing, save favourites intentionally, verify grouped versions, leave playback normally, and switch profiles when the next person begins. This prevents most organisational confusion without duplicating the library.

A shared catalog does not need a complex filing system. It needs consistent viewer context. The catalog belongs to the connected source; profiles, favourites, progress, history, and preferences help people navigate that shared set in a personal way where supported.

## Start with recurring viewers, not devices

List the people who return regularly and need their own progress or preferences. A television, phone, and browser used by the same person do not automatically need three profiles. Conversely, two people sharing one television may benefit from two viewer contexts.

Use the [individual household profile guide](https://norva.tv/blog/individual-household-media-profiles/) to decide who needs separation. Norva’s current public plans offer up to 2 profiles on Norva and up to 5 on Norva Family; these are profile limits, not device or simultaneous-playback claims.

## Agree on five household rules

This compact operating agreement is the original evidence element for the article.

### Rule 1: Select before browsing

Choose the correct profile before opening Continue Watching, favourites, or recommendations. Correcting the profile after playback starts may already have attached activity to the wrong context.

### Rule 2: Save with intent

Add an item to favourites only under the viewer who wants it. If an item is genuinely shared, each interested viewer can save it in their own context where the product supports that behaviour.

### Rule 3: Check the version

When grouped versions exist, verify the exact entry before playback. Different versions can expose different languages, text tracks, or media characteristics.

### Rule 4: End sessions normally

Pause and leave playback through the normal controls. This gives the interface a clear stopping point and reduces confusion when another screen resumes later.

### Rule 5: Hand over explicitly

When another person takes control, switch profiles before they browse. A five-second handoff is easier than repairing mixed history or progress.

## Separate personal signals from shared facts

Some information describes the source; other information describes a viewer’s interaction.

| Shared source context | Viewer context |
| --- | --- |
| Catalog titles and versions | Current profile |
| Available audio and text tracks | Preferred language when available |
| Media metadata supplied by the source | Progress, history, and favourites where scoped to the profile |

A human reviewer should verify the current product’s exact per-profile scoping before publication. Norva publicly confirms profiles and synchronisation of progress, favourites, history, and preferences, but a draft should not assume every category behaves identically.

For a focused explanation, read [how separate profiles prevent mixed-up history](https://norva.tv/blog/separate-profiles-watch-history/).

## Make shared discovery intentional

Personal profiles can reduce accidental mixing, but households still need ways to choose together. Use a spoken shortlist, a temporary note, or a shared planning moment rather than turning one person’s favourites into a universal household queue.

When a recommendation appears, remember that Norva surfaces recommendations from the connected source. Do not present them as a global catalog or an unverified artificial-intelligence feature.

## Review the system monthly

Once a month—or whenever household membership changes—ask:

- Are all recurring viewers using the correct profile?
- Are old profiles still needed?
- Are profile names non-sensitive and recognisable?
- Do language preferences still fit the viewer?
- Are apparent sync problems actually profile mistakes?

Do not delete a profile casually if its history or progress matters. Verify the current product behaviour and available controls first.

The guide to [profiles, favourites, and history](https://norva.tv/blog/profiles-favorites-history/) offers a maintenance map for these categories.

## Common mistakes and limitations

- Creating profiles for screens instead of recurring viewers.
- Using one profile for everyone and expecting personal progress.
- Treating profile names as a place for sensitive data.
- Assuming a profile is an access-control boundary.
- Assuming Norva Family includes parental controls.
- Forgetting that track availability remains dependent on the source and exact version.

This workflow organises the software experience. It does not change who owns the source or which media rights apply.

## Frequently asked questions

### Should guests get a permanent profile?

Only if they return often enough to need separate progress or preferences and a profile slot is available. A one-time guest may not justify permanent clutter.

### Should a couple share one profile?

They can if their viewing, progress, favourites, and preferences are genuinely shared. Separate profiles are more useful when those contexts regularly diverge.

### Is a profile the same as a private account?

No. Do not assume profile switching creates an authentication or privacy boundary. Account security remains separate.

### How should we handle an item everyone watches?

Choose the profile representing the current viewer or agreed shared session, then maintain that convention consistently. Do not switch mid-session without recording the context.
