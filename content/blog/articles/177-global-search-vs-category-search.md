# Global Search or Category Search: Which Is Better?

> **In short:** Use global search first when you know a title, person, or distinctive clue but are unsure how the record is classified. Use category search when the category itself is a firm requirement and its membership is trustworthy. If a scoped search returns nothing, repeat the same query globally before concluding the item is absent.

Neither route is universally better. Global search favours coverage; category search reduces noise by adding context.

## Use the search-scope decision matrix

| Item clue certainty | Category confidence | Best first route | Why |
|---|---|---|---|
| High | Low | Global | avoids wrong classification assumption |
| High | High | Global, then confirm category | verifies item before scope |
| Low | High | Category browse/search | context reduces uncertainty |
| Low | Low | Global distinctive clue | learns stored wording and type |

Category confidence means you have evidence the record belongs there, not merely that the category sounds plausible.

## Start global for a known item

Global search is usually safer when you remember:

- a full or partial title;
- a creator or cast member;
- an original or translated title;
- an approximate year;
- a series but not episode placement.

It can reveal that the record is a film, series, episode, special, or grouped version. After identification, category can become a verification clue.

## Start in a category for discovery

Category scope is useful when the task is “find any available family film” or “browse documentaries from this source,” rather than locate one named work.

The category should be:

- understandable;
- populated through a known mapping;
- broad enough for the decision;
- visible as an active scope;
- easy to clear or broaden.

Use [the first-filter decision guide](/blog/choose-first-library-filter/) if year, source, availability, or another facet may be more reliable than category.

## Test category coverage

Choose a known control item that should belong to the category. Search or browse within the category and record whether it appears. Then run the same item globally.

| Control | Global result | Category result | Interpretation |
|---|---|---|---|
| Known member |  |  | category mapping/index works or fails |
| Known non-member |  |  | scope is inclusive or overly broad |
| Target |  |  | search versus classification hypothesis |

If the known member fails only in the category, investigate category membership before refining the target query.

## Understand scope inheritance

A category page may retain source, year, audio, subtitle, favourite, or availability limits. Record every active condition. A global search reached from that page may or may not inherit them, depending on the current interface.

The Library of Congress Advanced Search guide shows how search terms and limiters can combine, while its Basic Search guide warns that facets may exclude records lacking the filtered metadata. In any catalogue, scoped search should make inherited limits visible.

## Use the two-route confirmation

For a difficult known item:

1. run the global distinctive-title query;
2. record candidates and their category labels;
3. open the expected category;
4. run the same query or browse for the candidate;
5. compare result presence and metadata;
6. clear the category before trying another hypothesis.

If global succeeds and category fails, the likely issue is classification, scope, or indexing. If both fail, use [the zero-results diagnostic](/blog/diagnose-zero-search-results/).

## Choose search or filtering deliberately

Global search finds a known clue across the catalogue; filters define a set of acceptable attributes. Use [the search-versus-filters guide](/blog/choose-search-or-filters/) when your task changes from identifying one item to choosing among many.

Norva can organise compatible authorised sources, but category membership and searchable fields depend on source metadata and current product behaviour.

## Common mistakes and limitations

- Searching a likely category without testing its coverage.
- Treating category absence as catalogue absence.
- Forgetting inherited filters.
- Using global results without checking record type.
- Repeating the same query in several categories.
- Assuming categories are mutually exclusive.

Category metadata can be incomplete or overlapping. Global coverage can also vary by indexed fields, so use controls.

## Frequently asked questions

### Is category search faster on TV?

It can reduce typing when the category is visible and trustworthy. A wrong category can cost more navigation than a short global title fragment.

### Can an item belong to several categories?

Yes. Genre, language, source, and household categories can overlap. Do not infer uniqueness from one category result.

### Should global search ignore filters?

Behaviour depends on the interface. Inspect active-state indicators and test a known control rather than assuming.
