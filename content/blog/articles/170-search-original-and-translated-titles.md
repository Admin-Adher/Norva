# When to Search Both Original and Translated Titles

> **In short:** Search both forms when you watched the work in another language or territory, remember a translated release name, use a profile language different from the source, see another script on artwork, or know the source may store the original title. Run each verified form separately with the same filters, compare results, and confirm identity before treating one as a missing alias.

Trying both forms is useful when there is a reason for title variation. It is not a licence to invent translations for every failed query.

## Use the two-form decision card

Answer these questions:

| Signal | Yes/No | Evidence |
|---|---|---|
| Was the work released under another title in your territory? |  |  |
| Is the remembered title in a different language from the source? |  |  |
| Does artwork show another script or name? |  |  |
| Is a verified original title known? |  |  |
| Is a verified translated or localised title known? |  |  |
| Could the remembered form be a transliteration rather than translation? |  |  |
| Did a source or profile-language change occur? |  |  |

One strong signal justifies the two-form test. With no evidence, switch first to a partial title, person, year, or plot clue.

## Keep four roles distinct

- **Original:** title associated with the original release context.
- **Translated:** meaning expressed in another language.
- **Localised:** market-specific adaptation that may not be literal.
- **Transliterated:** original title represented in another writing system.

The W3C language-tag guidance shows why language, script, and region are separate dimensions. A Latin-script form may still represent the original language through transliteration.

## Run the matched two-form test

Prepare verified forms:

| Form | Role | Language/script | Territory | Source | Distinctive token |
|---|---|---|---|---|---|
|  |  |  |  |  |  |

Then:

1. clear unrelated filters;
2. search the original form or its distinctive token;
3. record result count and best candidates;
4. search the translated or localised form with identical filters;
5. try a verified transliteration separately when relevant;
6. compare which form finds the intended work;
7. verify with year, creators, synopsis, type, and source.

Holding filters constant ensures the title form—not a hidden category or year filter—explains the difference.

## Know when both forms add value

Search both when:

- a household uses several languages;
- a work circulated under a well-known local release title;
- the connected source changed;
- original-script and Latin-script forms coexist;
- remakes share one translated name;
- a profile-language change altered display wording;
- artwork and the card show different verified titles.

For a monolingual work with one stable title and no source variation, a second-language query adds little.

## Interpret the outcomes

- **Both find the same work:** alias coverage appears useful; verify whether the forms are visible or only searchable.
- **Only original finds it:** the translated form may be absent, unindexed, unverified, or edition-specific.
- **Only translated finds it:** the source may prioritise a localised display title or omit the original alias.
- **Forms find different works:** check remakes, adaptations, and identity before adding aliases.
- **Neither finds it:** follow [the zero-results diagnostic](/blog/diagnose-zero-search-results/).

Use [the alternate-title search workflow](/blog/search-alternate-media-titles/) for additional forms. If a verified variant is missing from metadata, review it through [the title-variant handling guide](/blog/handle-localized-title-variants/).

## Avoid false alias corrections

A translated title may belong to a remake, a regional compilation, or an episode rather than the work you seek. Confirm source identifiers and edition scope. Do not replace the primary title merely because one search form succeeded.

The Library of Congress Search Help documents exact-phrase and keyword behaviour for its service; it also demonstrates that matching depends on indexed record text. A personal catalogue cannot find a translation that neither the source nor supported metadata contains.

Norva can organise compatible authorised sources, but title variants and searchable fields depend on source data and current product behaviour.

## Common mistakes and limitations

- Translating a title yourself and treating it as official.
- Combining original and translated words in one query.
- Confusing transliteration with translation.
- Changing filters between the two tests.
- Adding an alias before confirming work identity.
- Assuming a profile language guarantees translated metadata.

Some sources expose only one title form. Preserve a verified external clue in your search notes even when no supported alias field exists.

## Frequently asked questions

### Which form should I try first?

Start with the form you remember most confidently. If source or territory clues suggest a mismatch, run the paired form immediately afterwards.

### Should I translate only the distinctive word?

Only use a verified title form or documented alias. A literal word translation may not match the official localised title.

### Can both titles be displayed?

That depends on the interface and title policy. One can be primary while another remains in details or search aliases where supported.
