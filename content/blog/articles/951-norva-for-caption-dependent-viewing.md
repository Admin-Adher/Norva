# How to Evaluate Norva When Captions Are Essential

> **In short:** Use media whose caption or subtitle tracks you have confirmed at the source, then test every required supported screen. Check discovery, selection, visible text, timing, readability at normal distance, and preference behavior where available. Record each result separately. If an essential track is absent, unusable, or unreliable in the real viewing context, treat that as a stop condition rather than offsetting it with unrelated strengths.

Caption-dependent viewing turns a general product trial into an accessibility decision. Norva can retain language and subtitle preferences, but available tracks still depend on the connected source and the media. A useful evaluation must therefore separate source availability from how the required text track appears on each screen.

## Define the non-negotiable outcome

Write the requirement in observable language: “The known English text track must be selectable and readable from the sofa on the supported TV,” for example. Avoid vague goals such as “good accessibility.” State which screens, languages, programs, and household viewers matter.

Use the broader [Norva evaluation framework](/blog/norva-evaluation-framework/) for other requirements, but do not let its total score overrule a caption stop condition.

## Build a known-media sample

Choose three to five items from a compatible source you own or are authorized to use. Independently confirm which text tracks the source supplies. Include ordinary dialogue, rapid speech, speaker changes, and a scene with background detail. Use neutral item identifiers in notes rather than copying a private catalog.

If a badge and track list disagree, follow the [subtitle-track verification path](/blog/subtitle-badge-disagrees-track-list/) before judging the player.

## Separate presence from usability

For every item, record five different observations:

1. The required track exists at the source.
2. The track appears in the current selection controls.
3. Selecting it produces visible text.
4. Text timing remains useful during the sample.
5. Text remains readable in the intended environment.

A single “captions work” mark hides which layer failed. It also makes a support report difficult to reproduce.

## Test each required screen

Norva is available on Web, mobile, and TV on supported devices, but a pass on one screen does not prove another. Use the same account, profile, source, media item, scene, and track. Record application or browser surface, version, input method, viewing distance, display conditions, and timestamp.

For a multilingual household, combine this method with the [language evaluation checklist](/blog/norva-for-multilingual-households/) without assuming every item contains the same tracks.

## Judge readability without inventing controls

Check line visibility, contrast against bright and dark scenes, obstruction by player controls, comfortable reading distance, and whether long lines remain understandable. Use only settings that are actually visible and documented in the current product or device environment. Do not claim Norva offers a styling control that has not been verified.

Ask the person who depends on captions to judge the real viewing context. A technically visible track may still fail their requirement.

## Check timing with repeatable cues

Pick a distinctive spoken phrase and record the approximate scene position. Observe whether text appears early, late, disappears too quickly, or drifts during the short sample. Repeat once after a clean exit. This is a functional observation, not a laboratory timing measurement.

If the source track itself is mistimed, record that boundary rather than attributing the result automatically to Norva.

## Test preference behavior carefully

Where a subtitle or language preference is available, select it, leave through the normal flow, return to the same item, and inspect the actual selected track. Then try another known item and, if cross-screen use matters, one other supported device.

Preferences cannot create missing tracks. The later [subtitle preference audit](/blog/subtitle-accessibility-preference-audit/) provides a recurring cross-device review.

## Set pass, limited, and stop rules

Use “pass” only when the required track is present, selectable, visible, timed usefully, and readable on the required screen. Use “limited” for an acceptable but documented constraint. Use “stop” for a failed non-negotiable. Mark “unknown” when the source baseline or support status is uncertain.

Do not average a TV failure with a mobile success if TV is the essential household screen.

## Common mistakes and limitations

- Testing media without confirming its source tracks.
- Treating subtitles and every caption use case as identical.
- Testing only close to the screen instead of normal distance.
- Assuming preference retention guarantees track availability.
- Scoring design polish above an essential access failure.
- Publishing screenshots that reveal private titles or account details.

## Frequently asked questions

### Can Norva add captions that the source does not provide?

Do not assume so. Available languages and text tracks depend on the source and media; verify the required track before evaluating its presentation.

### Is one successful item enough?

No. Use a small varied sample because track presence, timing, and formatting can differ between items.

### Who should decide whether readability passes?

The person who relies on the text should evaluate it in the actual viewing environment. A reviewer can document the result but should not replace that judgment.
