# How to Organize Categories After Your First Import

> **In short:** Let the first import finish, preserve a record of the original structure, and observe which categories support real browsing tasks. Where Norva’s current controls permit, prioritise clear groups and hide redundant ones one change at a time. Keep an “All” route and verify that known items remain findable.

**Draft verification notice:** exact category controls and screenshots must be reproduced in the current product before publication.

## Do not reorganise during the first load

An incomplete import can look like a poor taxonomy. Wait for a stable catalogue and verify one known item before changing categories.

Capture or privately record:

- top-level sections;
- category names;
- approximate purpose, not fabricated item counts;
- obvious overlaps;
- one reference item’s location.

Do not expose private source information in screenshots.

The general principles appear in [How Categories Turn a Large Collection Into a Usable Library](/blog/media-library-categories/).

## Step 1: map real browsing tasks

Write three recurring tasks, such as finding a known movie, continuing a series, or browsing a live programme group.

For each task, identify the first category you expect to open. If several labels seem equally plausible, the structure may be ambiguous.

Categories should support decisions. Search, filters, favourites, and Continue Watching have different roles.

## Step 2: classify imported groups

Label each group:

- **Essential:** supports a recurring task.
- **Useful:** provides a distinct optional route.
- **Overlapping:** substantially duplicates another group.
- **Unclear:** its label does not predict contents.
- **Empty or unavailable:** currently provides no usable route.
- **Protected:** should not be changed until its role is understood.

These are editorial labels for your audit, not Norva product states.

## Step 3: identify the controlling layer

Some categories come directly from the source. The player may be able to display, prioritise, or hide them, but do not assume it can permanently rewrite the source taxonomy.

Before changing anything, ask:

- Is this label source-supplied?
- Does Norva currently expose a reversible control?
- Will the change affect one profile or the account?
- Does it apply on all supported devices?
- Can an item still be found through search or All?

If the answer is unknown, preserve the baseline and check support documentation.

## Step 4: change one thing

Where a documented control permits, make one reversible change: hide one redundant category, adjust one visible ordering, or rename one local label only if supported.

Return to the library and verify:

1. the category change is visible;
2. the reference item remains findable;
3. search still reaches it;
4. the All view remains available;
5. no unrelated section changed.

Do not bulk-hide groups simply to make the page shorter.

## Step 5: compare supported screens

A TV may present categories as remote-friendly rows while web or mobile uses a different layout. The category’s meaning should remain understandable even when geometry differs.

Check one other supported screen using the same account. Record whether the change appears, remains device-local, or is not supported.

The broader [category system for movies, series, and live programmes](/blog/category-system-movies-series-live/) helps separate top-level media types from narrow browsing attributes.

## A category decision log

| Category | User task | Distinct? | Action | Verification |
| --- | --- | --- | --- | --- |
| Example group | Browse a genre | Yes | Keep | Reference item found |
| Similar group | Same task | No | Review | Not changed yet |
| Unclear label | Unknown | Unknown | Investigate | Source checked |

Use real labels privately. The public article should use anonymised examples unless permission exists.

## Prevent over-organisation

A category for every year, language, rating, or temporary interest can create more navigation than it removes. Filters are often better for attributes; favourites are better for personal intent.

The article [How to Organize a Large Movie Collection](/blog/organize-large-movie-collection/) provides a wider workflow.

## Limitations

Source changes can restore, remove, or rename groups. Missing metadata can make a category incomplete. Norva’s current public baseline confirms catalogue organisation but does not verify every edit control used in this draft.

Do not claim a before-and-after improvement until an identified reviewer documents it.

## Frequently asked questions

### How many categories should I keep?

There is no universal number. Keep groups that support distinct recurring tasks and remain understandable.

### Should I delete empty categories?

Prefer reversible controls until you know whether the source may repopulate them. Verify current Norva capabilities.

### Can categories differ by device?

Presentation can differ. Whether category preferences sync must be checked in the current product.
