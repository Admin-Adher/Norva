# How to Find Media With Only Part of the Title

> **In short:** List the words you remember and rank them by certainty and distinctiveness. Search the rarest reliably spelled word first. If results are broad, add one neighbouring word or a reliable year. If there are no results, remove articles, punctuation, edition wording, and uncertain spelling one at a time, then try a verified original or localised title form.

Partial memory is often enough because a distinctive word can narrow a personal catalogue sharply. The challenge is choosing a fragment that the stored metadata is likely to contain.

## Build the partial-title information ladder

Write each remembered token in a row:

| Token | Certain? | Distinctive? | Spelling confidence | Could be article/edition word? | First-query rank |
|---|---|---|---|---|---|
|  |  |  |  |  |  |

Rank highly when a word is unusual, confidently spelled, and central to the title. Rank lower when it is a common word, an article, a guessed plural, or wording that may belong to an edition rather than the work.

For “The Something of the Moon,” a reliable unusual word is usually better than “the,” “of,” or an uncertain first word.

## Run the one-token baseline

Enter only the highest-ranked word. Record whether:

- no result appears;
- a manageable candidate set appears;
- many irrelevant results appear;
- a likely title appears under a different full name;
- only episodes, people, or categories match.

The baseline reveals how the interface matches a fragment. Do not assume prefix, substring, fuzzy, or whole-word behaviour without observing it.

## Narrow by adding one reliable clue

When results are broad, add one element:

1. a second remembered title word;
2. a reliable release year;
3. film versus series type;
4. a creator or cast name where supported;
5. series context for an episode.

Inspect results after each addition. [Exact-versus-broad search](/blog/exact-vs-broad-media-search/) explains why adding a constraint can improve precision yet remove records with incomplete metadata.

## Broaden by removing assumptions

When the baseline returns nothing:

- remove initial articles;
- remove punctuation and symbols;
- try singular and plural only when uncertain;
- test digits and written numbers separately;
- omit edition labels such as “extended” until identity is found;
- try one stable word from the subtitle rather than the main title;
- try a verified translated, original, or transliterated form;
- correct one likely spelling error.

The Library of Congress Search Help documents that exact phrases require exact term order in its system, while its catalogue guidance also supports broader “contains” behaviour. That contrast illustrates why a partial query should begin broad unless the interface’s exact syntax is known.

## Keep a minimal query diary

| Attempt | Query | One change | Results | Interpretation |
|---|---|---|---:|---|
|  |  |  |  |  |

If changing punctuation produces the same results, stop repeating punctuation variants. If removing one guessed word reveals the title, mark that word unreliable rather than adding several new constraints.

## Verify candidates with independent clues

A matching word is not proof. Compare year, creator, synopsis, type, series position, runtime, and source. Similar titles and remakes often share the fragment you remember.

When several candidates remain, add a disambiguating field rather than an increasingly long title guess. If none can be identified, save the candidate list and gather a person, plot, or approximate-time clue.

## Separate search failure from catalogue absence

Run a known control title from the same source. Clear active filters and confirm the expected source is connected and refreshed. If the control works but the target appears only under an unexpected title, the issue may be metadata rather than search.

Use [the typo-recovery sequence](/blog/recover-from-media-search-typos/) for uncertain spelling and [the zero-results diagnostic](/blog/diagnose-zero-search-results/) when every sensible fragment fails.

Norva can search and organise compatible authorised sources, but available records and searchable fields depend on those sources and current product behaviour.

## Common mistakes and limitations

- Starting with the longest remembered phrase.
- Using common articles as the only query.
- Adding a guessed year too early.
- Changing several words between attempts.
- Selecting a result from artwork alone.
- Assuming partial matching behaves identically on every interface.

A partial title cannot retrieve a record whose stored names are all different or whose metadata is absent. Alternate-title and person clues then become necessary.

## Frequently asked questions

### How many letters should I type?

There is no universal minimum. Use enough of a reliable distinctive word to produce useful candidates, then observe the interface’s matching behaviour.

### Should I search the subtitle after a colon?

Yes, if you remember a distinctive word from it. Some records index subtitles while others may not, so compare with a main-title or person clue.

### Does capitalisation matter?

Many search systems ignore case, including the Library of Congress search described in its help page, but test the actual media interface rather than assuming.
