# A Fast Triage for an Overlong Continue Watching Row

> **In short:** Do not try to perfect a long row in one session. Capture a baseline, review one screenful, and assign each entry to Act, Preserve, Investigate, or Confirmed Sample. Surface the next genuine watches first, test only one safe cleanup action, and schedule uncertain items for a later evidence-based review.

A fast triage should restore decision clarity, not maximise removals. Ten focused minutes can make the row useful again while protecting entries whose profile, version, or intention is still unknown.

## Prepare a four-lane board

| Lane | Qualification | Immediate action |
|---|---|---|
| Act | Real next-session intent | Add to the active window |
| Preserve | Deliberate pause or another viewer’s item | Leave unchanged |
| Investigate | Identity, profile, version, or state unclear | Record minimum question |
| Confirmed Sample | Preview is finished and scope is known | Eligible for one tested cleanup |

Use a private note, not a public household display. Record only title identity, lane, and missing field. Do not copy account details or sensitive viewing context.

## Capture the starting state

Before touching controls, note the date, active account and profile, device category, and first visible entries. A cropped screenshot can help if it does not reveal unrelated history. This baseline makes it possible to verify that a change affected the intended card.

Set a ten-minute timer as an operational boundary, not a performance claim. Stop earlier if uncertainty rises or current controls are unfamiliar.

## Scan without opening every card

Classify from visible evidence first. Ask:

- Do I know who owns this session?
- Can I identify the exact title, episode, and version?
- Is there a real return intention?
- Does language, subtitle, device, or source availability block the next action?
- Was this definitely a finished sample?

Unknown answers go to Investigate. Opening every item can create navigation overhead or accidental progress. DCMI distinctions for identifier, relation, format, and language show which identity fields matter most.

Move clearly actionable titles through [the active-item priority ladder](/blog/prioritize-active-resume-items/). Do not let unresolved entries compete with tonight’s decision.

## Process one safe candidate

Choose a non-sensitive Confirmed Sample with verified profile and version. Apply the available row action only if its effect is clear in the current interface. Reopen the row and compare with the baseline.

Check that the intended card changed. Do not assume the same action affects history, favorites, or another profile. If the outcome is surprising, stop and preserve evidence. Use [the sampled-title triage](/blog/clear-sampled-titles-from-resume-list/) for a full confidence check.

Avoid bulk operations during a first pass. Speed comes from postponing uncertain decisions, not from lowering the evidence standard.

## End with a useful active window

At the timer, select the small set of Act items that represent real near-term sessions. Preserve deliberate pauses. Give each Investigate item one specific question, such as “Which profile owns this?” or “Are these two versions distinct?”

Schedule the unresolved lane for [the weekly resume-row review](/blog/weekly-resume-row-review/). A triage succeeds when the next watch is clear and no uncertain state was destroyed, even if the row remains long.

## Original evidence: ten-minute triage board

Apply the four lanes to one screenful and record three outputs: number confidently classified, number unresolved, and the missing evidence categories. Do not report these numbers as a product benchmark; they describe only your review session.

Repeat a week later on a different screenful. If Investigate keeps growing, address the dominant cause—profile ambiguity, version identity, or forgotten intent—before doing more cleanup.

## Common mistakes and limitations

- Treating the timer as a reason to guess.
- Opening every entry before classification.
- Equating old with abandoned or short with sampled.
- Running a bulk clear without a one-item test.
- Ignoring other viewers on a shared profile.
- Measuring success only by row length.

The process cannot repair incorrect source metadata or create missing media access. It only improves the decision layer around visible state.

## Frequently asked questions

### What if almost every item is unresolved?

Stop the quick cleanup. Identify the shared missing field—often profile ownership or version identity—and solve that first.

### Must the triage take exactly ten minutes?

No. The time box prevents an exhausting bulk session; shorten or extend it cautiously without compromising evidence.

### Should completed items go directly to Confirmed Sample?

No. Completed and sampled are different states. Verify completion separately before deciding whether a card should change.
