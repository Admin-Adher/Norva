# Create Your First Household Profile With a Clear Purpose

> **In short:** Define one recurring viewer or shared viewing context, assign an adult account administrator, choose a neutral name, and create only that first profile through the current Norva interface. Record language and subtitle preferences where available, verify a small catalog and progress sample, and set a household handoff rule. Profile capacity does not state device limits, simultaneous playback, source rights, or parental controls.

A profile is most useful when everyone can explain what it is for. Starting with a clear purpose reduces mixed history, favorites, and preferences, while making later troubleshooting easier. It does not create media access or replace the authorization rules of the connected source.

## Define one recurring context

Write a one-line purpose before opening settings. The context might be one adult viewer, a shared household viewing context, or a recurring language preference. Avoid creating profiles for temporary guests, rooms, or every device unless a real recurring need exists.

The [complete onboarding journey](/blog/norva-onboarding-complete-journey/) places profile creation after account security and before broad device expansion.

## Assign an administrator

Name the person responsible for changes, deletion, source scope, and handoff. The owner should understand that the account and connected source are separate authorization boundaries. A profile label does not grant legal access to a source or transfer account ownership.

On a shared screen, agree who may edit the profile and who should ask the administrator first.

## Choose the right plan by recurring need

The Norva plan includes up to two profiles; Norva Family includes up to five. Use the number of stable viewing contexts, not the number of screens, as the decision input. The [profile-needs plan guide](/blog/choose-plan-by-profile-needs/) and [Norva versus Norva Family framework](/blog/norva-vs-norva-family-decision-framework/) provide structured comparisons.

Consult the current pricing page for the prices and conditions now in effect. Do not turn profile capacity into a promise about devices or simultaneous streams.

## Select a privacy-safe name

Use a short, neutral label that makes sense inside the household without revealing a full name, email address, birth date, location, health condition, religion, or source credential. “Shared Evening” communicates purpose more safely than a complete identity.

The [profile naming guide](/blog/name-norva-profiles-clearly/) contains a dedicated naming matrix.

## Create the profile through current controls

Follow the current Norva interface or support instructions because labels and screen order can change. Record the date, plan context, administrator, and purpose, but do not copy private account or source details into the worksheet.

If the expected profile control is absent, confirm account state and current plan information before repeating actions or creating a duplicate.

## Set preferences only where available

Norva can retain language and subtitle preferences, but the actual languages and subtitles depend on the source and media. Record the desired default and one sample item that exposes the relevant option. Mark unavailable rather than treating a missing source track as a profile failure.

Do not claim that the profile filters all unsuitable content. Norva Family describes profile capacity and should not be presented as a parental-control product.

## Verify the profile with a small sample

Use one source you are authorized to use and two or three neutral catalog samples. Check that the intended profile is active, then observe catalog visibility, one favorite, one progress state, and one available preference. Avoid using private titles or a complete watch history in evidence.

If you later change a source, preserve a [progress baseline](/blog/progress-baseline-after-source-change/) before judging whether the profile lost state.

## Test one shared-screen handoff

On a supported shared screen, practice switching away from and back to the profile using the current controls. Confirm the visible label and intended context before playback. Record only whether the handoff was clear, not who watched a particular item.

This small rehearsal is especially useful when personal and shared profiles coexist.

## Set a review trigger

Review the profile when its purpose changes, an administrator changes, the household adds recurring viewing contexts, or the current plan no longer fits. Delete or rename obsolete profiles only after recording what should happen to the household workflow; do not rush because of a temporary mismatch.

## Common profile mistakes

- Creating a profile for every device instead of every recurring context.
- Using a full identity or sensitive trait as the name.
- Assuming a profile grants source access.
- Treating profile capacity as a playback or device limit.
- Promising unavailable languages, subtitles, or parental controls.
- Adding several profiles before verifying the first.

## Frequently asked questions

### How many profiles do current Norva plans include?

Norva includes up to two profiles, while Norva Family includes up to five. Check the current pricing page before making a purchase decision.

### Is a profile the same as a device?

No. A profile represents a viewing context. Published profile capacity should not be converted into a device or simultaneous-playback claim.

### Does a household profile act as a parental control?

Do not assume that. Norva Family describes profile capacity; use only controls currently documented in the official product and support material.
