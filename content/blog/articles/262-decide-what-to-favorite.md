# What Deserves a Place in Your Favorites?

> **In short:** A title belongs in favorites when you can name the future action it should enable, identify the exact entity, expect to retrieve it again, and believe the reason will outlast tonight’s decision. Add an exit or review trigger. Temporary curiosity belongs in a temporary shortlist; immediate viewing intent belongs in next-up.

The favorite button is fast, but admission should still answer a purpose. Without one, saving becomes a reflex and the list turns into a second search result page.

## Use the five-question admission card

| Question | Answer |
|---|---|
| 1. What will I do with this later? |  |
| 2. Which exact entity or version is it? |  |
| 3. When will I realistically retrieve it? |  |
| 4. Will the reason remain useful beyond this session? |  |
| 5. What event triggers review or removal? |  |

If questions one or two have no answer, pause before saving.

## Question 1: name a future action

Strong actions include:

- rewatch a durable favorite;
- follow a series over time;
- compare a particular version;
- recommend the title to a named audience;
- preserve a reference for research or discussion;
- use it as a discovery seed later.

“Maybe” is not wrong, but it belongs in a bounded trial or tonight shortlist. Use [the temporary tonight workflow](/blog/create-tonight-shortlist/) when the decision expires soon.

## Question 2: identify the exact entity

Record work title, media type, release context, and series/episode hierarchy where relevant. If the future action depends on audio, subtitles, edit, or source version, preserve those attributes too.

DCMI metadata terms distinguish identifier, title, type, relation, format, and language. That separation keeps a favorite attached to what you actually meant to retrieve.

## Question 3: set a realistic retrieval horizon

Choose a useful horizon:

- tonight;
- this week;
- after finishing another title;
- next season;
- long-term reference.

If retrieval is near-term and viewing is already intended, [separate next-up from long-term favorites](/blog/long-term-favorites-vs-next-up/). A favorite can coexist with next-up only when it serves a distinct durable purpose.

## Question 4: test durability

Ask whether the reason depends on a passing context. A topical event, visiting guest, or temporary mood may deserve a themed shortlist with an expiry. A title that repeatedly informs choices or remains personally meaningful may justify long-term placement.

Durability concerns the intent, not whether media access is permanent. A favorite can remain meaningful when the current source changes.

## Question 5: define an exit trigger

Examples:

- after watching and deciding whether to rewatch;
- after sending the recommendation;
- at the next quarterly audit;
- when a comparison project ends;
- when the item no longer supports discovery.

An exit trigger does not force removal. It forces a deliberate review.

## Score by evidence, not taste

Use pass, uncertain, or fail for each question. Four or five passes support a long-term favorite. Two or three suggest a temporary or specialized list. Zero or one means there is not yet a useful save action. This is a workflow heuristic, not a universal judgment of quality.

## Test the saved result

After admission:

1. save the item;
2. leave its page;
3. locate it from the intended list;
4. confirm the right entity opens;
5. check scope if another viewer or device matters;
6. record the review trigger.

W3C label guidance supports controls whose name and state are clear. A favorite action should communicate whether it adds or removes the item.

Norva may retain favorites across supported devices under the same account. Confirm exact synchronization and profile behavior in the current build before relying on it.

## Reassess after watching

Completion does not answer whether the title remains useful. Apply [the post-watch favorite review](/blog/review-favorites-after-watching/) using the original future action.

## Common mistakes and limitations

- Saving because the card is visually prominent.
- Using “maybe” as permanent intent.
- Ignoring entity and version identity.
- Mixing immediate viewing with long-term preference.
- Having no review trigger.
- Treating the heuristic as a quality score.

## Frequently asked questions

### Can an unwatched title be a true favorite?

It can be saved for durable follow-up, but label the intent honestly. Preference based on prior viewing is different from interest.

### Should recommendations from friends be favorited?

Only if favorites is the right retrieval destination. A temporary candidate list may be clearer.

### What if I cannot name an exit trigger?

Use a scheduled audit date. If the reason remains unclear then, remove or reclassify the item.
