# How to Plan for a Media Library That Keeps Growing

> **In short:** Plan the intake process rather than guessing the library's final size. Define which new items belong, the minimum metadata they need, how sources and versions are identified, when active material moves to archive, and which observable thresholds trigger capacity or structure reviews. Assign an owner and sample new arrivals regularly.

A collection that works today can become inconsistent when new items bypass its rules. Growth planning makes every arrival a small controlled decision instead of waiting for another large cleanup.

## Create an intake gate

Before a new item or source enters the organised view, check:

- it is within the current collection scope;
- the source is compatible and authorised;
- the item is not an unresolved duplicate;
- required title, type, source, and language fields are available;
- the destination category has a defined meaning;
- the person responsible is known;
- any exception enters a review queue.

The gate should be quick enough to use. If it takes longer than the organisation saves, reduce the required fields.

## Preserve source and version context

Growth makes ambiguous duplicates more common. Keep enough context to distinguish:

- source;
- work versus exact version;
- episode or part relationship;
- language or subtitle options;
- availability state;
- date of last verification.

Do not encode every fact into a title. Use stable descriptive fields where the source and organiser support them. NARA's naming guidance recommends consistent, human-readable, machine-friendly patterns, but a media organiser may rely more on metadata than filenames. Apply only the principles relevant to your system.

Set the minimum rules through [the pre-import metadata guide](/blog/plan-metadata-rules-before-import/).

## Use triggers instead of distant forecasts

Avoid claiming that the library will reach a precise size without evidence. Define observable triggers:

- available storage falls below a household-set reserve;
- uncategorised queue exceeds a manageable batch;
- one source changes structure;
- duplicate-review backlog persists across two cycles;
- search tasks fail for new arrivals;
- active browsing becomes crowded;
- maintenance exceeds its agreed time budget.

Each trigger needs an owner, next action, and review date.

## Separate active use from retention

Not every retained item needs to stay in the busiest browsing path. Define when an item becomes an archive candidate and how it can return to active use. Archive status should be visible and reversible.

Use [the active-versus-archive framework](/blog/separate-active-and-archive-collections/) to avoid using deletion as the only response to growth.

## Build maintenance into the plan

Choose a cadence for:

- intake review;
- duplicate review;
- category health;
- source status;
- archive candidates;
- storage and backup status;
- retrieval testing.

The [maintenance cadence guide](/blog/design-library-maintenance-cadence/) turns these into daily, monthly, seasonal, and event-driven checks. NDSA preservation guidance treats inventory, storage, integrity, metadata, and file formats as connected areas; a household plan can borrow that layered view without copying institutional complexity.

## Prepare for source change

A growing collection may depend on sources that rename categories, change availability, or stop being used. Record what each source contributes and how much of the library depends on it. Preserve a non-sensitive source inventory and decision history.

Use [the source change resilience plan](/blog/plan-for-media-source-changes/) before a known transition.

## Common mistakes and limitations

- Predicting a final collection size as if growth will stop.
- Accepting new items without source context.
- Making every field mandatory.
- Using deletion as the first capacity response.
- Letting exceptions bypass the intake gate indefinitely.
- Creating categories for one new item.
- Waiting for a crisis before reviewing source dependencies.

Source-provided catalogue data and rights can change. Norva can organise a compatible authorised source but cannot guarantee permanent availability or complete metadata.

## Frequently asked questions

### Should I leave empty categories for future growth?

Only when they represent an approved near-term structure. Otherwise, document the planned category outside the live library until it has a defined use.

### How often should intake be reviewed?

Match the cadence to actual arrivals. Frequent additions may need a weekly queue; occasional additions may fit a monthly or event-driven review.

### What if growth comes from a new source?

Inventory the source first, sample its metadata, identify overlap, and test it inside the existing scope before combining the full view.
