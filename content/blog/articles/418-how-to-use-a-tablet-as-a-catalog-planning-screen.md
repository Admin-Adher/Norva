# How to Use a Tablet as a Catalog Planning Screen

> **In short:** Give the tablet a planning-only role: define the viewing occasion, browse under the intended profile, compare a small set of candidates using visible metadata, and record a next action without starting playback. The tablet can organise decisions around your connected source; do not assume it can edit source metadata or control another screen.

Planning is different from passive browsing. It answers a concrete question such as “What fits tonight?”, “Which episode is next?”, or “Which version has the needed language track?” A defined occasion keeps the session from becoming another endless scroll.

## Write the viewing brief

Use one sentence:

“Choose ___ for ___ people, on ___ screen, during ___ time window, with ___ audio or subtitle need.”

Leave fields unknown when they are not decided. Do not invent a hard duration requirement from an incomplete metadata label.

The [complete tablet viewing workflow](/blog/the-complete-guide-to-tablet-viewing-workflows/) helps prepare the profile, network, posture, and close-session state around this planning task.

## 1. Confirm the profile and source context

Open the intended Norva profile and identify the relevant library section. Norva is a software media player and organiser; it does not include a media catalogue. The user connects a compatible media source they own or are authorised to use.

Profile choice matters because favourites, history, progress, and preferences can follow the same account across supported devices.

## 2. Build a broad candidate set

Use category, year, availability, or other visible filters that match the brief. Apply one filter at a time and confirm its active state. Select a small set of candidates that clearly differ in a useful way.

Do not start playback to “save” a candidate. Use a favourite only when that action genuinely belongs to the profile and current interface.

## 3. Compare decision fields

For each candidate, record:

- full title and year, or series, season, and episode;
- visible duration;
- source or version label;
- audio and subtitle availability;
- existing progress;
- audience fit based on information you can verify;
- intended next action.

Languages and subtitles depend on the source and media. Do not infer a track from the title or artwork. Use [the tablet version matrix](/blog/how-to-compare-media-versions-on-a-tablet-detail-screen/) when grouped variants need closer inspection.

## 4. Make one of three decisions

Assign each candidate:

- **select:** fits the brief and identity is verified;
- **defer:** may fit another occasion;
- **verify:** a required field is missing or ambiguous.

A “verify” decision is useful. It prevents an unknown duration, episode, or track from being converted into an assumption.

## 5. Prepare the next screen

If the selection will be used later on another supported device, capture the stable identity fields and intended profile. Do not activate casting, mirroring, remote control, or playback unless those actions are currently verified and wanted.

Norva states that progress and preferences can follow the same account on supported devices. A future target still needs its own source access and item verification.

## 6. Close without changing history

Return to a stable library screen, confirm no candidate started playing, and save only the deliberate favourites changes. On a shared tablet, sign out or hand off according to the household routine.

The [tablet workflow checklist](/blog/a-tablet-viewing-workflow-checklist/) is useful when planning includes travel, offline eligibility, or a long session.

Keep the first pass to five candidates. If none fits, revise one part of the brief rather than adding an unlimited second page of options.

## Common mistakes and limitations

Avoid browsing without a question, treating artwork as identity, starting items to remember them, making favourites changes under the wrong profile, and assuming tablet planning edits the connected source itself.

Source metadata may be incomplete. Recommendations shown by Norva come from the connected source. This workflow organises decisions but cannot guarantee item availability, quality, or suitability for every viewer.

## Frequently asked questions

### Can the tablet edit my source's metadata?

Do not assume so. This workflow uses visible information for planning; source editing requires a separately verified feature and permission.

### Should every candidate go into favourites?

No. Use favourites for items the profile genuinely wants to retain. The planning board can hold temporary candidates without changing the library.

### Can I plan while someone else watches on TV?

Yes as a browsing workflow when supported, but do not infer simultaneous playback rights or remote-control behaviour. Keep the screens' roles separate.
