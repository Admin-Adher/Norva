# A Norva Evaluation Guide for TV-First Households

> **In short:** Confirm that the intended TV setup is currently supported, then evaluate Norva from the household's real seating distance with its normal remote. Trace visible focus through navigation, profile selection, catalog browsing, playback, tracks, exit, and resume. Include shared-screen privacy, caption readability, interruptions, and a recovery route. Keep one personal supported screen available when account entry or evidence capture is safer there.

Norva's TV experience is designed for remote navigation, but a TV-first household needs more than proof that an application opens. It needs a repeatable shared-screen workflow that people can understand from the sofa without exposing account or source details.

## Define what TV-first means

Record the percentage of household viewing that happens on the main TV only as a rough planning input, not a performance claim. More importantly, list the essential routes: choose a profile, find an item, inspect details, play, select an available track, leave, and resume.

Use the [general evaluation framework](/blog/norva-evaluation-framework/) for price and source criteria, then give TV routes the weight they deserve.

## Confirm the supported setup

Check current official Norva information for the intended TV environment. Do not infer support from another model, operating system, store, or household. Record the screen category, application version, remote type, network context, and date without copying device identifiers.

The [first-screen guide](/blog/choose-first-norva-screen/) can help decide whether initial account and source entry should happen elsewhere.

## Run a visible-focus route

Begin on a known element and press one direction at a time. Record expected target, actual target, and whether focus remains clearly visible. Include row boundaries, overlays, filters, the details area, and the Back action. Repeat the identical route once.

If focus becomes trapped or invisible, use the [TV focus diagnostic](/blog/tv-guide-focus-stuck/) to document the boundary instead of pressing rapidly until something moves.

## Judge the interface from the sofa

Use the normal seating distance and room lighting. Check title readability, selected state, metadata density, progress cues, track labels, and error messages. Do not evaluate only while standing close to the panel.

Ask each regular user to complete one short route. A technically possible path may still be too ambiguous for the actual household.

## Test shared-profile handoff

Create only the purposeful profiles the current plan supports. Norva includes up to two profiles and Norva Family up to five; these are profile capacities, not device or simultaneous-playback limits. Confirm that each tester can identify the intended profile without using sensitive labels.

Record whether returning to the profile is clear. Do not include private viewing history in the worksheet.

## Use known media samples

Choose three items from one compatible source the household is authorized to use: an ordinary movie, a series episode, and an item with known audio or subtitle tracks if those matter. Trace details, playback start, pause, seek, available track selection, clean exit, and same-screen resume.

Source-supplied languages, subtitles, artwork, and metadata must be evaluated separately from TV presentation.

## Check accessibility and remote recovery

At normal distance, verify required text-track readability and visible focus. Test what happens after opening and closing an overlay, after a playback interruption, and after returning from another screen. Use only controls visible in the current product; do not assume unverified style or accessibility options.

Record the shortest safe recovery route when a user lands somewhere unexpected.

## Keep a secondary-screen boundary

A TV-first workflow can still use a supported personal screen for private account entry, long text, secure support contact, or cross-screen continuity checks. This does not make the household mobile-first. It separates shared viewing from sensitive administration.

Use the [cross-screen evaluation](/blog/evaluate-norva-cross-screen-continuity/) if resuming elsewhere is a real requirement.

## Include storage and maintenance

Observe whether the TV reports storage pressure and whether the household understands how to review it through device controls. Offline access is conditional on device, source, and rights. Do not assume the TV supports offline storage. The [TV storage audit](/blog/tv-media-storage-audit/) is a later maintenance check, not a compatibility promise.

## Common mistakes and limitations

- Assuming support from a similar television.
- Testing focus by pressing several keys at once.
- Judging readability from beside the screen.
- Using unknown media to test tracks or metadata.
- Exposing source credentials on a shared display.
- Treating profile capacity as a device promise.

## Frequently asked questions

### Must every setup step happen on TV?

No. A personal supported screen may be safer or clearer for sensitive entry and support, even when viewing remains TV-first.

### Does remote-focused design guarantee every route will suit my household?

No. Test the actual supported setup, remote, seating distance, and essential routes before deciding.

### Should every household member run the full test?

No. Ask each regular user to complete one essential route, then assign deeper testing to the evaluator while recording any distinct access need.
