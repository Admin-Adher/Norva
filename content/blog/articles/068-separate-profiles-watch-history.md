# How Separate Profiles Prevent Mixed-Up Watch History

> **In short:** History is easier to interpret when every session begins under the correct viewer profile. The profile should be selected before browsing or playback, remain stable during the session, and be checked again before another person takes over. Separate contexts reduce accidental mixing; they are not a substitute for account security.

History is useful because it answers “what did this viewer open or watch?” In a shared context, that answer becomes noisy when several people use the same profile. One person’s brief preview can appear beside another person’s ongoing series, while progress may no longer match either person’s expectation.

## The profile must be correct before the event

Switching profiles after playback does not change the context under which the earlier action occurred. Build the habit at three moments.

### Before

Confirm the viewer profile before opening history, favourites, or Continue Watching.

### During

Keep the profile stable while browsing and playing. If the wrong profile is discovered, stop and record what happened before switching.

### After

Leave playback normally, then switch profiles before handing control to another viewer.

This before–during–after map is the original evidence framework for the article.

## Why mixed history creates wider confusion

A noisy history affects more than one list. It can make progress appear wrong, make favourites harder to scan, and blur the context used for discovery. The exact relationship between these categories must be verified in the current product before publication, but the organisational principle is clear: a stable viewer context produces more interpretable records.

Read [why individual household profiles matter](https://norva.tv/blog/individual-household-media-profiles/) for the broader decision framework.

## Distinguish a profile error from a sync error

When an item appears at an unexpected position on another screen, first ask:

1. Is the same Norva account active?
2. Is the same viewer profile active?
3. Is this the exact same catalog item and grouped version?
4. Was playback previously performed under another profile?

Do not report a synchronisation fault until those contexts match. Norva publicly confirms cross-device sync of progress, history, favourites, and preferences on supported devices; selecting another viewer context can reasonably produce another view.

The guide to [profiles, favourites, and history](https://norva.tv/blog/profiles-favorites-history/) helps map the categories without assuming they are identical.

## Use a one-week profile habit check

For seven sessions—not seven calendar days—record only non-sensitive context:

| Session | Correct profile chosen first? | Same profile through session? | Clean handoff? |

The goal is not to measure a household member. It is to test whether the routine is clear. If errors repeat, make profile names more recognisable or add a verbal “profile first” reminder.

Do not use sensitive personal information in a profile label.

## Profiles do not guarantee privacy from account users

A profile separates viewer context; it should not be described as a locked account, private vault, or permission system without verified controls. Someone with access to the main account may be able to switch contexts.

Use [the account-security guide](https://norva.tv/blog/secure-media-player-account/) for protection of sign-in and trusted devices. Norva Family should not be presented as a parental-control product; the confirmed distinction is a larger profile allowance.

## Common mistakes and limitations

- Switching profiles only after history looks wrong.
- Creating profiles by device rather than recurring viewer.
- Assuming a profile is a confidential boundary.
- Deleting history before recording the context that produced it.
- Diagnosing sync without checking the active profile and exact item.
- Claiming that every data category is scoped identically without current product verification.

## Frequently asked questions

### Can separate profiles repair existing mixed history automatically?

Do not assume so. They improve future context when used consistently. Review current product controls before deleting or changing existing history.

### Should profiles be named after devices?

Usually not. A recurring viewer may use several screens. A recognisable viewer label better supports cross-device continuity, provided it does not reveal sensitive information.

### Is a shared profile ever reasonable?

Yes, when viewers intentionally share the same history, progress, favourites, and preferences. The convention should be explicit rather than accidental.

### How many profiles are available?

The public pricing page currently states up to 2 for Norva and up to 5 for Norva Family. Profile count does not establish a device or simultaneous-playback limit.
