# The Complete Norva Onboarding Journey

> **In short:** Confirm that you own or are authorized to use a compatible media source, choose one supported screen, secure the account, create a purpose-based profile, and connect only that source. Verify a small catalog sample, then check progress, favorites, and preferences on another supported device. Record unresolved differences without exposing credentials or private source details.

Norva is a software media player and organizer. Its subscription covers the interface, organization, playback, and synchronization; it does not include a media catalog or access. Good onboarding therefore begins with authorization and a small, observable setup rather than a rush to connect everything.

## 1. Confirm the source boundary

Write down which compatible media source you plan to connect and why you are authorized to use it. Keep credentials, private addresses, and complete catalog exports out of the worksheet. The [authorized-source planning guide](/blog/authorized-source-connection-planning-guide/) provides a deeper preparation path.

If ownership or permission is uncertain, pause here. An application setup cannot grant rights that do not already exist.

## 2. Choose one practical first screen

Norva is available on Web, mobile, and TV. The Web version opens in a compatible browser without installation, while the TV experience is designed for remote navigation. Start where account entry and source verification feel manageable for you, not where every household member will eventually watch.

Use the [first-screen decision guide](/blog/choose-first-norva-screen/) to compare input method, privacy, screen ownership, and the task you need to complete.

## 3. Secure the new account

Use a unique password and store it in a reputable password manager. Verify that you are on the official Norva domain before entering account details. If the current account or authentication flow offers an additional verification factor, consider enabling it; do not assume a control exists until it is visible in the official flow.

The [new-account security checklist](/blog/secure-new-norva-account/) covers recovery, shared devices, phishing, and evidence handling.

## 4. Create a profile with a clear purpose

The Norva plan supports up to two profiles, while Norva Family supports up to five. These numbers describe profile capacity, not device limits or simultaneous playback. Create only the profiles the household can explain: for example, one personal viewing context and one shared context.

Choose a neutral label rather than an email address, full legal name, birth date, or health-related detail. See the guides to [creating a first household profile](/blog/create-first-household-profile/) and [naming profiles privately](/blog/name-norva-profiles-clearly/).

## 5. Connect one authorized source

Follow the current Norva interface and support guidance because labels and screen order can change. Add one compatible source you are authorized to use, then stop before adding another. Do not paste credentials into screenshots, notes, or support messages.

The source remains responsible for the media, available languages, subtitles, schedule, and access conditions. Norva organizes what the connected source provides.

## 6. Verify a small catalog sample

Choose a tiny, diverse sample: one item with artwork, one item with multiple metadata fields, and one item you can legally open. Record whether title, category, artwork, and available track information look consistent with the source. The [post-connection sample guide](/blog/verify-sample-after-source-connection/) helps separate source data from presentation.

Avoid treating a total item count as the only success signal. A small identity-based sample is easier to explain and repeat.

## 7. Check continuity on another supported device

Sign in on one additional supported screen only after the first setup is stable. Use the same profile and sample item. Where the relevant options are available, compare catalog visibility, playback progress, favorites, history, and preferences. Languages and subtitles still depend on the source and media.

Record device type, application or browser version, time, profile, and the exact sample. A difference is evidence to investigate, not permission to reset the account.

## 8. Review privacy and conditional features

Read the current privacy policy and terms before expanding household use. Offline access is conditional: it is available only when the device, source, and associated rights permit it. Do not promise another household member that every item or screen will support the same options.

## Common onboarding mistakes

- Connecting several sources before validating the first.
- Using a personal identifier as a profile or source label.
- Treating profile capacity as a device promise.
- Assuming every language, subtitle, or offline option is universal.
- Sharing screenshots that contain credentials or private addresses.
- Resetting a working setup before recording the original state.

## Frequently asked questions

### Does a Norva subscription include media?

No. It covers the software experience. You connect a compatible media source that you own or are legally authorized to use.

### Should I add every household screen immediately?

No. Stabilize and verify one screen first, then add another supported device with a documented comparison.

### Do profiles determine how many devices I can use?

No. The published profile capacities describe profiles only; they should not be converted into a device or simultaneous-playback claim.
