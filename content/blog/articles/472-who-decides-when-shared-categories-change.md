# Who Decides When Shared Categories Change?

> **In short:** First identify who defines the category: the connected source, a personal profile view, or a household organisation choice supported by the current interface. Personal filters belong to the profile owner; shared reversible organisation needs affected-user consultation; source-level or disruptive changes require the source steward and account coordinator. No household rule can override source rights or create an unsupported editing feature.

“Category” can mean a source-defined genre, a visible filter, a household label, or a navigation section. Decision rights depend on that layer.

## Classify the category

Use three classes:

- **personal view:** a filter or preference affecting one profile;
- **shared organisation:** a category or grouping the household intentionally maintains through verified controls;
- **source-defined structure:** metadata or categories supplied by the connected source.

Do not promise that Norva allows editing every class. Use only controls currently visible and supported.

## Assign proposal and approval roles

A profile owner can propose or change a personal view. Affected profile owners should be consulted for shared organisation. The source steward handles source-defined changes with the source owner and account coordinator.

The technical person who can reach a control is not automatically the decision owner.

Name a backup approver for periods when the usual coordinator is unavailable. The backup should follow the same impact check and should avoid redesigning shared organisation during a temporary handover. This keeps routine decisions moving without turning absence into permission for broad, undocumented changes.

## Evaluate impact

Before changing anything, list:

- profiles that rely on the category;
- saved favourites or planning habits affected;
- devices where navigation may differ;
- source refresh implications;
- whether the change is reversible;
- whether it could expose private titles;
- the rollback path.

If the impact is unclear, defer.

## Use proportionate decisions

| Change type | Decision path |
| --- | --- |
| Personal and reversible | Profile owner |
| Shared and reversible | Notify and consult affected users |
| Shared and disruptive | Household approval plus rollback |
| Source-defined | Source owner/steward verifies authorisation and controls |
| Unsupported or unclear | Do not proceed; ask support |

The [one-page household agreement](/blog/create-a-one-page-household-media-agreement/) should define these thresholds.

## Separate organisation from content control

Categories do not automatically filter content, rate media, or provide parental controls. Do not describe a shared category rule as a safety feature unless current verified product documentation supports it.

Norva organises the catalogue from an authorised compatible source. Source metadata can change independently.

## Test a reversible example

When the control is verified, make one small non-sensitive change. Check the result on one affected profile and supported device, then restore it. Record whether labels, navigation, and item identity remain clear.

Do not test by reorganising the entire library.

## Common mistakes and limitations

Avoid letting the person with remote access decide by default, treating source metadata as household-owned, changing every category at once, and using categories as unverified parental controls.

Available controls and source structure can change. This framework assigns household decisions but cannot grant editing rights or guarantee rollback.

## Add a disagreement fallback

For a disputed shared category, preserve the current structure, document the proposed outcome in neutral language, and choose a review date. Test a reversible personal view or temporary label when available instead of forcing a household-wide change.

If the category comes directly from an authorised source, record that boundary and avoid editing local household rules to imply control over source metadata. Escalate only the specific ownership or technical question.

## Frequently asked questions

### Can a profile owner rename a shared category?

Only if the current interface supports it and the household decision path allows it. Verify the affected layer first.

### Who decides a source-defined category?

The source owner or steward handles authorised source controls; household members can report impact and request review.

### Should every change require a vote?

No. Use proportionate thresholds. Personal reversible choices should remain lightweight.
