# How to Read a Video Quality Badge Carefully

> **In short:** Treat a quality badge as a claim about a defined media or product property, not proof of the complete picture reaching the viewer. Find the product's current definition, then separate available representation, active selection, decoded configuration, output signal, display capability and mode, and visible result. Mark every layer the interface does not expose as unknown.

A small label can be useful, but its meaning is service-specific. The same text may describe a title's available version, a selected file, a nominal resolution, a dynamic-range format, or another property.

## Capture the badge in context

Record exact text, icon, location, page, selected item or version, and whether the badge appears before playback, during playback, or in diagnostics. A pre-playback catalog-style label and an active playback indicator are not interchangeable.

Do not translate a branded label into a technical standard without an official definition.

## Find the official definition

Use current product documentation, help, or support information. Record date and platform because meaning or availability can change. For Norva, verify badge definitions against official product sources; do not infer them from mockups or unrelated services.

Norva organises and plays compatible sources users own or are authorised to use, so source metadata can remain dependent on those connected sources.

## Separate six evidence layers

1. the badge definition;
2. media properties available for the title or version;
3. the representation or file currently selected;
4. what the device decodes;
5. what the output sends;
6. what the display accepts and renders.

The visible picture is then influenced by source quality, processing, display settings, and environment.

## Original evidence: verification ladder

| Layer | Evidence | Verified value | Unknowns | Does the badge prove it? |
|---|---|---|---|---|
| Definition | Official source | Meaning | Missing detail | Yes/partial/no |
| Available/selected media | Player diagnostics | Value | Missing data | Result |
| Decode/output | Device diagnostics | Value | Missing data | Result |
| Display/rendering | Input status/mode | Value | Processing | Result |

Use "unknown" rather than filling a gap with visual expectation.

## Interpret resolution labels

A resolution badge may refer to encoded dimensions or an available representation. It does not describe bitrate, source detail, codec efficiency, frame rate, color depth, dynamic range, current selection, or output scaling. [The resolution-versus-bitrate guide](/blog/resolution-and-bitrate-why-they-are-not-the-same/) explains the distinction.

An upscaled lower-resolution source can also reach a higher-resolution output without becoming equivalent to native source detail.

## Interpret dynamic-range labels

An HDR-related badge may indicate a format or available version according to the product. It does not prove compatible decoding, correct transfer through intermediate devices, active display mode, or one consistent visual result. Use [the dynamic-range guide](/blog/dynamic-range-explained-for-everyday-viewers/) to map the path.

ITU-R BT.2100 defines HDR television system parameters; a product badge still needs its own official interpretation.

## Observe adaptation and fallback

Network playback may select among representations or fall back according to current conditions and device support. [The automatic-quality guide](/blog/how-automatic-quality-selection-usually-responds-to-conditions/) shows why an availability badge may not equal the active state.

Record quality switches, decode errors, and output changes separately.

## Check accessible presentation

The badge text or icon should remain understandable at supported larger text settings, from relevant shared-screen distances, and with keyboard or remote focus. Color alone should not carry its meaning. This interface accessibility does not change the underlying media claim, but it affects whether viewers can use it.

## Report a badge mismatch

Include badge text and screenshot without private data, official definition and date, platform, media version, active playback diagnostics, device, output, display mode, exact scene, observed mismatch, and unknowns. Do not include source URLs, tokens, or account identifiers.

## Common mistakes and limitations

Avoid treating a badge as a live bitrate meter, assuming it proves display output, comparing definitions across services, or claiming a visual result from metadata alone. Some layers may remain unverifiable to an everyday viewer.

## Frequently asked questions

### Does a resolution badge prove the current stream uses those dimensions?

Not necessarily. Check the product definition and active playback diagnostics where available.

### Does an HDR badge prove the display is in HDR mode?

No. Decode, output, intermediate devices, and display state still require verification.

### Can a badge guarantee better perceived quality?

No. Source, encode, delivery, decode, processing, display, and viewing context all contribute.
