# A Workflow Checklist for a Clear Series Library

> **In short:** Treat library clarity like a release gate. Verify identity and hierarchy first, then variants and tracks, viewer state, discovery routes, and recovery. Mark every check pass, fail, or unknown with evidence. Do not “fix” an unknown by guessing, and do not perform bulk cleanup until a reversible test proves the intended scope.

A checklist is useful only when it changes a decision. This one is not a list of cosmetic preferences. It identifies conditions that can send a viewer to the wrong episode, merge household progress, hide completed works, or lose a recovery path after source metadata changes.

Record the date, connected source, account or viewer context, and supported device tested. “Pass” without evidence is only an opinion.

## Gate 1: identity

- [ ] Canonical series and alternate titles are distinguishable.
- [ ] The work is correctly identified as a series, miniseries, or related format.
- [ ] Stable identifiers are retained where available.
- [ ] Remakes and similarly named works remain separate.
- [ ] Each episode under test belongs to the intended series.

EIDR’s public hierarchy distinguishes series, seasons, episodes, edits, and manifestations. Use that separation as a model for checking identity before presentation.

## Gate 2: hierarchy and sequence

- [ ] Seasons, specials, and split releases are not silently merged.
- [ ] Episode numbers and titles agree with authoritative evidence.
- [ ] Duplicate numbers are explained rather than deleted automatically.
- [ ] Sequence gaps are classified as confirmed, unavailable, or unresolved.
- [ ] The next episode is verified from the last completion.

Run [the episode gap workflow](/blog/find-gaps-in-episode-sequence/) whenever numbering is discontinuous.

## Gate 3: variants and languages

- [ ] Alternate edits or releases remain traceable.
- [ ] Grouped versions still expose their distinguishing attributes.
- [ ] Audio and subtitles are recorded separately.
- [ ] Required tracks are checked episode by episode.
- [ ] Language preferences are not mistaken for track availability.

Use [the season track-coverage matrix](/blog/check-language-across-episodes/) before declaring consistent language support.

## Gate 4: viewer and household state

- [ ] Resume position belongs to the correct episode and version.
- [ ] Completion is separate from availability and favorite state.
- [ ] Each household viewer has a known scope or external fallback.
- [ ] Shared sessions update only actual participants.
- [ ] Weekly release state is not confused with personal progress.

Test state with a harmless episode before touching valuable progress. Confirm on another supported device only when cross-device behavior matters.

## Gate 5: discovery and completion

- [ ] Active queues contain genuine unfinished viewing.
- [ ] Pilot samples are promoted, deferred, or cleaned deliberately.
- [ ] Completed series retain at least two useful retrieval routes.
- [ ] Favorites have a stated series-level or episode-level purpose.
- [ ] Spin-offs and companions use typed relationships.

Do not delete history to create visual emptiness. A clean library distinguishes urgency from long-term discoverability.

## Gate 6: source freshness and change control

- [ ] The source check has a timestamp.
- [ ] Metadata changes are classified before reconciliation.
- [ ] Before-and-after snapshots exist for important changes.
- [ ] Availability changes are not treated as identity deletion.
- [ ] Bulk actions are avoided when matches are uncertain.

Use [the post-update series audit](/blog/audit-series-after-source-update/) after renumbering, variant regrouping, or changed track metadata.

## Gate 7: interaction and recovery

- [ ] Focus, touch, keyboard, and TV remote paths work for the target device.
- [ ] Selected state and current focus are visually distinct.
- [ ] Destructive actions explain scope and require deliberate confirmation.
- [ ] Status messages state what changed and how to recover.
- [ ] A last known-good state is recorded before correction.

FIAF cataloguing guidance reinforces the value of precise, consistent audiovisual description. Interaction checks then ensure that people can actually reach and understand that information.

## Decide whether the library passes

Treat identity, hierarchy, resume context, and recovery as critical gates. A failure in one of them blocks “ready.” Language can also be critical when a viewer requires a specific track. Assign each unknown an owner and a next check; never convert it to pass because the interface looks plausible.

Norva can organize compatible authorized sources, group variants, and retain account state across supported devices. Exact behavior should be confirmed against current support material and the actual build under review.

## Common mistakes and limitations

- Checking only the first episode of each series.
- Calling an unavailable item deleted.
- Using one household progress state.
- Treating a product preference as source evidence.
- Passing a gate without dated evidence.
- Cleaning before preserving a rollback point.

## Frequently asked questions

### How often should the checklist run?

Run it after meaningful source changes, before a large cleanup, and when identity or progress symptoms appear. Stable libraries need only targeted reviews.

### Must every unknown block release?

No. Classify impact. Unknown episode identity or resume scope is critical; an unknown artwork credit may not affect playback.

### Can the checklist be automated?

Some comparisons can, but ambiguous relationships, household intent, and destructive cleanup still require human review.
