# Which Screen Should You Use First With Norva?

> **In short:** Start in a compatible browser when private text entry and a no-install Web session suit the task; choose mobile when a personal touch screen is easier to control; choose TV when remote navigation is the first behavior you must verify. Confirm current support, use one authorized source, test a small sample, and add other screens only after the baseline works.

Norva is available on Web, mobile, and TV, but “available” does not make every screen equally practical for every first task. The useful question is not which category is universally best. It is which supported screen gives you the clearest, safest path through the next onboarding step.

## Start with the job, not the largest display

Write one sentence that defines the first job. Examples include creating the account, entering source details privately, checking touch navigation, or confirming remote focus. A single job keeps the decision observable.

If you are completing the full setup, follow the [complete onboarding journey](/blog/norva-onboarding-complete-journey/) after choosing the screen.

## When Web is a practical first choice

The Web version runs in a compatible browser without installation. It can be a practical starting point when you want a physical keyboard, a private computer, and space to compare non-sensitive notes with the current interface.

First confirm browser compatibility through current Norva guidance. Avoid using a public or unmanaged computer for account or source details. A browser session does not remove the need to protect credentials, verify the official domain, or sign out on a shared device.

## When mobile is a practical first choice

Mobile can suit a person who prefers touch input, owns the device, and wants to verify the personal-screen experience first. It also creates a useful control later when the TV is shared.

Do not assume every mobile device or operating-system version is supported. Check the current official availability information. Consider whether notifications, screenshots, clipboard history, or screen sharing could expose sensitive entry fields before you begin.

## When TV is a practical first choice

The Norva TV experience is designed around remote navigation. Start there when the essential first question is whether directional focus, selection, and back behavior make sense on the supported TV setup the household will actually use.

TV may be less convenient for long or sensitive text entry, depending on the device and input method. Do not photograph credentials or move them through an insecure message merely to make entry easier. The [screen-for-source-setup guide](/blog/choose-screen-for-source-setup/) helps separate setup convenience from viewing preference.

## Check ownership and privacy

Prefer a screen you control, can update, and can sign out from. On a shared screen, decide who owns the account session and who may change source or profile settings. Keep full names, email addresses, private source locations, and credentials out of comparison notes.

The [account security checklist](/blog/secure-new-norva-account/) covers safe entry and shared-device cleanup.

## Confirm support before investing time

Use current Norva product and support pages rather than assuming compatibility from a screen category alone. Record the browser or application version and device type without exposing unique identifiers. If support is uncertain, make that the first question instead of building a large configuration.

## Use the same small verification sample

Whichever screen you choose, connect only a compatible source you are authorized to use and verify a tiny catalog sample. Record one title or neutral identifier, expected category, visible artwork state, and whether the item opens as expected. The [sample-verification guide](/blog/verify-sample-after-source-connection/) provides a repeatable method.

Do not compare different profiles, sources, or items and call the result a screen difference.

## Plan the second screen deliberately

Once the first baseline is stable, add one other supported device. Use the same account, profile, source, and sample. Where relevant options are available, compare catalog visibility, progress, favorites, history, and preferences. Languages and subtitles depend on the source and media.

The second screen is a verification step, not a requirement to configure every device at once.

## Common decision mistakes

- Choosing by screen size instead of the first task.
- Assuming every browser, phone, or TV is supported.
- Entering secrets on an unmanaged or observable screen.
- Comparing different profiles or samples across screens.
- Adding every device before the first baseline is stable.
- Treating profile capacity as a device limit.

## Frequently asked questions

### Is Web always the easiest place to start?

No. It is useful when a compatible browser and keyboard suit the task, but ownership, privacy, and the behavior you need to verify may point elsewhere.

### Should I begin on TV if that is where I will watch most often?

Only if remote navigation is the first priority and sensitive entry can be handled safely. Viewing preference and setup convenience are separate decisions.

### Can I switch screens after onboarding?

Norva supports Web, mobile, and TV on supported devices. Establish one baseline, then compare a second supported screen with the same account context.
