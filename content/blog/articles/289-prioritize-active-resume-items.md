# How to Prioritize Genuinely Active Resume Items

> **In short:** Prioritise by declared next action, not by row position, recency, or progress percentage alone. Place a title in “next session” only when the right viewer, version, and access needs are confirmed and a realistic session is planned. Keep deliberate pauses visible but separate, and move unknown entries into investigation rather than the active queue.

A tidy row can still be poorly prioritised. The key test is simple: does the first visible choice represent what the viewer genuinely expects to resume next?

## Use a four-tier intent ladder

| Tier | Qualification | Review trigger |
|---|---|---|
| Next session | Named viewer, exact item, realistic near-term plan | After the session |
| Active soon | Clear intention, but no immediate slot | Weekly |
| Deliberate pause | Return condition or date exists | On that condition |
| Unresolved | Viewer, version, or intent is unclear | Investigate separately |

The ladder avoids a false precision score. A title does not become more important merely because its progress is higher. A nearly finished episode may be paused for months, while a newly started film may be tonight’s priority.

## Require one observable reason

Each first- or second-tier item needs a sentence with a verb and context:

- “Resume episode 4 on Friday with the shared viewer.”
- “Finish the last section on mobile during the authorised offline trip.”
- “Continue version B because it has the required subtitles.”

“Recent” or “looks interesting” is not enough. The reason should identify owner, action, and constraint without recording unnecessary personal detail.

If the decision is really between pause and abandonment, use [the paused-or-abandoned framework](/blog/paused-or-abandoned-title/) before assigning a priority.

## Confirm readiness, not just desire

Check exact version, episode, source availability, language or subtitle needs, device context, and expected session length. Languages, subtitles, and offline access depend on the source, media, device, and associated rights. A high-intent item that is not currently ready belongs in a blocked list, not at the top of an actionable row.

DCMI metadata distinctions for relation, format, and language make a compact readiness check possible. You do not need a large catalogue record; only the fields that affect the next action.

## Balance personal and shared priorities

On a shared screen, do not let one person’s queue silently become household order. Confirm the active profile and label the planned session personal or shared. Before group viewing, follow [the shared-session row review](/blog/prepare-resume-row-for-shared-session/) so the group choice is visible without displacing legitimate personal intent.

W3C’s focus-order guidance addresses interfaces whose navigation order preserves meaning and operability. The same principle is useful editorially: the order of a viewing queue should support the intended action sequence, not create confusion.

## Create a small active window

Select only the items needed for the next few real decisions. This is not a universal numerical cap. The window is small enough when the viewer can explain each entry and select the next one without reopening unresolved research.

Review it in this order:

1. remove completed actions from the active window after verifying state;
2. promote an “active soon” item only when a session becomes realistic;
3. keep deliberate pauses tied to their trigger;
4. investigate unknowns outside the decision path;
5. leave the underlying row untouched when current controls are unverified.

Use [the weekly resume-row review](/blog/weekly-resume-row-review/) to refresh tiers without constantly reorganising them.

## Original evidence: the intent ladder card

Choose eight non-sensitive entries and write one observable reason for each. If you cannot write a reason, classify the item unresolved rather than active. Then ask: could the top tier guide the next session without opening another list?

Record only the number of items that changed tiers and the missing field—viewer, version, timing, or readiness. This creates a reproducible prioritisation audit without inventing engagement data.

## Common mistakes and limitations

- Ranking solely by recency or percentage.
- Treating every paused title as abandoned.
- Mixing blocked and actionable items.
- Prioritising shared sessions without consent.
- Assuming row order can always be manually changed.
- Recording sensitive viewing reasons in a household note.

The ladder organises decisions. It does not promise that the current interface supports custom ordering, pinning, or every queue operation.

## Frequently asked questions

### How many titles belong in “next session”?

Only as many as represent real near-term choices. Use decision clarity rather than a universal count.

### Should almost-completed titles always come first?

No. Completion proximity is one context signal; viewer intent and readiness determine priority.

### What happens to unresolved items?

Review them separately for identity, profile, and version. They should not block the active viewing decision.
