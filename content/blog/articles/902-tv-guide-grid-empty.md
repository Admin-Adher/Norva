# TV Guide Grid Empty? Start With These Context Checks

> **In short:** First decide whether the grid has no channel rows or has rows with no program cells. Record the authorized source, channel group, selected date and time window, device clock and time zone, account, profile, filters, device, application version, and timestamp. Verify two source channels and one schedule sample, remove one visible restriction, compare another supported device, and avoid repeated refreshes or source edits before saving evidence.

An empty-looking guide can describe several states: no channels, channels without listings, listings outside the selected window, hidden rows, or a layout that fails to reveal existing cells. Naming the state determines the next check.

## Classify the blank screen

Record whether channel labels appear, row outlines exist, a loading or empty message is visible, the time header appears, and focus can enter the grid. Do not call a layout or focus problem missing data until channel and program presence are checked.

The [TV guide troubleshooting handbook](/blog/tv-guide-troubleshooting-handbook/) provides the layer matrix.

## Verify the guide window

Record selected date, visible start and end times, device-local clock, time zone name, and offset where visible. Return to the current time using only documented controls. A correct schedule outside the visible window can look empty.

## Confirm account, profile, and filters

Record the active Norva account and profile, channel group, source selection, favorites-only or availability controls, category, search query, and any other visible filter. Remove one restriction, observe, then restore the baseline.

Use the [guide-filter checklist](/blog/guide-filter-hides-channels/) when channel rows return after a filter change.

## Check source channel coverage

Through the provider's official authorized route, confirm that at least two sampled channels currently exist and expose schedule information for the same time window. Record masked channel labels, source identity cues, and timestamps. Do not export the complete channel list.

## Separate no channels from no listings

If channel rows are absent, focus on source selection, group, filters, profile, and channel availability. If rows exist but program cells are blank, focus on schedule coverage, time context, and channel-to-schedule identity. The [partial coverage guide](/blog/one-channel-guide-others-blank/) handles mixed rows.

## Check channel identity cues

Record visible channel name, number or source identifier where shown, logo state, and source label. Same-name channels may not share schedule data. Do not infer mapping from name or logo alone.

## Compare another supported device

Use the same account, profile, source, channel group, filters, date, guide window, and close timestamp. Record both device clocks, zones, and application versions. A populated grid on one screen establishes a cross-device difference, not its cause.

## Preserve the request timeline

Record when the guide was opened, any visible loading state, first blank observation, navigation changes, source update, refresh request, application update, and current state. Use only current official timing guidance; otherwise report elapsed observation without a deadline.

## Avoid destructive first steps

Do not clear application data, reinstall, remove and reconnect the source, edit channel identifiers, change device time manually, or repeat refreshes before preserving the baseline. Those actions create new variables.

## Classify the evidence

Use no channel rows, channel rows without listings, guide window mismatch, time-zone mismatch, filter or group difference, source schedule absent, channel identity uncertain, device-specific display, focus or layout issue, or unknown. Keep the observation separate from implementation theories.

## Prepare a redacted support record

Use the [guide evidence template](/blog/guide-issue-support-evidence/) with screen classification, stable context, sampled channels, source observations, clock and zone, device versions, timeline, and actions. Exclude credentials and full source addresses.

## Common mistakes and limitations

- Treating no rows and empty rows as the same symptom.
- Ignoring selected date, time window, or time zone.
- Comparing different channel groups or profiles.
- Matching channels by logo alone.
- Repeating refreshes before recording the first state.
- Sharing a complete private channel list.

## Frequently asked questions

### Should I change the device clock manually?

Not as a first diagnostic step. Record the current automatic or manual state and follow official device and Norva guidance.

### Does a blank grid prove the source has no schedule?

No. Verify source coverage, channel identity, guide window, filters, and another supported device separately.

### How many channels should I sample?

Use a small representative set, commonly two affected channels and one normal control when available.
