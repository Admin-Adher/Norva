# Poster Artwork Missing? A Data and Display Checklist

> **In short:** Confirm the catalog entry and text metadata exist before treating a blank poster as a missing item. For a few samples, record item and version identity, poster state at the authorized source, Norva poster state by view, account, profile, filters, grouping, device, application version, network, and timestamps. Compare a normal control and another supported device, avoid source image edits first, and never assume an undocumented image pipeline or cache.

A poster is one artwork type in one presentation context. A blank poster, generic fallback, wrong image, missing season image, and absent media card are different symptoms.

## Define the poster state

Record whether the poster area is blank, generic, incorrect, low quality, intermittently visible, or absent with the entire card. Note category, search, detail, favorite, progress, series, or episode view. Do not merge view-specific observations.

The [category and metadata handbook](/blog/category-metadata-troubleshooting-handbook/) provides the field matrix.

## Confirm entry and text presence

Record whether title, year, description, category, source label, and detail or playback controls appear. A card with text but no image is an artwork symptom. A missing card belongs in the import and sync investigation.

The [artwork-after-import guide](/blog/artwork-missing-after-import/) covers the wider import timeline.

## Confirm item and version identity

Compare source, year, media type, season, episode, duration, edition, language cues, and grouped-version state. A wrong poster may belong to another visible version rather than being missing for the intended one.

## Verify source poster data

Through the provider's official authorized route, record whether a poster currently appears for the same sample and whether another image type exists. Note source update time. Do not copy private image addresses or infer how Norva retrieves or transforms images.

## Freeze view context

Record account, profile, enabled sources, availability, category, filters, search, sorting, grouping, device, application version, network, and time. Compare the same sample in each view without switching source versions.

## Use affected and control samples

Choose up to three affected items and one normal poster from the same source, media type, and view. Assign privacy-safe codes. A normal control narrows scope but does not prove the affected image's cause.

## Compare another supported device

Use the same account, profile, sources, filters, grouping, sample, and close timestamp. Record both application versions and networks. A poster visible on one screen only is cross-device evidence. Use the [older-catalog comparison](/blog/one-device-shows-old-catalog/) before clearing local state.

## Compare one trusted network

If practical, use the affected device on another trusted network while keeping context stable. Record result and certificate state. Do not disable certificate validation, filtering, privacy controls, or install unknown profiles to force an image to load.

## Preserve timing

Record source poster confirmation, source edit, import or refresh, first Norva observation, device comparison, and current state. An image appearing later establishes a timeline but not the internal reason. Use official timing only when currently published.

## Avoid image edits first

Do not replace, resize, rename, or relocate the source image, repeat imports, clear application data, reinstall, or remove the source before saving evidence. A correction changes the input and can hide the original issue.

## Classify the result

Use source poster absent, source poster present, wrong source version, one-view difference, one-device difference, network-specific observation, generic fallback, entry or text also missing, resolved by documented step, or unknown. Avoid claims about storage, transformation, or cache behavior.

## Build support evidence

Use the [metadata evidence pack](/blog/metadata-support-evidence-pack/) with redacted sample codes, source and Norva poster states, identity cues, views, context, devices, versions, networks, and timestamps. Exclude full catalog screenshots and private image addresses.

## Common mistakes and limitations

- Treating a missing poster as a missing media item.
- Comparing different source versions or views.
- Editing the source image before preserving evidence.
- Calling the cause a cache failure without proof.
- Disabling network security controls.
- Sharing complete catalog screenshots.

## Frequently asked questions

### Does a blank poster mean the import failed?

No. Confirm entry, text, source image, view, device, network, and timing as separate observations.

### Should I upload a new poster immediately?

Not before preserving the original source and Norva states and confirming metadata ownership and support guidance.

### What if the poster appears on another device?

Record both contexts, versions, networks, and timestamps. That establishes a cross-device difference, not its internal cause.
