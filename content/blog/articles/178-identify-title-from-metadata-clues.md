# How to Find a Forgotten Title From Metadata Clues

> **In short:** Write every clue you remember, then rank each by certainty, rarity, searchability, and independence. Start with the strongest searchable clue—often a distinctive person, series, original-title fragment, or narrow time period. Add one independent clue only after seeing candidates. Verify the final title with at least two details that were not generated by the same potentially wrong record.

Memory often preserves attributes rather than names: “a short film in French,” “a series with this actor,” or “an episode before a cast change.” Turning those memories into a disciplined fingerprint avoids random browsing.

“Independent” means the clue does not merely repeat another one. Poster colour and a remembered actor face may come from the same visual memory; year and a real-world life event may be more independent.

## Choose the first route

Use the highest-ranked searchable clue:

- person → [person-credit search](/blog/find-media-by-person-credits/);
- reliable year plus name fragment → [title-and-year search](/blog/search-by-title-and-year/);
- original or translated wording → [two-form title search](/blog/search-original-and-translated-titles/);
- known series → browse hierarchy before global episode search;
- source and type → browse or filter when both are certain.

Do not enter a free-form plot paragraph unless the interface explicitly searches synopsis text.

## Intersect one clue at a time

Suppose person search returns 40 titles. Add one reliable type or year band. If the set drops to five plausible works, inspect them before adding language or source.

| Attempt | Base clue | Added clue | Candidates | Did expected class remain? | Next step |
|---|---|---|---:|---|---|
|  |  |  |  |  |  |

If zero results appear, remove the latest clue. The sequence identifies which memory or metadata field conflicts.

## Separate clue meaning from field meaning

“French” might describe the original title, audio, subtitles, country, or where you watched the work. “2018” might be release, episode broadcast, edition, or viewing year.

Dublin Core terms distinguish title, creator, date, language, type, relation, subject, and description. Use that separation to translate memories into the correct field hypothesis.

## Generate a candidate evidence table

For the shortlist:

| Candidate | Person | Date meaning | Type | Language role | Story clue | Source/version | Decision |
|---|---|---|---|---|---|---|---|
|  |  |  |  |  |  |  |  |

Require two independent matches. Artwork can support recognition but should not override creator, type, or source identity.

The Library of Congress Advanced Search guide demonstrates separate indexes for title, creator, series, identifiers, and subjects. A personal-media interface may expose fewer searchable fields; use non-searchable clues to verify results rather than assuming they constrain the query.

## Use negative clues carefully

“It was not a series” or “probably before 2020” can remove candidates, but uncertain negatives are dangerous. Mark them as soft exclusions and revisit them if the candidate set becomes empty.

Do not use a disliked genre or presumed source as a hard exclusion unless the memory is reliable.

## Turn failure into metadata evidence

If a title is found through browsing but not through the strongest expected clue, capture the record, field, query, source, and device. The person credit, alternate title, year, or series relationship may be missing or unindexed.

## Common mistakes and limitations

- Treating every memory as equally certain.
- Combining five clues in the first query.
- Confusing audio language with title language.
- Using viewing year as release year.
- Searching a plot field that is not indexed.
- Accepting one matching clue as identity proof.

Memory can be wrong, and source metadata can be incomplete. Preserve several candidates when evidence remains balanced.

## Frequently asked questions

### Which clue is usually strongest?

A distinctive verified person, series, or title token often carries high information. The best clue is the most certain, rare, and searchable in the current interface.

### Should I start with genre?

Genre is interpretive and often broad. Use it after a stronger clue unless the catalogue is small and the genre requirement is firm.

### What if I remember only a scene?

Extract concrete entities—person, place, object, period, episode context—then use those as search or verification clues. Avoid inventing a title from the scene.
