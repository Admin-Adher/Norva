# Audio Badge Disagrees With the Track List: Which Evidence Wins?

> **In short:** Confirm the exact source version, then record the audio badge on cards and detail views, source language fields, visible track list before playback, track list during authorized playback, selected track, profile language preference, device, application version, and timestamp. Preserve each label exactly, compare another supported device, and treat badge and track list as separate observations. Do not infer how Norva builds badges or promise track availability from a badge alone.

A compact badge summarizes something visible in the interface; a track list describes options exposed in another context. When they disagree, neither should be silently rewritten into the other.

## Confirm version identity

Record source label, title code, year, media type, duration, edition, season, episode, and grouped-version state. Different source versions can expose different audio options while sharing title and artwork.

The [category and metadata handbook](/blog/category-metadata-troubleshooting-handbook/) provides the identity matrix.

## Record badges by view

Capture the exact badge text on category card, search, detail page, episode list, and grouped-version card where shown. Note abbreviations, separators, order, “multi” wording, or absent badge without translating it into a guessed language set.

## Record source language fields

Through the provider's official authorized route, transcribe visible audio-language or track fields for the same version. Keep labels and codes exact. Do not assume the provider field and Norva badge have identical scope.

## Record pre-playback tracks

If Norva exposes a track list before playback, capture each visible label and default selection. Record profile, interface language, filters, and source version. An absent pre-playback list does not prove tracks are unavailable during playback.

## Record in-playback tracks carefully

When authorized playback is appropriate, open the language control without advancing content unnecessarily. Record visible track labels, selected track, and timestamp. Do not test every track or expose household viewing history. Preserve existing progress.

## Keep labels literal

Record “French,” “fr,” “Original,” “Multi,” regional variants, descriptive audio, or unknown exactly as shown. Do not normalize codes, merge variants, or infer language from speech unless current official guidance requires such verification.

The [localized-title guide](/blog/localized-title-mismatch/) explains why interface, metadata, and track languages remain distinct.

## Freeze profile and device context

Record account, profile language or audio preference where visible, filters, grouping, device, operating system, application version, and time. A selected or preferred track can differ by profile without proving the badge changed.

## Compare another supported device

Use the same account, profile, source version, and close timestamp. Record both application versions. A mismatch on one screen is cross-device evidence, not proof that the source or badge is globally wrong.

## Compare subtitles separately

Do not treat subtitle tracks as audio evidence. If the subtitle badge disagrees with its list, use the [subtitle verification path](/blog/subtitle-badge-disagrees-track-list/) and keep both tables separate.

## Avoid metadata edits and repeated imports

Do not rename tracks, change language codes, edit the badge, remove the source, or repeat imports before preserving the baseline. Confirm metadata ownership and follow current support guidance.

## Classify the mismatch

Use different source version, badge differs by view, source fields differ from badge, pre-playback list differs, in-playback list differs, profile selection difference, device-specific presentation, labels ambiguous, or unknown. Do not invent a badge-generation or track-selection rule.

## Build support evidence

Use the [metadata evidence pack](/blog/metadata-support-evidence-pack/) with item cues, badge screenshots or text, track tables, profile context, devices, versions, and timestamps. Exclude credentials and unnecessary titles.

## Common mistakes and limitations

- Comparing different source versions.
- Treating a badge as a guaranteed track inventory.
- Mixing subtitle and audio evidence.
- Normalizing ambiguous labels without documentation.
- Changing progress while testing tracks.
- Editing source language metadata before baseline capture.

## Frequently asked questions

### Which evidence should I trust for playback options?

Record the exact in-playback list for the same authorized version and context, while keeping the badge as a separate presentation observation.

### Does “Multi” guarantee specific languages?

Do not infer a language set from an undefined label. Record the visible track list and current official documentation.

### Should I rename source tracks?

Not before preserving evidence, confirming metadata ownership, and following provider and Norva support guidance.
