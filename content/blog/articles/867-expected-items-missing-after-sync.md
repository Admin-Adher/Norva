# Expected Items Missing After Sync? An Evidence Checklist

> **In short:** Verify each sampled item still exists in the authorized source and that household access remains valid. Record the sync timeline, Norva account, profile, device, application version, source label, categories, filters, search query, grouping, and last known appearance. Compare visible title, year, media type, season, episode, duration, and version cues. Check another supported device, separate missing artwork or metadata from entry absence, and avoid repeated syncs.

“Expected” needs evidence. A title remembered from last week, an item visible in another profile, and an entry currently available at the source create different investigations.

## Choose a small representative sample

Select up to three expected entries: one recently visible, one from the affected category, and one version or episode when relevant. Assign neutral sample codes. Do not export the household catalog or list sensitive viewing interests unnecessarily.

The [import and sync handbook](/blog/catalog-import-sync-troubleshooting-handbook/) keeps the sample within a broader evidence matrix.

## Confirm current source availability

Through the source provider's official authorized route, verify whether each sample is currently present and accessible to the same source account. Record time, media type, category, and visible version cues. If the source no longer exposes an item, classify that observation separately from Norva display.

## Reconfirm authorization and scope

Check source owner permission, account status, household role, and any visible access restrictions. Do not assume a working sign-in gives every profile or account the same catalog scope. Never put source credentials in the checklist.

## Preserve the sync timeline

Record the last known Norva appearance, source change time, sync request or visible state, completion or last change, and first observed absence. Include device-local timezone. If official guidance states a timing window, cite the current page; otherwise report elapsed time without inventing a deadline.

## Freeze the Norva context

Use the same account, profile, source selection, category, availability, year, rating, audio, subtitles, search query, sort, and grouped-version state. Remove one visible filter at a time, return to baseline, and record the result. A filtered item is not an absent item.

For an entirely empty result, use the [zero-item guide](/blog/import-finishes-with-zero-items/).

## Compare visible item identity

Search and browse using title, year, media type, season, episode, duration, source label, and version cues where shown. Similar artwork or names do not prove the same entry. The sample may appear under another version or group.

Do not infer hidden identifiers or alter metadata at the source merely to force a match.

## Separate entry absence from presentation gaps

If a card exists without artwork, the entry is not missing. If metadata text appears without a usable card, record both observations. Use the [artwork diagnostic](/blog/artwork-missing-after-import/) or [metadata-without-entry guide](/blog/metadata-visible-media-entry-missing/) rather than merging the symptoms.

## Compare one other supported device

Use the same trusted account, profile, source selection, filters, grouping, and approximate observation time. Record application versions. If only one device lacks the item, preserve the cross-screen difference before clearing local data or reinstalling.

## Avoid evidence-destroying changes

Do not repeat the sync, remove and reconnect the source, rotate valid credentials, bulk-edit categories, clear history, or erase application data before the baseline is saved. If one action already occurred, add it to the timeline.

## Classify each sample

Use source no longer exposes, authorization or scope differs, hidden by view control, found under another identity, device-specific display, metadata-only issue, or unknown. A three-item sample cannot prove that every catalog entry behaves the same.

## Common mistakes and limitations

- Relying on memory instead of current source evidence.
- Comparing different profiles or access scopes.
- Leaving search, language, availability, or grouping filters active.
- Treating missing artwork as a missing entry.
- Repeating sync before preserving the first timeline.
- Sending a complete private catalog to support.

## Frequently asked questions

### How many missing items should I document?

Use a small representative set. Three privacy-safe samples usually reveal more useful distinctions than a complete catalog export.

### Does source availability guarantee Norva will show the item?

No guarantee should be inferred. It confirms one layer; context, eligibility, metadata, identity, and display still require observation.

### Should I change the title metadata at the source?

Not as an initial diagnostic step. Preserve current identity cues and ask support before changing source data.
