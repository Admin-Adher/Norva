# What to Prepare Before Adding Your Media Source

> **In short:** Confirm that you own the source or are authorised to use it, verify that Norva supports its connection method, collect current settings from a primary record, choose one known item for checking, and prepare a private setup note. Resolve unclear permission or compatibility before you enter any credentials.

Preparation keeps three different failures from being mixed together: an unauthorised source, an unsupported method, and an incorrect setting.

## Gate 1: establish authorisation

Record the source owner or operator, the account holder, the basis of access, the applicable terms, and the date you checked them. Note any household, device, territory, duration, or connection restrictions without trying to interpret them more broadly.

Norva’s terms require a compatible source that the user owns or is legally authorised to access. Technical acceptance does not establish permission.

The practical [authorised-media checklist](/blog/authorized-media-access-explained/) explains this boundary without offering legal advice.

## Gate 2: confirm compatibility

Find current Norva documentation for the source’s connection method. Do not infer support from similar-looking form fields or an old third-party article.

Save the official documentation URL and date. If the answer is unclear, ask [Norva support](https://norva.tv/support) before entering private details.

“Available on TV” and “compatible with this source” are different claims. Verify both if both matter.

## Gate 3: collect current settings privately

Obtain settings from the source owner’s official account page or administrator. Create a private table:

| Field | Primary origin | Checked | Sensitive? |
| --- | --- | --- | --- |
| Source address or file |  |  | Possibly |
| Account identifier |  |  | Yes |
| Password or token |  |  | Yes |
| Guide information |  |  | Possibly |
| Display name | Your choice |  | No |

Do not write real secrets in a worksheet that might be shared. Reference their secure storage location instead.

Norva’s privacy policy says selected source settings are used to connect the service to the source on the user’s behalf.

## Gate 4: choose one reference item

Pick one item whose identity and expected availability you can confirm from the source. It should have an unambiguous title and, where relevant, a year, season, episode, or known schedule.

If languages or subtitles matter, choose an item whose available options you can confirm. Do not use an obscure or intermittent item for the first test.

The [connection-verification guide](/blog/verify-media-source-connection/) uses this reference to separate catalogue, metadata, and playback results.

## Gate 5: control the setup context

Write down:

- Norva account and profile;
- device type;
- visible app or browser version;
- general network context;
- date and local time;
- whether another source is already connected.

Use a device you control. On a shared device, understand session and credential storage before proceeding.

## The source-readiness worksheet

Mark each gate **ready**, **unknown**, or **blocked**:

| Gate | Status | Evidence or next action |
| --- | --- | --- |
| Permission is documented |  |  |
| Connection method is supported |  |  |
| Settings came from a primary record |  |  |
| Reference item is selected |  |  |
| Device and account are controlled |  |  |

Unknown is a legitimate result. It prevents a guess from becoming a hidden setup assumption.

## Plan secure credential handling

Before entry, decide how you will transfer a secret to a TV, keep it out of screenshots, remove it from an old device, and change it if exposed. Use only official forms and support routes.

Do not disable platform security, publish private links, or use an unverified converter merely to make settings fit.

## Keep the first connection narrow

Do not prepare every category or device. Connect one source, wait for a stable result, and verify the chosen item. Adding several sources at once makes duplicate and category issues harder to trace.

When all gates are ready, continue with [How to Connect a Compatible Media Source to Norva](/blog/connect-compatible-media-source-norva/).

## Limitations

This worksheet cannot interpret a licence or guarantee compatibility. Current product documentation, source terms, and situation-specific advice remain authoritative.

A correct connection can still contain incomplete metadata or unsupported media. Preparation creates a diagnostic baseline; it is not a playback guarantee.

## Frequently asked questions

### Do I need several test items?

One stable item is enough for the first connection. Add another only to test a different section such as series or live guide information.

### What if I know the password but not the source owner?

Pause. Possession of credentials does not demonstrate permission.

### Is a second device required?

No. Prepare one only if cross-device continuity is part of the current check.
