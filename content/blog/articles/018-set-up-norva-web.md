# How to Set Up Norva on the Web

> **In short:** Visit Norva through its official HTTPS site in a compatible browser, sign in to the intended account, connect one compatible source you are authorised to use, and verify one known item. The web version requires no app installation, but browser, device, source, network, and media conditions can still affect playback.

**Draft verification notice:** supported browser names, versions, screenshots, and exact labels must be confirmed before publication.

## Prepare a controlled browser session

Use a browser listed as compatible in current Norva documentation, on a device you control. Prepare your Norva account, one authorised source, and one known reference item.

On a shared computer, understand whether the browser stores passwords, sessions, or downloaded data. Avoid initial setup over a network you do not trust.

Norva states that its web experience opens in a compatible browser without installation.

## Step 1: verify the official destination

Enter [norva.tv](https://norva.tv/) directly or use a trusted bookmark. Check the HTTPS connection and domain before entering credentials.

Do not install an extension or executable simply because another page claims it is required.

**Observable result:** the official site loads and exposes its current account route.

## Step 2: sign in to the intended account

Use the current authentication flow. Confirm account and profile identity after sign-in, particularly on a household computer.

If the browser offers to save credentials, decide based on the device’s trust level and your password-management policy.

**Observable result:** the correct account context is visible.

## Step 3: connect one source

Follow the current source-management path and [the compatible-source connection guide](/blog/connect-compatible-media-source-norva/). Keep passwords and private addresses out of screenshots and public notes.

Browser extensions can affect requests or page behaviour. Record the extension environment before disabling anything, and do not broadly weaken security controls.

**Observable result:** the source is accepted or a specific error appears.

## Step 4: inspect a known item

Allow the catalogue to reach a stable state. Find the reference item and verify its title, year or episode, category, and source-supplied language information.

A partial catalogue is different from an empty connection. Record which sections load.

**Observable result:** one expected record is identified or its absence is documented precisely.

## Step 5: test playback

Open the item, select only visibly available options, and begin briefly. Check video, audio, controls, return behaviour, and progress where documented.

If catalogue browsing works but playback fails, note browser and version, operating system, extensions relevant to media or networking, sound state, and whether another known item behaves differently.

Do not publish speed or quality claims from a single item.

## Step 6: check account continuity

Add one favourite or save one progress point. If another supported device is available, check the same account state there after a normal reconnection.

The [first-session guide](/blog/norva-getting-started/) keeps this baseline deliberately small. The planned [browser use-case guide](/blog/when-to-use-browser-media-player/) will cover when web is preferable to mobile or TV.

## Sign out on shared computers

When finished:

1. sign out through Norva;
2. confirm the session closed;
3. remove saved credentials if they should not remain;
4. clear only downloads that belong to you;
5. review trusted devices from a controlled session.

Do not clear all data on someone else’s browser without permission.

## Browser troubleshooting boundaries

Private browsing can change storage and session behaviour, so it is not always a neutral comparison. An extension-free profile can be useful only when created safely and documented.

The public baseline for this draft does not provide a full supported-browser matrix. Publication remains blocked until a reviewer verifies it.

## Limitations

A successful test in one browser does not establish universal compatibility. Playback can depend on the browser, device, source, network, and media.

The player cannot create missing source metadata or language tracks.

## Frequently asked questions

### Must I install Norva for web use?

Norva states that its web version opens in a compatible browser without installation.

### Should I grant every browser permission?

No. Grant only what the official flow explains and your intended feature needs.

### Why can the catalogue load while video does not?

Catalogue retrieval and media playback are separate paths. Preserve device, browser, source, and item context before contacting support.
