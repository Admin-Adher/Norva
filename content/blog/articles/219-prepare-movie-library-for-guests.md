# How to Prepare a Personal Movie Library for Guest Browsing

> **In short:** Start guests in a bounded, neutral browse view rather than your full personal history. Define the authorized source scope, clear task-specific filters, prepare a varied set of verified movies, and decide which personal favorites, ratings, or progress should remain outside the decision. Test the exact TV, mobile, or web route, including focus, Back, search, filter reset, and playback handoff.

Guest browsing should make choosing easier without pretending the library is a public catalogue or exposing personal state that does not help the group.

## Create a neutral starting state

Clear old search text, narrow categories, availability toggles, and temporary filters. Confirm the correct profile or equivalent browsing context without assuming a guest mode exists.

If personal history or ratings would bias or expose unrelated choices, use a curated project list or separate profile where the current product supports it. Do not invent a product capability; verify available controls and choose the safest existing route.

## Curate breadth, not volume

Prepare a set with meaningful contrasts:

- familiar and new;
- shorter and longer options that fit the session;
- two or three broad categories;
- verified audio and subtitle choices;
- at least one ready fallback.

Avoid a hundred-item “shortlist.” Use [the movie-night funnel](/blog/plan-personal-movie-night/) to reduce the field before guests arrive while leaving enough agency for the group.

## Verify representative versions

For every finalist or featured row, check the current source, cut, runtime, and required tracks. Guests should not discover at launch that the displayed version lacks the needed subtitle or exceeds the available time.

For shared constraints, use [the shared-viewing filter contract](/blog/filters-for-shared-viewing-choice/) to separate accessibility needs and vetoes from preferences.

## Test the device route

On TV, map:

1. starting focus;
2. movement into the first row;
3. route to filters or search;
4. route to details;
5. Back behavior;
6. playback and return behavior.

W3C focus-visible guidance explains the need for a perceivable keyboard focus indicator. The same requirement is fundamental to remote navigation: guests must see where directional input will act.

[The large-library TV filter guide](/blog/filter-large-library-on-tv/) provides a D-pad route map. On mobile, test drawer close, scroll preservation, and touch targets. W3C mobile accessibility mapping covers operation across mobile input contexts.

## Explain only essential controls

Give a short orientation: move, open details, return, search or filter, and start playback. Let visible labels and focus states do the rest. If a filter's meaning requires a long explanation, simplify the starting view instead of training guests on the whole library.

## Protect the choice from accidental state changes

Decide who can modify favorites, ratings, progress, or saved filters. When the interface cannot isolate those actions, keep the host in control of destructive or persistent changes while guests direct the selection.

Afterward, record any progress intentionally, clear the temporary guest search or filters, and restore the prior personal context from a saved note.

Norva supports web, mobile, and TV experiences and can organize compatible authorized sources. Exact profiles, controls, and persistence should be verified in the current product and on the intended device.

## Common mistakes and limitations

- Opening the full library with stale personal filters.
- Presenting too many nearly identical choices.
- Failing to verify tracks and runtime.
- Assuming a separate guest capability exists.
- Ignoring Back and focus behavior on TV.
- Leaving guest filters active afterward.

## Frequently asked questions

### Should guests see personal ratings?

Only if they help the current decision and the account owner is comfortable sharing them. A neutral view is often clearer.

### How many choices should be prepared?

Enough to offer real contrast without recreating the full catalogue. Use the group's time and decision style to set the size.

### What if guests want to search outside the shortlist?

Provide a verified route to broader search while preserving source scope and a simple return to the prepared view.
