# How Much Storage Should You Reserve for Offline Video?

> **In short:** There is no reliable universal amount of storage per hour. Measure one or more representative eligible downloads on your own device, use their observed sizes to budget the trip, and preserve working headroom for the operating system and apps. Verify the final free-space figure after every download.

Duration alone does not determine file size. Resolution, bitrate, compression, frame rate, audio tracks, subtitles, and source packaging can all change the result. A short high-data file can be larger than a longer efficient one, so generic estimates may mislead.

## Start with actual device storage

Open the operating system's storage view and record:

- total device capacity;
- currently used storage;
- currently available storage;
- the media app's current storage footprint;
- any large categories you can safely review.

Apple provides a device-storage view under Settings, while Android provides storage and space-management controls whose names can vary by version and manufacturer. Use the current official instructions for your device.

Do not plan to consume every remaining unit. The operating system, app updates, temporary files, photos, messages, and normal app operations also need space. There is no single correct reserve for every device; choose a conservative buffer based on how the device is used.

## Measure a representative eligible item

Choose an item similar to what you intend to take: comparable duration and, when the app offers it, the same quality choice.

1. Record free storage before the download.
2. Download one eligible item completely.
3. Verify that it plays in a disconnected test.
4. Record free storage again.
5. Calculate the observed difference.
6. Repeat with a second representative item if sizes vary widely.

Use the app's reported size when it provides one, but compare it with the device-level change. Temporary files or system accounting can make the numbers differ, so treat the result as an observed planning range rather than an exact universal formula.

The [offline playback guide](/blog/offline-playback-explained/) explains why eligibility and disconnected verification come before storage planning.

## Build a simple storage budget

Use this relationship:

`planned local items + working reserve <= currently available storage`

For episodic content, sample at least two episodes if durations or quality differ. For mixed viewing, budget each class separately rather than multiplying one film measurement across every item.

If quality choices are available, measure the options you would genuinely use. A smaller display may not require the same choice as a large screen, but visual preference is personal. Do not lower quality solely on an assumed number; compare a short section on the actual device.

For the related network trade-off, see [how quality choices affect mobile data](/blog/video-quality-mobile-data-use/).

## Preserve operational headroom

A nearly full device can make downloads fail and can affect normal device operations. Before starting a batch:

- remove completed offline items through the app;
- review downloaded files from other apps;
- move personal photos only through a backup process you understand;
- avoid deleting system or app-managed files manually;
- complete pending operating-system updates before the trip;
- recheck storage after every few items.

The [offline download removal guide](/blog/remove-offline-downloads/) gives a safe cleanup order. If a device offers app offloading or archiving, read the platform explanation carefully: removing or offloading an app may affect its locally stored items.

## Common mistakes and limitations

- Multiplying a generic size-per-hour figure across an entire trip.
- Measuring a queued or incomplete transfer.
- Ignoring temporary storage or app updates.
- Assuming every episode has the same size.
- Filling the device to its displayed limit.
- Deleting app folders through a file manager.
- Confusing cloud capacity with free local device storage.

Storage reporting may take time to refresh. Restart the storage view or wait briefly before recording a result.

## Frequently asked questions

### Does higher quality always use more storage?

It often can, but the exact difference depends on the source files and encoding. Measure the available choices on your own device rather than assuming a fixed ratio.

### Is cloud storage available for offline playback?

Offline playback requires data available to the device. Norva states that eligible offline items are encrypted and stored on the device, not uploaded to Norva.

### Should I delete the app to reclaim space?

Use the app's offline-item removal controls first. Deleting or offloading an app can remove or alter local app data, and platform behaviour differs.
