# The Complete Guide to Searching a Personal Media Library

> **In short:** Write down what you remember, mark each clue certain or uncertain, and begin with the rarest reliable title fragment. Inspect the first results before adding a year, person, series, type, or language clue. If nothing appears, remove uncertain constraints, try verified alternate titles or name forms, and check filters and catalogue coverage. Change one query element at a time so every result teaches you something.

Search is a conversation between memory and metadata. The catalogue can only match fields and forms it actually contains, while memory often keeps a scene, face, translated title, or approximate year instead of the stored name.

## Create a clue inventory

Before typing, list clues under six headings:

| Clue | Example form | Certainty |
|---|---|---|
| Title | one unusual word or phrase | certain/probable/guess |
| Person | actor, director, creator, character |  |
| Time | exact year, decade, before/after event |  |
| Structure | film, series, season, episode, special |  |
| Language | original or localised title, audio, subtitle |  |
| Visual/story | artwork, location, premise |  |

Only some of these fields may be searchable in a particular interface. The inventory still helps you choose terms and inspect results.

## Use the memory-to-query search ladder

Climb one rung at a time:

1. **Distinctive title fragment:** enter the rarest correctly remembered word.
2. **Broader title form:** remove articles, punctuation, or uncertain words.
3. **Alternate access point:** try an original, translated, localised, or transliterated title.
4. **Person clue:** search a verified cast or creator name form where supported.
5. **Context clue:** add or filter by year, type, source, genre, or series only after seeing results.
6. **Catalogue diagnosis:** clear filters, check source availability, and investigate metadata when expected content remains absent.

Start at the highest rung supported by your most reliable clue. Do not type title, year, actor, genre, and plot into one long request unless the search interface explicitly supports those fields and operators.

## Know when to narrow or broaden

Narrow when results are numerous but relevant candidates are present. Add a high-confidence year, person, media type, or series context. Broaden when zero results appear or every result seems unrelated. Remove the least certain term first.

The Library of Congress distinguishes keyword, exact-phrase, and fielded searches and notes that filters can exclude records whose metadata lacks the filtered value. The same lesson applies broadly: a narrower query can improve precision but also hide a valid record.

Use [the exact-versus-broad decision guide](/blog/exact-vs-broad-media-search/) and [partial-title technique](/blog/search-with-partial-titles/) for query construction. Do not assume quotation marks, wildcards, or Boolean operators work in Norva unless current interface guidance confirms them.

## Inspect result evidence

Do not open the first familiar poster automatically. Compare:

- title and verified alternate title;
- release year or range;
- film, series, episode, or special type;
- creator or cast;
- synopsis premise;
- runtime and edition;
- source and version label;
- series and season context.

Use two or three independent clues. Artwork can be wrong, and two works can share a title.

## Keep a query change log

For difficult searches, record:

| Attempt | Query/filter | One change | Result count | Best clue learned | Next step |
|---|---|---|---:|---|---|
|  |  |  |  |  |  |

Changing one element per attempt reveals whether a title word, year, or filter removed the expected record. It also prevents repeating the same query with different punctuation and expecting a different result.

## Search episodes through hierarchy

When an episode title is forgotten, search the series first, then navigate season context. Use remembered guest person, plot word, approximate season, or neighbouring episode only where that metadata is present and searchable.

If the episode appears in global search but not inside the series, the problem may be a parent relationship rather than the query. Search should sometimes lead to a metadata audit.

## Handle names, numbers, and languages carefully

Try a person’s surname, full verified name, and known credited form separately. For numbered titles, test the number as digits and words, and remove punctuation if results are empty. For multilingual works, try both original and familiar localised forms.

Search behaviour differs across systems. The Library of Congress Search Help documents exact phrases, operators, and identifier rules for its own service; those examples demonstrate why users must learn the syntax of the interface they are actually using.

## Diagnose zero results systematically

Follow [the zero-results workflow](/blog/diagnose-zero-search-results/): clear filters, shorten the query, test alternate forms, verify the relevant source is connected and refreshed, and search a known control item. Separate “search did not match” from “record is absent or misidentified.”

Norva organises compatible authorised sources across supported devices, while searchable metadata and available records depend on those sources. Search does not provide a media catalogue by itself.

## Common mistakes and limitations

- Entering every remembered clue at once.
- Treating an approximate year as exact.
- Assuming punctuation and case are always significant or always ignored.
- Selecting by poster alone.
- Leaving restrictive filters active.
- Repeating the same failed query without changing a hypothesis.

No search technique can retrieve metadata that is absent, incorrectly matched, or outside the connected authorised source.

## Frequently asked questions

### What is the best first query?

Use the rarest correctly remembered title word. If you remember no title text, start with the most distinctive searchable person or series clue supported by the interface.

### Should I add the year immediately?

Add it when results need disambiguation and the year is reliable. An approximate or territory-specific release year can remove the correct record.

### When is the problem metadata rather than search?

Suspect metadata when a known control search works, filters are clear, the source is available, and the expected record appears under a wrong title, parent, identifier, or field.
