# Subtitles Missing? Nine Checks to Try First

> **In short:** Confirm that subtitles are enabled, select a specific track, test during clear dialogue, and verify the exact item and version. Then compare another track, item, or supported device one variable at a time. If the source does not provide the needed text track, a player preference cannot create it.

Missing text can result from a simple “off” state, an unsuitable scene, the wrong version, unreadable styling, or a track that is not present. The fastest route is an ordered check, not rapid changes across every menu.

If you are setting up a normal session rather than repairing one, start with [how to select subtitles with minimal interruption](https://norva.tv/blog/select-video-subtitles/).

## Run these nine checks in order

### 1. Confirm subtitles are enabled

Open the text-track control and make sure it is not set to “off” or an equivalent disabled state. Record the currently active entry before changing it.

### 2. Select a named track

Choose a specific language or caption entry. A language badge on a catalog card may only summarise availability; it does not prove that a particular track is active.

### 3. Test during clear speech

Move to a short passage with obvious dialogue. A silent scene cannot confirm whether dialogue text is working.

### 4. Wait for a complete line

Allow enough time for one full spoken sentence. Do not switch tracks repeatedly during the same moment, because you will lose track of which change produced the result.

### 5. Check visibility, not only presence

Look closely for text that may be difficult to see because of size, contrast, picture brightness, or viewing distance. The [caption-readability guide](https://norva.tv/blog/make-captions-easier-to-read/) helps when text exists but is uncomfortable to read.

### 6. Confirm the exact item and grouped version

Two entries with similar artwork can expose different track inventories. Return to the details and note the exact version you opened.

### 7. Try another listed text track

If another track exists, select it and replay the same dialogue passage. One working track suggests that text rendering is possible and narrows the issue to the original entry or its metadata.

### 8. Compare another item on the same device

Choose an item that visibly lists a text track, if available. Keep the profile and device unchanged. If subtitles work elsewhere, the first item or version becomes the stronger lead.

### 9. Compare one other supported device

Use the same account, profile, item, version, and named track on another supported screen. This is a diagnostic comparison, not proof that every device supports every media configuration.

## Record the result as a four-layer diagnosis

The nine checks form an original diagnostic card with four layers:

| Layer | What you tested | Evidence to record |
| --- | --- | --- |
| Selection | Enabled state and named track | Active label |
| Content | Dialogue passage, item, version | Title, version, timestamp |
| Rendering | Visibility and another track | What appeared on screen |
| Environment | Another item or device | Device and comparison result |

This record avoids an unhelpful report such as “subtitles are broken.” It lets support see whether the failure follows one track, one item, one device, or the account context.

## What the result means

- **No named text track appears:** the selected item or version may not expose one through the source.
- **Another track works:** the original track or its labelling is the narrowest lead.
- **Another item works:** focus on the first item and version.
- **The same item works on another device:** record the device-specific difference without assuming a cause.
- **Nothing works in either comparison:** escalate with the completed card.

Norva can retain subtitle preferences when relevant options exist. It cannot add a missing language or caption track. The W3C distinguishes captions as text that includes dialogue and important non-speech audio; a generic subtitle label may not indicate that fuller purpose.

## Common mistakes and limitations

- Testing only a scene with no dialogue.
- Assuming a badge is the complete track list.
- Changing audio and subtitles at the same time.
- Comparing different profiles or grouped versions.
- Treating tiny or low-contrast text as absent without checking closely.
- Reinstalling before recording the active track and item details.

Language and text availability depend on the media and authorised source. Norva does not guarantee a specific subtitle language for every item or supported device. For audio selection context, see [how to choose the right audio track](https://norva.tv/blog/choose-audio-track/).

## Frequently asked questions

### Why does a subtitle badge appear when no text is visible?

The badge may summarise metadata rather than the active state. Open the full selector, choose a named track, and test a dialogue passage.

### Can I add a missing track through a saved preference?

No. A preference can select among available options but cannot create a track that the item and source do not provide.

### What should I include in a support request?

Provide the device type, profile, exact item and version, selected track label, dialogue timestamp, and results from another track, item, or supported device. Never include passwords or source credentials.
