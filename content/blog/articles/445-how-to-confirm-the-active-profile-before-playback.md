# How to Confirm the Active Profile Before Playback

> **In short:** Before pressing play, read the active profile's text label, then confirm the item, version, proposed progress, and language preferences fit that context. Use colour or avatar only as supporting evidence. If the profile is unclear, return to the visible selection screen and resolve it before playback; do not use history cards to guess.

A few deliberate checks prevent activity from landing in another household member's history. The check should happen before play or resume, not after an unexpected position appears.

## Start from the account layer

Confirm that Norva is signed into the intended account on the supported device. Account sign-in and profile selection are separate. A familiar library does not prove the active profile.

On a borrowed or untrusted device, review whether signing in is appropriate before exposing account context.

## Read the text label

Open the visible profile context or selection screen and read the label. Treat icon, colour, and row position as secondary clues because they may change or appear similar.

W3C consistent-identification guidance reinforces stable identification of repeated functions. Actual profile presentation must be observed in the current Norva interface.

For shared TVs, [clear profile naming](/blog/how-to-name-profiles-clearly-on-a-shared-tv/) makes this step more reliable from the normal viewing distance.

## Cross-check the candidate item

Before playback, ask whether the displayed state makes sense for the chosen profile:

- expected title or episode;
- plausible progress;
- relevant favourites;
- expected language or subtitle preference;
- intended version.

These are consistency clues, not proof. A profile with little history may show limited context, while two profiles may watch the same series.

## Confirm focus before activation

On TV or keyboard input, identify the visibly focused play or resume control before pressing Enter or the remote selection key. W3C focus-visible guidance explains why keyboard focus should be perceivable.

On touch, wait for motion to stop and tap the clear primary control once.

## Stop when context is unclear

If no profile label is visible, do not begin playback to discover whose history changes. Return through documented controls to profile selection or a stable account area.

If the interface repeatedly hides the active context, capture a privacy-safe screen and contact support.

## Run the preflight

Use this order:

1. account;
2. profile label;
3. item or episode;
4. version;
5. progress;
6. audio/subtitles;
7. focused or targeted play control.

The [complete household profile guide](/blog/the-complete-guide-to-household-media-profiles/) places this sequence within naming, shared-screen, and review rules.

## If the wrong profile is selected

Return to profile selection before playback. If playback has already started, pause immediately and document the item, version, and position. Follow [the wrong-profile progress prevention guide](/blog/how-to-prevent-progress-from-landing-on-the-wrong-profile/) rather than clearing another person's history.

Do not change favourites or preferences while the profile identity remains uncertain.

## Common mistakes and limitations

Avoid identifying by avatar alone, trusting a continuation card, treating account sign-in as profile confirmation, and pressing play to test context.

Profiles organise viewing state but do not provide account security or parental controls. Current plan capacity, device support, source access, and media tracks remain separate.

## Add a check after interruptions

Repeat the profile preflight after sign-in, app restart, device wake, a household handoff, or an update. These boundaries can return the interface to a default or previously active context. Confirm the visible text label again rather than assuming the last selected profile persisted.

If no unambiguous label is visible, pause playback and return to the verified selector instead of testing with someone else's progress.

## Frequently asked questions

### Is the profile avatar enough?

No. Use the text label as primary evidence and the avatar as a supporting clue.

### What if the profile has no history yet?

Confirm the profile label directly. The absence of history is not a reason to borrow another context.

### Should every device remember the last profile?

Do not rely on that assumption. Verify the active profile on each supported device before playback.
