# How to Investigate a Favorite Saved to the Wrong Profile

> **In short:** Do not remove the item immediately. Confirm the account, inspect each relevant profile, and match the exact item or version before recording its favourite state. Identify who intended to save it and on which device the action occurred. Then make one deliberate change per profile, verifying the visible result after each activation.

A misplaced favourite is an attribution problem. Similar artwork, grouped versions, and wrong-profile entry can make it appear that an item moved when a different variant was saved.

## Preserve the first state

Record:

- account;
- currently active profile;
- full item identity;
- version or source label;
- favourite control state;
- device;
- approximate time of the action;
- who intended to save it.

Do not toggle the favourite control while gathering evidence.

## Inspect the suspected wrong profile

Open the profile directly and find the item from the library or favourites section. Confirm title and year, or series and episode context, plus version when relevant.

W3C consistent-identification guidance supports stable identification of repeated controls, but the current favourite icon and active state must be observed in Norva.

## Inspect the intended profile separately

Switch through documented controls and repeat the same identity check. Do not assume the absence of one card means the entire profile state has refreshed; return to a stable list or detail screen.

Use [the active-profile preflight](/blog/how-to-confirm-the-active-profile-before-playback/) before any correction.

## Distinguish three outcomes

- **Wrong profile only:** saved under the unintended profile and absent from the intended one.
- **Both profiles:** each currently shows the favourite.
- **Identity mismatch:** profiles refer to different versions or similarly titled items.

Only the first outcome is a simple attribution correction. The others require an ownership decision or more identity work.

## Agree on the intended state

Ask the affected profile owner before removal. Favourites may be genuinely shared even when the original action was accidental.

Do not use another person's profile as a temporary planning list without agreement.

## Apply one change at a time

Let the screen settle, activate the visible favourite control once, and confirm its state. Then switch to the intended profile and add the exact item only if that is the agreed result.

For a larger review, use [the tablet favourites workflow](/blog/how-to-review-favorites-on-a-larger-touch-screen/), which separates keep, verify, and remove decisions.

## Investigate repeat incidents

If favourites repeatedly land on the wrong profile, review profile names, shared-screen entry state, focus visibility, and who hands off the device. The problem may be profile recognition rather than the favourite control.

The [complete household profile guide](/blog/the-complete-guide-to-household-media-profiles/) provides naming and shared-state rules.

If the visible control changes the wrong item despite confirmed identity, preserve exact steps and contact support.

## Common mistakes and limitations

Avoid toggling before documentation, identifying from artwork alone, removing another person's favourite without consent, and assuming all grouped versions share one favourite state.

Source metadata, interface refresh, and product behaviour can vary. This workflow establishes visible attribution but cannot guarantee restoration of an earlier state.

## Check propagation before editing again

After one authorised correction, wait for the normal supported synchronization path and inspect the same item on the original device and one relevant second device. Record profile label, item identity, timestamp, and result. Repeated add-and-remove actions can obscure whether the first change propagated.

If devices disagree, preserve both views and escalate the bounded evidence rather than rebuilding several profiles or sources.

## Frequently asked questions

### Can the same item be a favourite on two profiles?

It may appear in both when each profile saves it. Verify each profile independently rather than assuming duplication is an error.

### Should I remove the wrong favourite first?

Preserve evidence and confirm ownership first. The correct order depends on the agreed final state.

### What if the favourite icon is unclear?

Do not activate it. Capture a privacy-safe screen and consult current product help or support.
