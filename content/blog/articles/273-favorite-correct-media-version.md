# How to Favorite the Correct Version of a Title

> **In short:** Confirm the work first, then compare edit, release context, runtime, source, audio, subtitles, and device behavior. Save the version that satisfies the future action, leave the detail page, and reopen it from favorites. If favorites resolve only to a work-level page, record the required version separately instead of assuming it will be selected later.

Two cards with the same poster can lead to different edits, tracks, or source availability. A favorite that preserves only the title may fail the exact reason it was saved.

## Confirm the underlying work

Compare stable identifiers, media type, release year, creator or series relationship, and title variants. EIDR’s public hierarchy distinguishes creative work levels, edits, and manifestations. Do not compare language tracks until both cards are shown to represent the intended work.

## Compare version attributes

Check:

- edit or cut;
- runtime;
- region or release context;
- image quality label only as supplied;
- spoken-language tracks;
- subtitle and caption tracks;
- audio-description or other accessibility attributes only when explicitly identified;
- source and current availability.

DCMI metadata terms distinguish format, language, identifier, and relation. Keep those fields separate rather than inventing a single “best version” score.

## Tie the choice to intent

Examples:

- follow the version with required subtitles;
- compare two edits later;
- rewatch a specific dub;
- preserve the version compatible with the household TV setup.

The right favorite is the one that fulfills the stated action, not necessarily the card with the most badges.

## Test on the target device

Open the version on the device that matters. Confirm the version label, selectable tracks, and playback readiness with an authorized source. A metadata badge can be incomplete or stale. A short playback check confirms selection, not the full editorial quality of every track.

Norva can group versions and retain language or subtitle preferences across supported devices, subject to current product behavior. Actual tracks depend on the source and media.

## Save once and reopen

1. Note the current favorite state of both versions.
2. Save the intended one.
3. Leave the detail page.
4. Open favorites from a clean route.
5. Confirm which entity or version opens.
6. Check required tracks again.
7. Compare another supported device if cross-device retrieval matters.

If both versions appear or the wrong one opens, use [the duplicate-favorite diff](/blog/resolve-duplicate-favorites/).

For a version-sensitive choice, repeat the clean-route retrieval after signing out and back in or after a normal refresh. The saved result should still expose the source and required track evidence; otherwise record the favorite as work-level or unresolved, not version-confirmed.

## Handle work-level favorites

Some interfaces may save only the work and choose a version later. If verified behavior works that way, keep a private minimal note with version identifier, source, required language, and review date. Do not claim version-specific persistence.

## Respond to later unavailability

If the version disappears, preserve identity and use [the unavailable-favorite tree](/blog/handle-unavailable-favorites/). Another version should replace it only when it still meets the future action.

If the favorite itself seems missing after regrouping, use [the recovery map](/blog/recover-missing-favorite/) before adding again.

## Common mistakes and limitations

- Favoriting by poster or title alone.
- Comparing tracks before confirming work identity.
- Treating more badges as universally better.
- Assuming work-level saves preserve version choice.
- Testing only on a different device.
- Re-adding after regrouping without checking duplicates.

## Frequently asked questions

### Should two useful versions both remain favorites?

Yes, when each supports a distinct documented action and the interface preserves their identity.

### Does a language badge prove track availability?

No. Inspect the exact version’s track list and test selection on the intended device.

### What if the favorite always opens the default version?

Treat the save as work-level and preserve the required version in a separate minimal note or verified product mechanism.
