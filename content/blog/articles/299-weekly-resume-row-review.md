# A Weekly Review Routine for Continue Watching

> **In short:** Once a week, verify the active profile, surface the next genuine sessions, confirm completed entries, triage deliberate samples, protect other viewers’ state, and give unresolved cards one specific question. Stop after a short time box or the first unexpected action. The routine succeeds when choosing what to resume becomes easier—not when the row reaches a target length.

A weekly cadence keeps intention fresh enough to remember while avoiding constant maintenance. It is a review meeting with your future viewing choices, not a demand to clean every card.

## Prepare the review context

Choose one account, profile, and supported screen. Record the local date, visible profile, and first few cards. If the screen was recently offline or another profile was just active, resolve that context before reviewing.

Use a private note with four columns: Item, State, Evidence, Next Review. Avoid credentials and unnecessary viewing details.

## Follow the six-step agenda

### 1. Surface next sessions

Identify titles with a named viewer, exact version, and realistic near-term plan. Use [the active-item priority ladder](/blog/prioritize-active-resume-items/) to separate next session, active soon, deliberate pause, and unresolved.

### 2. Confirm completed entries

Do not clear from memory alone. Compare exact title or episode, version, endpoint, and visible row state. When completion looks wrong, preserve a checkpoint for diagnosis.

### 3. Triage samples

Ask whether brief playback had preview-only intent, known ownership, and a closed decision. Apply [the sampled-title card](/blog/clear-sampled-titles-from-resume-list/) before using any current row action.

### 4. Protect shared state

On shared screens, label entries personal, shared, or unresolved. Ask before changing another viewer’s state. A long row with known ownership is safer than a short row produced by guesses.

### 5. Name each unresolved question

Use one of four prompts: Which profile? Which version? Which episode? Is there still return intent? Record only the missing field and a review event.

### 6. Verify one action

Choose a non-sensitive, confirmed candidate. Apply one action whose effect is understood in the current build, reopen the row, and compare with the baseline. Stop if the result is unexpected.

## Keep the routine time-boxed

Ten minutes is a practical agenda size, not a performance benchmark. End earlier if uncertainty grows. If the row is very long, process one screenful and use a separate investigation session for exceptions.

W3C focus-order guidance emphasises sequences that preserve meaning and operability. Apply that idea to the agenda: next-session clarity comes first; destructive cleanup comes only after identity and scope.

## Track useful outcomes

Record outcomes that improve future decisions:

- active items with a concrete next action;
- completed items with verified state;
- samples confidently closed;
- shared entries with an owner;
- unresolved entries with one question;
- unexpected actions requiring support.

Do not turn the counts into product claims. They describe one viewer’s maintenance session. DCMI metadata distinctions help keep item identity compact and consistent.

Finish with [the Continue Watching quality checklist](/blog/continue-watching-hygiene-checklist/) after a source, device, or household change; the weekly routine need not repeat the full audit every time.

## Original evidence: weekly agenda

Run the six steps for four weeks and record only whether the next intended title was identifiable at the start, how many entries remained unresolved, and which missing field dominated. Do not publish the data as a general benchmark.

The comparison shows whether the routine improves this household’s decision clarity. If it becomes burdensome, shorten the active window and address the dominant ambiguity instead of adding more review steps.

## Common mistakes and limitations

- Reviewing several profiles in one undifferentiated pass.
- Clearing by age or percentage.
- Treating the timer as pressure to guess.
- Recording sensitive reasons in a shared note.
- Repeating a failed action.
- Measuring success only by card count.

The routine cannot fix incorrect source metadata or create unavailable media, languages, or subtitles.

## Frequently asked questions

### Must the review happen on the same day each week?

No. A consistent trigger helps, but any cadence that keeps intent understandable can work.

### Should every household member attend?

Only when shared ownership is unresolved. Personal profiles can be reviewed by their owners.

### What if no cleanup is needed?

Confirm the active next sessions and stop. Maintenance should not create work when the row is already useful.
